Snippet Locations

Store snippets in the default Flare location, or in the dedicated snippet folder for the solution or platform.

Default location:

  • Content > Resources > Snippets

Platform locations:

  • Content > Platform > AIS > AIS Snippets
  • Content > Platform > RCM < RCM Snippets

Solution specific locations:

  • Content > AML > STAR > STAR Snippets
  • Content > AML > SAM > SAM Snippets

To create a new snippet:

  1. Search the entire project for reusable content from other LOBs or from Platform.
  2. Highlight the content, right-click and select Create Snippet.
  3. In the Snippet File field, give the snippet a name.
  4. Select the Project Folder to save the snippet.
  5. Select Replace Source Content with the New Snippet.
  6. Insert the new snippet into your topic. (Copy it from the source topic, use Insert > Snippet, or Ctrl + R)
  7. Fill in the Snippets Usage Table.
  8. Let the source content owner know you create a new snippet.

Snippet Tips and Guidelines

Here are some basic principles to follow when reusing content.

  • Do not link your TOC directly to a topic from another LOB.
    • This is to prevent you changing the original content by mistake without the original author realizing!

    •  

  • Do not use conditions inside of snippets (or at least avoid it) .
    • If you use conditions in the snippet, every target using the snippet has to be updated with include/exlude settings.
    • If nothing is set in the target, the conditioned content will appear in each guide that contains the snippet.

    •  

  • Do use the snippet as is.
    • 100% of the snippet's content should be relevant.
    • If not, consider breaking the content into smaller snippets.
    • If not, consider converting the snippet to text and modify as needed.
