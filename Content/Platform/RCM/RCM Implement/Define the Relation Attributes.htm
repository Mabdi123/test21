Define the Relation Attributes for the Alert Type

This tab also enables you to define the child alert type for each relation type and configure the specific target field of the CWI. You can also define whether or not the consolidation is transient or standard.

Go to the Related Item Types tab of the CWI Alert Type. Note that a CWI can have a parent CWI.

To add a related item type association:

  1. Click the Add New Relation Type Association icon.
  2. Select the relation type. The Case Items Relation is available for parent items that have the Case Items alert type category. The child object associated with the parent cannot have the Case Items alert type. The Case Items Relation type cannot be edited or deleted.
  3. Select the child object for this association. In this configuration, the child object is the TWI or child CWI. Two alert types can be related by more than one relation. For example, a person can be related to an account as Owner, as Payee or as a Suspect.
  4. Select whether or not to enable/disable audit propagation. When audit propagation is enabled, the history audit of the consolidation is logged in the CWI and the related standard TWI.

  5. Select one of the following options from the Supports Consolidation field:

    1. None – The consolidation does not occur.
    2. Standard – The TWI is consolidated to a CWI, and the TWI is generated as a separate and viewable work item.
  6. Transient – The TWI is consolidated to a CWI, but the TWI is not generated as a separate and viewable work item. The following message is displayed when the user adds a manual work item that is defined as a Transient TWI: This option is not available for the Case Items Relation type.

If a TWI has a standard consolidation relation to one CWI and a second transient consolidation relation to another CWI, then the TWI will be created but it will not be related to its transient CWI. Any update of the TWI will not be updated in the transient CWI as they are no longer related.
