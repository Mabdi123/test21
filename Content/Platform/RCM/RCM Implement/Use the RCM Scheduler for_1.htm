Use the RCM Scheduler for Workflow Deadlines Jobs

The RCM Scheduler has a workflow escalation job that may be used instead of the Workflow Asynch Execution Tool.

This job of the RCM Scheduler checks the deadlines and deadline actions on a regularly set interval. It is controlled by the following configuration parameters in the Config Parameters tab of admin.jsp:

  • actimize.workflow.deadline.auto.enable – A Boolean (Yes/No) parameter that defines whether or not the deadline checking Job is enabled. By default the parameter value is ‘false’ (disabled).
  • actimize.workflow.deadline.checkInterval – An integer value that defines the interval in minutes between executions of the deadline checking job. This value must be greater or equal to 1 minute and has a default value of 15 minutes. Note that an upgraded RCM to version 5.1 has the value of 1440 minutes (24 hours). Only one instance of this job runs at any time and therefore it only occupies one thread on the scheduler.
  • actimize.scheduler.user – Defines the authorized RCM user (default is admin). Failure to define this parameter or to set a user that does not exist results in an error in the job. The RFI job also uses this parameter and when this parameter is not defined (empty) then its value is the user that enabled the RFI feature.

For details see The Scheduler.
