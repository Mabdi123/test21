Visual Story View Framework

The Visual Story View is a framework for providing a uniform user interface across solutions when working with work items in RCM. It is comprised of UI components that use an XML layout configuration to control the UI.

The Visual Story View is installed in the RCM as follows:

VFS - app_gui_items\rcm\workItemDisplayInfra

RCM WAR - app_gui_items\rcm\workItemDisplayInfra

The figure below shows the files required to implement the Visual Story View.

Make sure you select the Use Visual Story View checkbox for the alert type. For details see Location of XSL Files.

  • The singleview.xsl file contains the configurations of the Visual Story View (singleview) XML files and the Work Item Display Infra XSL files. It is set in the RCM Designer, and is associated with an Alert type in RCM.
  • The singleview.xml file contains the configuration (layout and contents) of each section of the UI.
  • The workItemInfraDisplay.xsl file in the RCM Work Item Display Infra package renders the HTML display. The workItemInfraDisplay.xsl file refers (via the singleview.xsl file) to the Visual Story View XML files that hold the configuration values, thereby controlling what is or is not displayed, and how the displayed items appear on the screen. Likewise, the workItemInfraPDF.xsl file refers (via the solutionPDF.xls file) to the same XML files that hold the configuration values, thereby controlling what is or is not displayed, and how the displayed items appear in the PDF.

The following files can be used to modify the configuration values in the Visual Story View:

  • additions_1.xml – This file is written by the implementer. Its name must have the format <name>_additions_1.xml. It contains parameter values that override some or all of the settings in singleview.xml, and may contain additional items to display (not included in singleview.xml) and their settings. See The Additions XML Files.
  • additions_2.xml – This file is written by the customer implementer. Its name must have the format <name>_additions_2.xml. It contains parameter values that override some or all of the settings in singleview.xml or additions_1.xml, and may contain additional items to display (not included in singleview.xml or additions_1.xml) and their settings. See The Additions XML Files.

The additions XML files are merged with the Visual Story XML file, overwriting existing values with custom values and adding any new elements, fields and values.
