Using the RCM Designer to Setup Report Definitions for RCM

Once you have a report defined for use you can setup the RCM Report Definitions for this report using the RCM Designer.

This process includes the following tasks:

  • Define required display categories that appear within the RCM Reports module (e.g., Management Reports, Supervision Reports, etc.)
  • Provide a report with a name, description and identifier
  • Indicate the provider of the report (e.g., JReport, External File, etc.)
  • Select the appropriate report policy, materialization format and archive format
  • Indicate whether the report will be available for on-demand generation of reports from the RCM.
  • When JReports are used:
  • Select the appropriate JReport definition file you want included in a specific category
  • Select the appropriate JReport catalog file for the report
  • Map any JReport parameters to Business Unit Attributes.

The current version supports only JReport as the report provider.
