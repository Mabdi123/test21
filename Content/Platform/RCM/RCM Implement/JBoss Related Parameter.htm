JBoss Related Parameter

Parameter name

Default value

Description

actimize.cache.disableEviction

False

RCM uses the JBoss Cache for storing frequently used objects during runtime operation.
In the default configuration JBoss Cache uses different algorithms for evicting cached objects based either on time limits or maximum number of objects allowed.
When parameter is enabled it is used to cancel any limits for cached objects. An object will reside in the cache until RCM determines to delete it.
