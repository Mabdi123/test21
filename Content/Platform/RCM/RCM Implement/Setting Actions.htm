Setting Actions

The Settings Activity Bar includes the following actions:

Action

Description

New Setting

Opens the New Settings wizard.

Select Setting to enable IT professionals to configure the application for connection and use of thresholds, lists and reference information.

The New Settings wizard includes the following options (according to the type of setting selected):

  • Select the type of setting – Threshold, List, or References
  • Select the alert type associated with the setting
  • Provide a name for the new setting and a title for the page associated with that setting
  • Define the connection to the appropriate database
  • Once the connection is selected, you can select the required table from this database 
  • Define or edit the SQL query required to retrieve information

When you enter a Select statement with a Join, you must select Manual SQL for the Select table. This will automatically change the display options that are available for the users, to Read only.

Use advanced options that enable you to:

  • Define a subtitle for the activity bar
  • Enter an SQL script that can be run from a Restore Default button from the RCM Threshold/Lists/References module
  • Enable the Import xsl/csv option that can be run from the RCM Threshold/Lists/References module

For more information about the advanced features, see Advanced Settings)

Select Conditional Settings Template in order to create a template for a Conditional Setting.

  • Provide a name for the new setting, identifier and a title for the page associated with that setting
  • Define the connection to the appropriate database
  • Once the connection is selected, you can select the required table from this database 
  • Define or edit the SQL query required to retrieve information. The conditional setting that is at the end of the SQL query has the following structure:
    {$<parameter name>:<default value of parameter>.
    In this example the default value of the parameter parameter_name is 10:
    {$parameter_name:10}
    Single quotes should be entered at the beginning and the end of the setting when the parameter is not numeric (e.g., ‘{$name:John}’)

When you enter a Select statement with a Join, you must select Manual SQL for the Select table. This will automatically change the display options that are available for the users, to Read only.

Use advanced options that enable you to:

  • Define a subtitle for the activity bar.
  • Enter an sql script that can be run from a Restore Default button from the RCM Threshold/Lists/References module.
  • Enable the Import xsl/csv option that can be run from the RCM Threshold/Lists/References module.

For more information about the advanced features, see Advanced Settings)

Set Full Access, Read only, Update Existing Entries display features which determine the option buttons that appear in the upper right corner of the RCM end user application as follows:

Full Access – Displays Add, Delete and Edit option buttons, thereby enabling RCM end users to add or delete rows (as well as edit values) for a threshold, list or reference table.
(For example, when users are requested to define the upper and lower limits for transactions according to different score values. In this case, the user can add a new row in order to specify threshold values for a score that does not appear within the table. Users can also select any row and edit the values, or delete any row.)

Read Only – Displays only option buttons that can be performed in a read only mode, such as Print table information or display History.

Update Existing Entries – Displays Edit option button but does not display Add or Delete buttons, thereby enabling RCM end users only to edit values within a threshold, list or reference table.
(For example, if the users are requested to define the upper and lower limit of a specific AIS variable, there is no meaning to adding a row to the threshold table. In this case the user can only edit the value of the thresholds).

Set up validation rules that enable the application to check all inserted data for:

  • Data Type
  • Field Length
  • Sign (for numeric)
  • Date

Edit Setting

Enables authorized personnel to edit selected setting definitions.

The settings definitions must be synchronized with the AIS server in order for the AIS to retrieve the appropriate threshold values that are defined within the Risk Case Manager application. Changing these settings can affect the ability of the application to receive alerts and, therefore, should only be done in consultation with authorized Actimize IT professionals.

Delete Setting

Deletes a selected Setting definition. This action should only be done in consultation with authorized Actimize IT professionals.
