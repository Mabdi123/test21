Date Formatting

It is vital that dateInputFormat and dateDisplayFormat should be in the same order. For example, if you change dateDisplayFormat from MM/dd/yyyy to dd/MM/yyyy then you must change dateInputFormat from M/d/y to d/M/y.

If the dateInputFormat is: M/d/y and the dateDisplayFormat is MM/dd/yyyy then the following user input samples are translated to 01/07/2005:

  • 1/7/05
  • 01/7/2005
  • 01/07/2005

Date format guidelines:

Format

Details

d

Accepts day of month in one or two digits

dd

Accepts day of month only in 2 digits

M

Accepts month in one or two digits

MM

Accepts month only in 2 digits

y

Accepts year in 2 or 4 digits format

yy

Same, accepts only 2 digits

yyyy

Same, accepts only 4 digits

The flow of the date-format resolution process is as follows:

  1. If no formatter information is supplied then the configuration parameter date format is used (either date or datetime, depending on the field’s data type).
  1. If the formatter information contains one of the built in constants (acm_date, acm_time & acm_datetime) then the appropriate format is used accordingly.
  1. If the formatter contains another string then that string is used as the specific date format (for example: mm-dd-yyyy).
