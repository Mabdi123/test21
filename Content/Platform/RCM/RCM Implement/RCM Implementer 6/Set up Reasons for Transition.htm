Set up Reasons for Transition

This feature enables the user to provide more information for the transition process.

The user can select a reason from the list of displayed reasons as displayed in the following.

Before defining the reasons, you must create and fill in at least one list of reasons in the Platforms List module. For more information on Platform Lists refer to Setting up Platform Lists and the Platform Lists chapter in the RCM Reference Guide.

The reason feature is related to the alert, not to the current step. If a reason has been previously selected when changing from step A to step B, (meaning the alert already has a reason assigned to it), then it is not mandatory anymore. It is mandatory only if the user has not selected any reason previously.

To setup the reasons for a transition:

  1. Select the reasons List checkbox and select a Reason List from the drop-down menu. These lists are created via the Platform Lists module. Click the Refresh button to view an updated list. Click the Open () button in order to view and optionally modify the selected Reason list.
  2. Select the reason code field and reason name field from the fields of the selected Platform List. These are the Platform List’s view fields. Only string fields can be selected.
  3. Optionally, select the mandatory checkbox in order to make it mandatory for the user to select a reason in order to move to the next step in the workflow.
  4. Optionally, filter the reasons that can be viewed by the user via the filtering table.

For platform lists that require four eye review, only approved reasons are displayed to the user that changes the step and selects a reason.
