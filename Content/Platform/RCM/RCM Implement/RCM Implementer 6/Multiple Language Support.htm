Multiple Language Support for User Defined Custom Fields

NOTE:

This functionality is not supported in RCM versions 5.1 and later. Existing data should be migrated via the RCMMD module during dbupgrade.

In order, for example, to support the translation for the User Defined Custom field Execution Date, you need to run the following SQL command in the database of RCM (e.g. SQL server, Oracle, etc.):

insert into acm_md_resource_strings (resource_string_identifier, base_value, resource_string_max_length, module_name, fl_common, fl_internal) values ('myrs.customFields.executionDate','Execution Date',18, ' RCM ', 0, 1)

  • resource_string_identifier: an identifier that you specify. Verify that it is unique.
  • base_value: the English word that is translated
  • resource_string_max_length: the English word's length multiplied by 1.3. In this example it is 18 (14*1.3=18).
  • module_name: RCM (constant)
  • fl_common: 0 (constant)
  • fl_internal: 1(constant)

After inserting the SQL command, open the Designer and use the identifier as the custom field display title. In this example you should enter: ${myrs.customFields.executionDate}

The translated custom field appears in the related alert views.
