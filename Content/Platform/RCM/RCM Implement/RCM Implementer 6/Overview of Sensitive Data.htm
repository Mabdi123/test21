Overview of Sensitive Data Fields

This feature is currently available for alerts custom fields.

When the RCM application user views the work item's history of actions (audit), the sensitive data fields are always masked, regardless of user permissions. Unauthorized users view a different value for these fields that is either part of the data (for example, the last four digits of a credit card number) or a constant (for example, xxxx instead of the actual credit card number).

An RCM application user without sensitive data fields’ permissions can edit only the non-sensitive fields of the work item, and a user that has permissions to view sensitive fields can edit those fields.

Refer to the RCM Self Developer Guide for information on the API for masking a value according to sensitive data permissions.
