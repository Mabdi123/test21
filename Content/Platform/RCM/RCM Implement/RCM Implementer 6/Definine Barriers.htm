Define Barriers

By default a transition has no barrier (Barrier weight equals 0). In order to provide a barrier weight of 1 to a transition, select the Mark as barrier checkbox. In order to define a barrier weight that is greater than 1 you should also select the Require at least x users on this Work Item to advance to the target step checkbox and enter the exact number of users.

This displays the properties of a transition that has a barrier weight of 2 - at least 2 distinct users are required to review the step before a barrier is removed.

NOTE:

It is possible to define up to 10 barriers per transition.

Deadlines\Global Deadlines

Deadlines and global deadlines bypass all barrier rules and therefore when any type of deadline occurs, the transition is made regardless of the various barrier settings in the workflow.

Changing or Updating a Step via a Web Service

The ChangeAlertStatus or updateAlert Web Services use administrator rights and therefore bypass all barrier rules.

Changing a Step via a Plugin API

If a step transition is forbidden via barrier rules then the transition fails when a Plugin API attempts to move to the relevant step.

Behavior for Bulk Actions

  • Changing multiple steps from Alerts table: If one or more of the marked items is blocked due to review rules then the respective steps in the change step window are grayed out due to review rules for all the marked items and the bulk action cannot be performed.
  • Change multiple via from Plugin API: If for example, you attempt to change a step to two alerts at once, review rules and the review rules allow only the first transition then the first transition succeeds and the transition for the second alert fails (An error appears in the related logs about the second alert failed transition).
