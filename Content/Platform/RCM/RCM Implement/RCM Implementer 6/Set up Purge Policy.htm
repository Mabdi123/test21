Set up Purge Policy

 The Purge policy determines when the materialized report should be purged (i.e., when the materialized file is removed from the RCM). This is different from user deletion of a report. When a user deletes a report, the report remains on-line for the amount of time stipulated by the archive policy. The materialized instance of the report is only removed from the system according to the Purge policy.

The options that are available within the Purge Policy page are:

Option

Description

Purge X days after the report is archived

When this policy is selected, the materialized report can be purged at the specified amount of time after the report is archived. The reports are purged by running the CMPeriodicMaintenance utility, which purges the reports not earlier than the time specified in the purge policy. (For more information about this utility, refer to your RCM Installation Guide).

This option is only relevant if you select, in the previous Archive page, that the report definition supports archiving. If you indicated that the reports should never be archived then the option to purge after archive is not applicable.

Purge X days after the report is deleted

When this policy is selected, the materialized report can be purged at the specified amount of time after the report is deleted by the user.

The reports are purged by running the CMPeriodicMaintenance utility, which purges reports that were deleted after the time specified in the purge policy.

Never purge

When this policy is selected, the materialized report is never removed from the RCM.
