Location of XSL Files

If you specify that the alert details are retrieved using an XML, then you must specify the XSL location.

  • Select the Use default display for case item details checkbox to use the default display for case items in the RCM UI. This includes all default display elements in the Case Details page.

Print Details is not supported for case item details. You can only export to PDF.

To specify a location of the XSL:

Display Tab

In this tab, you can customize how the alert will be displayed on screen

  • Enter the XSL file name that is required by the XML file, specified above. The relative path is presented within the fields in this section. You must add the full file name for the XSL. Make sure that this file is, in fact, located in the specified location.

If you want additional properties to be added to the Alert Type then select the following checkbox: .

When this checkbox is selected the XSL implementer can use the different properties of the alert details.

If you want to view the exact structure of the enriched XML, do not specify an XSL in the RCM Designer. When opening the alert details screen for an alert of that specific type, RCM displays the alert enriched XML. For more information about XML/XSL in RCM, refer to the RCM Self Developer Guide.
