Create a New Language and Assigning General Information

You can create a new language as follows.

To create a new language:

  1. From the Languages tab, click the icon.

The screen on the right side changes to include the following tabs: General, and Translation tabs.

  1. Using the options within the General tab, provide a name for this language.
  1. Enter the language Identifier that you want to appear within the RCM activity bar.
  1. Provide a description, if required. You can access a larger text area for the description by clicking the Edit button beside the Description field box.
  1. Select the relevant language code (e.g., Spanish –es) and the country code (e.g., Spain –ES). If the language is read form right to left (e.g., Hebrew) then select the Right to Left checkbox.
  1. Select the pattern for date formatting of this language. The display date is based on the Java Simple Date Format for displaying dates. If you are using a custom format and not one of the provided options then you must verify that it works by using it in the RCM date renderer and in the date picker. Textual date formats are only supported for English. For other languages, use only numeric date formats.
    Examples:
    MM/dd/yyyy (e.g., 03/26/2006)
    yyyy-MM-dd (e.g., 2006-03-26)
  1. Select the pattern for time formatting of this language by selecting a valid Java Simple Date Format syntax for displaying time. Examples:
    h:mm a (e.g., 4:08 PM)
    HH:mm (e.g., 16:08)

NOTE:

If you do not specify a date or time format for the language then the formats are taken from the actimize.common.timeDisplayFormat and actimize.common.dateDisplayFormat parameters. These parameters can be edited in the Config Parameters tab of admin.jsp.
