Define a new Report Definition – Adding General Information

NOTE:

This feature is available as part of the ERCM license.

To start a new report definition:

  1. Select the category under which you want this report to appear.
  2. Click to open the three tabs, on the right side of the window. These enable you to enter the required information for this new report definition.

  1. Enter the name and description that you want for this report definition. You can use the Edit button, beside the description field box, to open a window that provides more room for entering the required text for the description.
  2. The external Identifier for this report is automatically generated according to the name of the report. You can (optionally) change this identifier if required.
  3. Select the Report Provider for this report. The options are:
  • JReport – The report was created using the JReport Designer
  • External URL – The report was created externally and can be accessed via an external URL web service.
  1. Select the Display Category that is appropriate for this report. Note that the category that you selected from the Design Entities list automatically appears in this field. You can change the selection if you want.
  2. Select the Policy that is appropriate for this report. For a detailed description of the policy options see Set up New Report Policies.
  3. Select the format in which you want the materialized instance of this report to be saved. Report instance materialization is when the report becomes a static file managed by RCM. A report instance must first be materialized before archiving is possible. The options are:
  • None – This should be selected only if you never want a materialized instance to be created for this report. Until a report is materialized, it can be displayed in the JReport DHTML format, allowing ‘slice and dice’ operations.
    Text format
    Excel format
    HTML format
    PDF format
  1. Select the format in which you want the archived report to be saved. When a report is archived, the materialized report instance, along with supporting data such as notes, is copied to an external file system. Once archived, the files can be managed by a third party document management system or managed as part of other file system backup processes. The options are:
  • None – An archived file is not saved for this report. If a materialized instance of the report is not created, you must select None for Archive format since a report can only be archived after materialization.
    Text format
    Excel format
    HTML format
    PDF format
  1. Check on the Use alternative archiving materialization layout if you want the layout of the archived file to appear according to a layout that was specifically defined for archiving purposes. When checked on, you are prompted to enter two different report files and you must upload both reports – the first one is used for the materialized report that displays within the RCM and the second one is used to define the layout for the display of the archived file.
  2. Check on the Enable on-demand report generation for this report if you want the report to be available within the On-Demand Reports module, as a report that can be generated On- Demand.
