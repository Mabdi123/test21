Define Maximum Widgets per Session

I In order to prevent memory leaks from happening as the users open more and more widgets during their session, set the maximum number of widgets per session in the actimize.widget.maximumWidgetsInSession configuration parameter.

When a user reaches the limit (for example, by opening multiple browsers with multiple widgets), RCM will start to remove old widget objects from the session, and a server error will be displayed to the user when the older widgets are accessed.

NOTE:

The limit set by this configuration parameter applies to all widgets, regardless of type (that is, both DART and work items widgets).
