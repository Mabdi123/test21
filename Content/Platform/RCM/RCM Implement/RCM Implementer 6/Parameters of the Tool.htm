Parameters of the Tool

The workflow_asynch_execution_tool.bat (workflow_asynch_execution_tool.sh for Unix) accepts the following parameters:

  • acm_nodes (required) – URL that specifies location of a single ACM application (http://acmhost:8080/acm) or location of several ACM nodes (in a cluster or load balanced environment) separated by a comma (http://acmnode1:8080/acm;http://acmnode2:8080/acm)
  • user (required) – User name
  • password (required) – Password

If you are running the tool in SSL (https) you are also required to pass these additional parameters:

For server side only authentication:

  • ks – path to keystore file containing server certificate
  • kspassword – password for keystore
  • For client side authentication:
  • ks – path to keystore file containing client certificate
  • kspassword – password for client keystore
  • ts – path to client truststore file
  • tspassword – password for client truststore

Example for running the tool:

>workflow_asynch_execution_tool.bat (workflow_asynch_execution_tool.sh for Unix) -acm_nodes=http://actimizetest:8081/actimize -user=dan -password=mypass
