Add Steps to the Workflow

The Workflow tab enables you to create the possible flow of steps that the user may select for specific alert types.

To link a workflow to an alert/item type, see Assign the Workflow of Steps or the Available Steps.

To define the step type:

  1. Click the Toolbox icon of the RCM Designer toolbar or press Ctrl +B to view the list of available steps in the Toolbox window.

  1. Select the step that is the Default Start Node for this workflow and drag it into the workflow window. A Default Start Node step indicates the initial step of all alerts whose alert type(s) are linked with this workflow. This node can be identified by its green background color and thick borderline.

    Each workflow must have one step that is the Default Start Node. There cannot be more than one Default Start Node for a workflow.

  1. Select an additional step and drag it into the workflow window.

  1. Click the Properties icon of the main toolbar or press Ctrl + P to view the properties of the step.

  1. You can change the selected step by selecting another available step from the Step drop-down list:

  1. Select one of the following definitions for this node:

    • Regular – The step node can be inserted anywhere in the workflow excluding the initial step for the alert (item) types that are linked to this workflow. It can be identified by its blue background color.
    • Start Node – This step is only available as an initial step for the alert (item) types that are linked to this workflow. The RCM user has the option to select this step as the starting step. It can be identified by its green background color.
    • Default Start Node – See Step 2. Note that the Default Start Node has additional significance for alerts: An alert can be distributed with a step that is linked to any of the valid start nodes. However, manual alerts or alerts without a specified step use the default start node.
