Managing JNDI Connections

When defining a JNDI connection, the following parameters must be populated:

Field

Description

Connection Name

The internal CM name that you want to assign to this connection.

JNDI name

The name that was configured for the current connection in the server.

Database Type

The type of database for the new connection. The options are:

  • DB2
  • Hive-Cloudera
  • Hive-Hortonworks
  • Impala (using the hive2 driver)
  • Impala (using the impala driver)
  • MSSQL (Unicode)
  • MSSQL
  • Oracle (service)
  • Oracle (sid)
  • Oracle 10
  • Spark
  • Sybase

NOTE:

The Database Type must always be configured.

In order to improve performance it is recommended to define a name space in the application server and RCM. In RCM, this parameter is defined in the actimize.jndi.namespace parameter that is in the Config Parameters tab of Admin.jsp. You should then define the initial context in actimize.jndi.environment parameter that is in the Config Parameters tab of Admin.jsp. An example for a value:

actimize.jndi.environment = java.naming.provider.url=iiop://<host>:<server_port> [^]
