Set up Sign-off Policy

The sign-off policy for a report depends on the nature of the report and the requirements of your organization. It is possible to create one report definition that requires sign-off (e.g. Trade Blotter reports that enable supervisors to monitor daily trades) and a different report definition for those reports that are for information only and do not require sign-off.

The options that are available within the Sign-off Policy page are:

Option

Description

Sign-off is required

When this option is selected, an icon appears within the Reports list of reports to indicate that this report must be signed off.

Sign-off is optional

Sign off is supported for this report but is not required.

No need to Sign-off

Sign off is not supported for this report. When this option is selected, no icon appears within the reports list, and if the user attempts to sign-off a message appears informing the user that sign-off is not supported.
