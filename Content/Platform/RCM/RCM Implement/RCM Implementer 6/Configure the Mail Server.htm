Configure the Mail Server Parameters

Go to the Configuration Sets tab of admin.jsp and select the Email Collaboration action link.

The Email Collaboration page includes the following fields:

 
  • RFI Enabled: Enables the RFI feature. This parameter should be set to True.
  • Admin E-mail Address: This email address is used for receiving RFI related errors and RFI answers that were not matched with an item or case.

The admin email account must be configured as a different account from the one used for RFI email.

Incoming Mail Parameters

  • Protocol: Defines the incoming email protocol. This can be either POP3 or IMAP. If you are using IMAP then it is recommended to define the configuration parameter “rfi.email.imap.attachments.fetchSize” which sets the size of the buffer in KBs. The parameter is defined in the Config Parameters tab of admin.jsp. The default value is 64 KB and minimum value is 16KB. It is advised to set the parameter to a higher value. A larger buffer size increases the speed of downloading attachments. Modifying the value of the parameter does not require a server restart.
  • Host : The IP address of the incoming mail server.
  • Port: The port of the incoming mail server.
  • Encryption: The encryption method used by the incoming mail server. Select none, SSL, or TSL. See also Configure the Mail Server Parameters.
  • Username: The username for logging to the incoming mail server.
  • Password: The password for logging to the incoming mail server.
  • Check Mail Interval: The frequency value for incoming mail (in minutes). The default is 10.
  • IMAP Fetch Size (kb): The size for receiving attachments.
  • Folder Name (requires restart): Mail folder name.

Outgoing mail (SMTP)

  • Title: The description of the outgoing mailbox account. This is usually the full name of the email account owner. For Example, John Smith when the actual email is JohnSmith@exampe1234.com
  • E-mail address: The e-mail address of the outgoing mailbox account.
  • Host (SMTP): The IP address of the outgoing (SMTP) mail server.
  • Port: The port of the outgoing (SMTP) mail server.
  • Encryption: The encryption method used by the incoming mail server. Select none, SSL, or TSL. See also Configure the Mail Server Parameters, page 1.
  • Username: The username for logging to the outgoing (SMTP) mail server.
  • Password: The username for logging in to the outgoing (SMTP) mail server.
  • Attachments Size (MB): The attachment size limits for RFI.
  • Copy to sender: Provides you with the option to add the sender of the original message as CC or BCC.
  • Enable Authentication: Select True to enable SMTP authentication.

LDAP (Email lookup)

These parameters are for the optional LDAP feature that enables auto-completing the email address. For example, the user enters ‘Joh’ in the To: field of the RFI and the address auto-completes to JohnSmith@example1234.com

  • Lookup Filter: The LDAP filter to use with search for matching names with email addresses. NOTE: The {0} is a required place holder for the search word. You should also ensure the result has a ‘mail’ attribute with content.
  • Host: The LDAP server host name.
  • Port: The port of the LDAP server.
  • Username: The username to connect to the LDAP server.
  • Password: The password to connect to the LDAP server.
  • Base Domain: The base domain from which to begin searches.

Kerberos Authentication

  • Enable Kerberos: Select True to enable mail authorization using Kerberos and False to not enable.
  • Authorization ID: Authorization ID of client used for Kerberos authentication.
  • Realm: The realm required for Kerberos validation.
  • Realm KDC: The location of the Key Distribution Center (KDC) for the specified realm.
  • Login Configuration File: Path to the jaas.conf configuration file.

For details, see Configure Kerberos Authentication.
