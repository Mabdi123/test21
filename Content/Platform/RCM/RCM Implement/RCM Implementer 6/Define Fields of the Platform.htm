Define Fields of the Platform List

The fields that are added in the Fields tab are displayed in the RCM UI.

These are fields that were originally defined in the AIS.

The icons available within the Fields tab are:

Icon

Description

Enables you to add a field to the list. You can only add fields that have been previously defined in the AIS. Note that all the key fields must be added to the list.

Enables you to add all the AIS fields to the list. Appears only for AIS defined lists.

Deletes a selected field. Appears only for RCM defined lists.

Cut, copy and paste editing icons. Appears only for RCM defined lists.

Moves a field up in the list. Appears only for RCM defined lists.

Moves a field down in the list. Appears only for RCM defined lists.

Sets a key field. Appears only for RCM defined lists. There must be a key field in a Platform list.

Displays all the key fields of the Platform List.

Refreshes the list of fields

 

The following options are available in the Fields tab:

Option

Description

Field Reference
or
Field Identifier

Displays the name of the field as it was defined in the AIS or RCM Designer.

Display Name

Enables you to modify the display name of the field. The display name of the field appears in the RCM UI.

Type

Displays the data type of the field.

Field

Displays where the field is stored in the ACM_INTERNAL_LIST_ITEMS table of the RCM repository. This field can be modified by clicking on the arrow next to it and selecting another field.

You may change the field mapping to a specific field reference only in the following conditions

You must change to the same field type (there are five fields for each field type)

Field type cannot be used by other field references

Data does not already exist in the RCM.

Has Range

Select from the following:

True – User will view two fields that allow entering data for a specific range

False – One field appears for the end user.

Value Type and Value Fields

Enables you to attach an existing drill down query, UDDT or BU picker.
