Actions that Do Not Return the Status to Unread

The following actions do not return the work item to an unread status:

  • Viewing the details page of the work item.
  • Printing the work item.
  • Exporting the work item.
  • Sending the work item via email.
  • Viewing an attachment or note of a work item.
  • Adding confidential notes or attachments to the work item.
  • Changing field values of related child work items even when the audit of the parent work item is updated as part of the Child Audit Propagation feature.

A work item always returns to an unread status when the actimize.isConfidentialData parameter is enabled and the owner of the item (assigned or unassigned) does an action that is confidential.
