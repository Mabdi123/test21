Define the Execution Settings in the Execution Tab

The Execution tab enables you to configure the AIS Execution plan that is used to run each of the tests for this Tuning Context.

To define the settings in this tab:

  1. Select the AIS connection – Click the Refresh button and select the AIS connection from the dropdown menu.
  2. Execution Plan – Click the Refresh button and select a specific Execution Plan from the dropdown menu that displays all the Execution Plans of the connected AIS.
  3. Arguments Table – The Arguments table displays the mapping of the arguments. These are the arguments to the selected Execution Plan. They are automatically synchronized from AIS when the connector or selected Execution Plan is refreshed. You can define the argument value type by clicking on the arrow of the Value Type field. Each argument can be populated via one of the following methods:

  1. RCM Internal value – The value is set by an RCM session-based parameter such as the user or tenant. One of the arguments must be set to the Test Identifier internal value. In a Multi-Tenant mode another argument must be set to the Tenant Identifier internal value. The value(s) are not visible to the end users.
  1. Regular (default) – The value is provided by the user as a free-text string.
  1. Constant – The value is hard-set on this screen. The value is not visible to the end users.
  1. Value list – The value is selected from a list of values configured in the General section.
  1. Drill-Down Query (DDQ) – The user is presented with a picker for selecting a value based on the results of a runtime query.
