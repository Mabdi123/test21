Overview of Tuning Contexts

Tuning Contexts enable managing and testing new tuning configurations. These contexts enable defining different thresholds and configurations in order to test them. The following terms are used for this module:

Term

Definition

Tuning Module

The module in the RCM web interface main menu and in the RCM Designer

Tuning Context

An instance of the module. For example, “Account Security Blanket Sandbox” or “Monthly Thresholds Sandbox”.

Test Value

A value for a Platform List entry which is different than the “production” value of that entry.

Test Item

A Platform List item with one or more test values.

Trial Run

The results of a single run with a set of test values

Defining Tuning Contexts requires the following tasks:

  • Providing a name and description for the Tuning context
  • Mapping navigation tree(s) and selecting the type of VTL definition
  • Selecting the appropriate AIS connector, Execution Plan and relevant arguments
  • Providing VTL Definitions via an embedded file or via the VTL page
