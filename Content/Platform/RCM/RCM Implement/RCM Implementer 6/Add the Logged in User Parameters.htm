Add the Logged-in User Parameters to the DDQ

This enhancement enables passing the parameters of the logged in user to the DDQ. This feature provides added security to the DDQ results by displaying filtered data according to the related Logged-in User. It supports passing the following parameters as part of the DDQ:

  • Logged-in User's Identifier –User mandatory attribute, which always has a value. Its convention is {$UserIdentifier}
  • Logged-in User's associated BU – Optional User's attribute that can contain a null value. Its convention is {$UserBU :<default>}. Note that the default option is used in scenarios where User does not have an associated BU. If default user does not exist then an empty value is passed. For example, if the SQL is defined as "select F1, F2 from T1 where F3={$UserBU}" and the user's BU does not exist, then the query is interpreted as "select F1,F2 from T1 where F3='' ".

NOTE:

The identifier name has to be typed in exactly as it is in the text area for the SQL Query. It should not be added as a new (client side) parameter (i.e., do not click the icon in order to add it to the parameter list).

These two parameters can be used in the ‘where’ clause and in the ‘select’ statement.

Examples:

  • select login_name from acm_users where user_identifier = '{$UserIdentifier}'
  • b3312 is the default BU select name from acm_md_business_units where bu_identifier = '{$UserBU:b3312}'
