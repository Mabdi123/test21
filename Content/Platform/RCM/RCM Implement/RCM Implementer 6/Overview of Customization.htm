Overview of Customization Procedures

RCM Customization can be performed using the following applications and options:

  • Using tools and options available from the RCM Designer:
  • Setting up Alert Types and Alert Views.
  • Setting up Case Types and Case Views.
  • Setting up Report Types (based on existing Report templates, designed using an external application such as JReport Designer).
  • Setting up Drill Down Queries.
  • Setting up Business Units.

It is possible to work with the RCM Designer without a connection to the RCM database. In this case the user only has access to the Business Units and Reports modules. When there is no connection, users can set up Business Unit hierarchies and can define Report definitions.

When connection to the RCM database fails, the full database message regarding the connection problem is available to users within the details of the error message box.

A log is also provided which includes all connection messages. Authorized users can control the level of messages in the log by changing a value within the log configuration file. For details about changing the level of messages, see Appendix D: Changing the Message Level for RCM Designer.

  • Manually setting up Report Displays by customizing the RCM database parameters.
  • Using the options available within the Admin site for implementers:
  • Verifying parameters within the RCM_md_config_params table and editing values, if required.
  • Setting up Thresholds, Lists and References
  • Checking the Connection Manager Configuration
  • Customizing the GUI (display of tabs and options available within each tab)
