Add Meta Data Resource Strings

The Meta Data resource strings belong to the RCMMD module that includes resource strings in existing objects and objects that may be created by the implementer such as alert type names and custom fields names. To add RCMMD module resource strings click the button. A window appears:

By default, the resource strings are grouped by the following columns: Entity, Type and Item name. You have the option to group the resource strings by additional columns. This is achieved by dragging a column heading into the grouping panel. Select the specific resource strings that you want to import by checking the checkbox in the ‘Add Resource String’ column. For example, in this screenshot a specific case type is selected:

Additional options in this wizard window include

- Show/Hide Grouping panel

- Check all values – All resource strings are checked

- Check selected rows – The highlighted resource strings are checked

- Uncheck all values – All resource strings’ checkboxes are cleared

After you have selected all the resource strings that you wanted to add, click the ‘Next’ button. A summary screen displays all the selected resource strings. You have the option to group the resource strings by the “value’ column. After you have reviewed the resource strings that have been selected, click the ‘Add’ button.

The RCMMD resource strings that have been selected are added to the main list of resource strings. An icon indicates next to a row indicates that the resource string has been manually added to the list.

Click the ‘Save‘ button in order to confirm this addition of resource strings. All other language lists are then synchronized with the update base language resource strings list.
