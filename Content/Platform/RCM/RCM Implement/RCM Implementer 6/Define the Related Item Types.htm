Define the Related Item Types for the Alert Type

Go to the Related Item Types tab of the relevant Alert Type. Note that the associations are only defined in parent work item (alert) types. This tab is always enabled since all child items can also be parent items.

To add related item types for an item:

  1. Click on the Add New Relation Type Association icon.
  1. Select the Relation Type and the Child Object for this association. There can be more than one relation attribute for an Alert Type.

  1. In the Audit on Relation column, select which relation you want displayed in the RCM work item audit history. This feature allows you to hide the addition or removal of a relation that contains sensitive data in the audit history from those who do not have permission to view.

    The options are:

    • Both: Both the parent and child information are audited.
    • Parent only: Only the parent is audited.
    • Child only: Only the child is audited.
    • None: Neither the parent or child is audited.
  1. Select whether to enable child audit propagation for the Relation Type. This feature provides propagated audits (History of Actions) for the parent work item about the child item. Specifically it pertains to the top most parent who does have audit propagation enabled for it. For example, if item A has parent B which has parent C, then audits for A are only viewed in C (the original parent) in addition to A.

The options are:

  • True: Child work item audits also appear in parent history.
  • False: Child work item audits do not appear in parent history (only appear in child history).

The parameter actimize.alerts.audits.consolidated.actions in the Config parameters tab of admin.jsp defines the actions that require audit propagation. The value includes a text description of the actions that are to be logged in the history of actions. The text descriptions should be separated by a comma. The following actions are supported:

  • Alert manually updated
  • Alert updated
  • Note added
  • Owner assigned
  • Score changed
  • Status changed
  • Change Status Back
  • Status changed automatically
  • Relation was created
  • RFI created
  • Received response to RFI
  • Attachment added
  • Attachments removed
  • Owner assigned
  • BU and User assigned
  • Business Unit assigned
