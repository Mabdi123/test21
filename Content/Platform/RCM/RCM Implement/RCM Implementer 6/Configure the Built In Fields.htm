Configure the Built In Fields

Go to the subtab of the Alerts Type Fields tab. All fields in this list are in an update mode that provides the user with the option to update the value of the field. The aggregation function defines the TWI's aggregation in the CWI. The following aggregations are available:

  • First/Last – Populates this field with the value of either the first or last item to be consolidated. Note that this does not necessarily mean the earliest or latest work item, merely the first or last to be consolidated. Note that null values are ignored and the value of the first consolidated TWI in the CWI remains, even if the first consolidated TWI is no longer part of the CWI.
  • Sum/Min/Max/Count – Mathematical functions.
  • Concatenate – When values are mapped to this field, they are concatenated. Applicable only to String fields.

The following built-in fields appear:

  • Business Unit Identifier – A mandatory field for consolidation. Note that the available aggregation functions for this field are First and Last.
  • Score – An optional field for consolidation. All aggregation functions other than Concatenate may be used for this field.
  • Item Type version field – An optional field for consolidation. The aggregation functions Sum, Min and Max may not be used for this field.
  • Item Date – An optional field for consolidation. It is added when an alert type is defined as a consolidation target. The field is also added to any existing consolidation alert type when it is imported via an APF file to RCM version 5.1.1 or later.
