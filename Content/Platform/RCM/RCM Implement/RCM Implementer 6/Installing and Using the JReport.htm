Installing and Using the JReport Designer

NOTE:

    The license for the JReport Designer is purchased separately.
  • The JReport Designer is not provided with your RCM installation.
  • To create and use reports in RCM, you must use JReport Designer V13 Update 3 (Version Number: 13.0.3.0).

 

Creating a new RCM Report using JReport Designer requires the following tasks:

  • Create a connections.mf file under JReport/Bin
  • Copy the Actimize.jar file from your installation CD to the JReport/lib folder
  • Edit the JReport.bat file
  • Create a New Catalog
  • Add Parameters to the Query
  • Define Reports for the Catalog
  • Utilize RCM UDF and UDS in JReport (optional)
  • Configure JReport for a Non-English language (optional)

NOTES:

When defining reports within the JReport Designer, it is possible to use the acm_db_views that are deployed with your RCM Installation. The views are designed to enable you to query the RCM database. When using the RCM views, you can continue to use any reports that are based on the views, even if changes are made to the RCM database schema. For detailed information about these views, see Appendix B of this guide.

Create a file called connections.mf under JReport/Bin

To create the connections.mf file

  1. Using a text editor, create a new file and save it as connections.mf.
  1. Type in version information and the information for each connection that is required for your system, as shown in the example below:

The connections.mf file should include the following information, per connection:

  • Name:<connection_name>
  • ConnectionURL:<full URL for the connection driver>
  • Username:<user name for accessing the connection>
  • Password:<password for above user>
  • Databases that require a catalog (e.g. Sybase and SQL) require the following additional information:
  • DefaultCatalog:<database_name>
