Summarized Description of the Consolidation Process

When a TWI is distributed or created, RCM verifies whether or not it needs to be consolidated into a CWI:

  1. RCM verifies whether or not this TWI has a consolidated relation defined in a CWI. See Define the Relation Attributes for the Alert Type.
  2. If a consolidated relation exists then the consolidation process continues
  3. If a consolidated relation does not exist then a TWI is generated in RCM without any consolidation action. Note that if the TWI is defined as Transient then it is not generated.
  4. RCM reads the condition expression defined in the consolidation key target field.

    See also Configure a Target Field.

    • If this condition is true or no condition exists, the consolidation process continues.
    • If the condition is false, the TWI is generated in RCM without any consolidation action.
  5. RCM finds the potential CWIs into which this TWI will be consolidated (there can be only one CWI per relation). Therefore, RCM searches per relation for the list of potential CWIs based on the consolidation key field and the consolidation criteria. A consolidation relation is the relation between TWI type A and CWI type B, where CWI type B is set as a consolidation target. Consolidation criteria are the conditions defined in the Consolidation Criteria tab. See Define the Relation Attributes for the Alert Type and Configuring Consolidation Criteria. RCM does the following for each consolidation relation when it is provided with a TWI that includes a consolidation key:
    • RCM finds a CWI that matches both the consolidation key and the consolidation criteria.
    • If a CWI is not found then RCM creates a new CWI.
    • If one CWI is found, RCM uses it.
    • If more than one CWI is found, RCM uses the first CWI according to the sorting order defined in the Consolidation Criteria tab.
  6. RCM creates a relation between the TWI and CWI.
  7. CWI field aggregation is performed according to the definitions in the CWI Alert Type fields tab and the Related Item Types tab. See Configure the Built In Fields and Configure the Custom Fields.
    • If the condition is true or no condition exists then the aggregated CWI field value is updated, based on the defined aggregation.
    • If the condition is false then the aggregated CWI field is not affected by the current consolidated TWI.
