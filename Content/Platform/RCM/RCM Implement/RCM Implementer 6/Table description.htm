Table description

All fields within the RCM Report module are defined in the RCM database table RCM_MD_UI_COLUMN_DEFINITION.

The following fields, within this table, should not be changed (that is, they must be left hard coded):

  • GRID_DEFINITION_ID
  • DESCRIMINATOR
  • COLUMN_IDENTIFIER
  • The GRID_DEFINITION_ID and DESCRIMINATOR fields are used by Hibernate, and COLUMN_IDENTIFIER is an identifier used in the code.

The user can change the following fields for each column within the RCM Report Module display:

  • VISIBILITY
  • DISPLAY_NAME
  • DISPLAY_ORDER
  • SORTING_FIELD
  • IS_ASC_SORT

It is possible to change any of the above fields. However, if a column is set to invisible (i.e. Visibility is set to 0), you do not have to change the rest of the configurable fields (since this column will not appear within the display).

Multiple sorting is not currently supported. Only the field with the minimal value is used for system's default sorting configuration. Therefore the sort direction is valuable only for the field with the minimal value.
