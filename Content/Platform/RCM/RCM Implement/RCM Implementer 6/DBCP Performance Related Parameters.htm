DBCP Performance Related Parameters

Parameter Name

Default value

Description

actimize.repository.maxActive

50

Maximum number of active connections in a DBCP connection pool. The number should be configured to be twice of the concurrent requests to the DB. 0 indicates an unlimited amount.

actimize.repository.maxIdle

20

Maximum number of idle connections in a DBCP connection pool

actimize.repository.minIdle

5

Minimum number of active connections in a DBCP connection pool

actimize.repository.maxWait

5000

Maximum wait interval (in milliseconds) for DB timeout in a DBCP connection pool

actimize.repository.maxOpenPreparedStatements

20

This parameter determines the maximum number of open statements that can be allocated from the statement pool simultaneously:

  • 0 – No pooling
  • -1 – Pooling with an unlimited number of open statements

n > 0 – The maximum number of open statements
