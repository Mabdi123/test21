Define Maximum Widgets per Session

In order to prevent memory leaks from happening as the users open more and more widgets during their session, you should set a maximum number of widgets per session in the configuration parameter actimize.widget.maximumWidgetsInSession.
When a user reaches the limit (for example, by opening multiple browsers with multiple widgets) RCM will start to remove old widget objects from the session, and a server error will be displayed to the user when the older widgets are accessed.

NOTE:

The limit set by this configuration parameter applies to all widgets regardless of type (i.e., both DART and Work Items widgets).
