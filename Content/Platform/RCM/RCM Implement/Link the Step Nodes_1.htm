Link the Step Nodes

There are a number of Transition features that are currently only available for alerts. For details see Set Up Workflows.

  • Link the step nodes with a transition arrow by dragging an arrow from the first step node to the second step node: drag the mouse from the first node when the cursor icon changes to the shape of a pointing hand () and then drag it to the second node.

    Define the transition between the two steps by clicking on their mutual transition arrow and by defining the transition arrow’s properties. Note that the transition arrow’s color changes to green when it is selected.

The transition properties include the following parameters:

  • Allow automatic transition when touched – Select checkbox to enable this parameter. A case is considered as “touched” when one of the following actions occur:

    • The case priority is modified
    • The case is assigned to a user and/or BU.
    • A note is added to the case.
    • The case is edited.

Each Step Node can have only one transition node that enables automatic transition when touched.

  • Allow manual transition – This checkbox is relevant only when an automatic transition between the initial step and the selected escalated Step has been defined via deadline settings of the step. See also Add Deadline Parameters to a Step. When this checkbox is selected the user can manually move to the next step that has been defined as an automatic transition before the deadline occurs.
  • Restrict Back – By default all steps enable recalling a step.
    • Recalling a step enables the user to go back to the previous step as part of the workflow transition. To disable this type of transition for the specific step select the Restrict Back checkbox.

  • Require note on transition – Select one of the following options from the drop-down menu:

    • No – The user does not need to add a note in order to move to the next step.
      • On Forward – The user must add a note in order to move to the next step.
