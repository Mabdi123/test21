Use the Filtering Table

The filtering table enables you to filter the reasons.

The filtering table includes the following columns:

  • Field – The reasons list view field.
  • Filter Type – The type of the filter: 
  • User Defined – May be selected for all view fields.
  • Alert Type Identifier – May be selected only for view fields of the String data type.
  • Alert BU Identifier – May be selected only for view fields of the String data type.
  • User BU Identifier – May be selected only for view fields of the String data type
  • Tenant Identifier – May be selected only for view fields of the String data type.
  • User Attribute – May be selected for all view fields.
  • Filter – The filter value for ‘User Defined’ or ‘User Attribute’ filter types. The filter is created by combining all the conditions and using an “AND” operator between them. This filter definition is used in the RCM’s change step window, exposing reasons according to its conditions. Each alert may have one or more reasons associated to it which may be added or removed throughout the alert lifecycle.

The types of filters include the following:

  • User Defined – Enables a user defined filter based on the field data type. Once this type is selected, the filter column is enabled and exposes the user to an editor for filter definition according to the view field data type. If you are adding a filter for a view field of the type String, then the sub-condition for this entry is created by using the OR operator between the string values.

  • Alert Type Identifier – The filter includes a given entry of the list only if its field value equals to the current Alert Type Identifier. The current alert type is the type of the alert to which the RCM user changes the step.
  • Alert BU Identifier – The filter includes a given entry of the list only if its field value equals to the current Alert BU Identifier.
  • User BU Identifier – The filter includes a given entry of the list only if its field value equals to the logged in user’s BU identifier.
  • Tenant Identifier – The filter includes a given entry of the list only if its field value equals to the logged in user’s tenant identifier.
  • User Attribute – The filter includes a given entry of the list only if its field value equals to the selected user attribute. The list of user defined attributes is set according to the logged in user. The exposed user attributes in the Filter column are only those with the same data type as the field data type. The user defined filter may also include the following special values in a string view field filter:
  • ${current_user_bu_identifier} – Equivalent to User BU Identifier
  • ${alert_type_identifier} – Equivalent to User BU Identifier
  • ${alert_bu_identifier} – Equivalent to Alert BU Identifier
  • ${tenant_identifier} – Equivalent to Tenant Identifier
