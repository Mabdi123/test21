Export RCM Objects

From the main menu, select File and then click the Export option.

This action moves you to the first screen of the export wizard. You have the option of selecting the object(s) that you want to export with or without their dependent objects. Click Next. There will be an interval of a few seconds until all the objects are populated in the wizard screen. The following objects can be exported:

  • Admin Objects – These include configuration settings, connection strings, database drivers, GUI items, RCM settings, Users/Groups, Role definitions and user defined data types.
    • User defined data types are defined through the RCM table: ACM_md_uddt. Currently, there is no GUI for this feature.
    • The export/import of User/Groups includes Users, Groups, relations between Users/Groups and between Groups, and all Users/Groups related data (Roles, BUs and Hierarchies). Currently, you cannot partially export/import Users/Group.
  • Alerts – These include Alert Types, Alert Views, and Alert configuration settings.
  • Cases – These include Case Types, Case Views and Case configuration settings.
  • Queries – All Queries that are currently in the Designer. This also includes the logs of DART Views.
  • Blotters – These include Blotter Types, Blotter Settings, Transactions Types, Transactions Views, Transactions Settings and User Filters.
  • General – These include Business Units, Business Unit Hierarchies, Settings Navigation Trees, Settings, Form Types, and Languages, User Defined Data Types.

  • Click the Show Identifiers icon to view/hide the identifiers of the objects that can be exported.
  • Select the Sort by Identifier checkbox to view the objects sorted according to their identifier.
  • To save a selection of objects click the Save Selection icon (selection is saved as an AHO file). The selection can later be loaded via the Load Selection icon.
