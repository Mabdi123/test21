Actions that Return the Status to Unread

If setUnreadAfterNonOwnerUpdates is enabled, the following actions return a work item to an unread status when performed by a user other than the item's owner:

  • One or more of the built in or custom field values in the work item have changed
  • A child work item is added or removed.
  • The work item has been added/removed to/from a parent work item.
  • A work item is added/removed to/from a case.
  • A note or attachment is added (this does not include confidential notes or attachments)
  • User sends or received RFI messages.
  • RFI messages related to the work item have been created, sent, received or removed.
