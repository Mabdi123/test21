The function call syntax

The XSL function syntax is as follows:

<xsl:value-of

    select="actimize:createWidget(‘widgetId’, ‘htmlContainerId’, ‘dataSourceIdentifier’, $optionalParams)” disable-output-escaping="yes" />

widgetId – A string, defined by the implementer, that represents the ID of the Widget on the client side. The id is used for referencing the client-side Widget object when trying to access its selection or other properties.

htmlContainerId – The ID of the container element that stores the Widget HTML (e.g. id of a DIV, SPAN etc.).

dataSourceIdentifier – The identifier of the DART data source that the Widget is based on (taken from RCM Designer).

$optionalParams (optional) – An XSL variable that stores all the optional Widget parameters, including a baseQuery identifier, additional filtering criteria, etc. If you do not want to use any of the optional parameters, you can either leave the variable content empty or remove it completely from the function call as displayed in the following example:

<xsl:value-of

    select="actimize:createWidget(‘widgetId’, ‘htmlContainerId’, ‘dataSourceIdentifier’)” disable-output-escaping="yes" />
