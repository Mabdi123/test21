The Condition Syntax

The condition syntax may include the following elements:

  • Boolean, Numeric and String values like true, false, “abc”, 3, -6.76 etc.
  • Custom field placeholders in the format of ${custom_field_identifier}.
  • Operators such as '!', 'and', 'or', '+', '*', etc.

Evaluation: The condition evaluation process consists of two stages:

  1. Replacement of Custom field placeholders with their respective values (taken from the grid line)
  2. Evaluation of the resulting expression using Spring Expression Language (SpEL) engine.

Boolean operations:

  • ${cb1}
  • !${cb1}
  • ${cb1} or !${cb2}

Mathematical equations:

  • ${ci1} > 2
  • ${ci1}*${ci2} / ${ci3} == 6 + ${ci4}

Textual operations:

  • ${cs1} == 'a'
  • ${cs1} < 'block'
  • ${cs1} + 'b' == 'ab'

Regular Expressions:

  • ${cs1} matches '\d{3}-\d{7}'

The following examples demonstrate the various types of expressions that can be used. In these examples, we assume that:

  • cb1, cb2, … are the identifiers of Boolean custom fields
  • ci1, ci2, … are the identifiers of Integer custom fields
  • cs1, cs2, … are the identifiers of String custom fields
