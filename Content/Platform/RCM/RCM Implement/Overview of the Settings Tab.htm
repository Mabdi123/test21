Overview of the Settings Tab

The Risk Case Manager application enables users to adjust values for the following three types of settings in order to filter or affect the alerts and related information that they receive:

Setting

Description

Thresholds

Thresholds enable you to define a query that presents a lookup table, within the Thresholds tab of the Risk Case Manager application. This lookup table is referenced by an AIS Model in order to customize the Model behavior.

This option is usually used to allow the end user to enter values for thresholds, weights or other types of settings (e.g. required dates) that are used by the AIS Models when generating alerts. For example, a Model may request a threshold that sets the minimum number of transactions that will be considered when generating an alert. Authorized Risk Case Manager users can adjust the threshold values, using the Risk Case Manager application, in order to define the scope of the alerts. The AIS server retrieves these threshold values and uses them during analysis.

Lists

Lists enable you to define a query that presents a lookup table within the Lists tab of the Risk Case Manager Application.

Risk Case Manager users can add or remove items from Exclude/Include lists that are used by an AIS Model when generating alerts. These lists are referenced by the AIS server and factored into the analysis.

References

The Risk Case Manager application enables special queries to data sources, which are not necessarily part of the AIS alert process. This provides additional information that is related to alerts and information for general purposes. The related data is displayed within a Reference tab within the Risk Case Manager application.

For example, a query may be defined to retrieve daily trade information for reference purposes while reviewing alerts.
