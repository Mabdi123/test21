The Scheduler

The RCM scheduler is also in charge of running jobs for indexing the Solr server.

You can view the Scheduler status for Solr indexing by doing the following:

  1. Go to the Connections tab of admin.jsp and click on the Diagnostic action link
  1. A window opens. Scroll down in order to view the Scheduler’s current settings. 

Related Solr indexing parameters include the following:

actimize.scheduler.enable

Enables/Disable the RCM scheduler. When this parameter is set to false, the scheduler does not run at all. This affects both Solr indexing and RFI. Modifying this parameter requires restarting the RCM.

actimize.scheduler.threadPoolSize

Defines the size of the thread pool used by the scheduler. Required to be between 5 and 40. Solr uses half of the pool size.

actimize.globalSearch.indexing.jobInterval

Defines how often, in minutes, the global search indexing job should run. Required to be between 1 minute and 1440 minutes (1day).

actimize.globalSearch.indexing.jobSize

Defines how many items can be indexed in one execution of the indexing job. This parameter affects only the time needed for each execution, not the memory used by the indexing job. Required to be between 1,000 and 10,000.
