Methods for TWI Fields with Null Values

These methods are only relevant in scenarios where the TWI fields can be null. This implies that the TWI fields are not mandatory and are set to populate the CWI fields. The methods were added to avoid occurrences of empty fields or ‘null’ appearing as a string, which may affect future data or related processes. Use these methods when you define an expression, such as in the Source Field/Expression column or Condition Expression column. There are three methods/expressions you can use:

  • nvl – Example: nvl (${a}, '') + 'b' (nvl(${score}, '') + ${bunit_identifier})

    If the score field is null then replace it with '' (meaning that the value will be left out without any spaces and only the bunit_identifier will be shown).

  • Elvis operator – Example: ${a}?:'' + 'b' (${score}?:'' +${bunit_identifier} )

    If the score field is null then replace it with '' (meaning that the value will be left out without any spaces and only the bunit_identifier will be shown).

    Note that if you want to place the 'b' value before the 'a' field then you must use parentheses (for example, ('b' +${a}?'')).

  • Concat – Example: concat (${a},'b') (concat (${score},${bunit_identifier} )

    If the score field is null, leave only the bunit_identifier.
