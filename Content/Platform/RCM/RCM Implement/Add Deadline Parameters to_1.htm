Add Deadline Parameters to a Step

Adding a deadline to a Step is optional. A Step can be without a deadline

To add a deadline to a step:

  • Select the checkbox in order to add a deadline to a step.

    Enter the number of days/hours/minutes that should pass before the deadline occurs or use the arrow icons to display the desired number of days:

    The following checkboxes may also be used in order to add parameters to the deadline:

    • Send email Notification – Enables you to indicate the number of days/hours/minutes before the deadline in which an email is sent to the user.
    • Create Reminder – Enables you to indicate the number of days/hours/minutes before the deadline in which a reminder about the deadline is displayed to the user.
    • Change Step to – Enables you to select the step of the case after the deadline is reached. The required escalation step needs to be added to the workflow first. The selection of the escalated step creates an automatic transition between the currently in focus step and the selected escalated step. Note that the Allow manual transition checkbox for the transition arrow is enabled when this parameter is defined.

When creating a deadline, verify the following:

  • The case should belong to an case type that has been associated with a workflow definition.
    Even though a deadline may be defined for a step, it will only apply to a case in that step when the case type is associated with the workflow.
  • All the "mail.smtp.*" RCM configuration parameters must be defined.
  • The case must have an owner.
  • The case owner must have a defined email address.
