v_acm_report_instances2

Description: This view returns the following report information for each report instance:

Field Name

Description

report_instance_id

The internal RCM ID for the report instance

report_name

The Report name

report_type_join_id

The Join ID to the Report’s Report Type (v_acm_report_types)

business_date

The business date reference to the report

create_date

The date when the report was generated

Step

The current step of report (e.g., deleted, archived, etc.)

signof_flag

An indication of whether the report was signed off

signoff_date

The date when a report was signed off

archive_date

The date when a report was archived

generation_system

The source that generated the report (this can be the AIS, the RCM Designer or On-demand by the user)

owner_join_id

The Join ID for the user who is responsible for this report (v_acm_users2)

bu_join_id

The Join ID for the Business unit with which this report is currently associated (v_acm_business_units2)

original_bu_join_id

The Join ID for the Business unit originally assigned to the report (v_acm_business_units2)

signoff_user_join_id

The Join ID to the user who signed off the report. (v_acm_users2)
