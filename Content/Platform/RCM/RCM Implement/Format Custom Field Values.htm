Format Custom Field Values for Translation

The Translated String option of the Column Display Formatter for a custom field appears in all views (Alert Views, Case Views, etc.).

It is for translating the actual data in the field and displaying the translated values in the RCM. Specify the module for the data translation in the Format String field. If this field is left empty then the module is ‘Actimize’.

The formatter is used for translation of simple base values (e.g. ‘yes’,’no’). If the base value requires parameters (e.g. "hello {0}") then the data in the column needs to contain the parameters that use the same syntax in the XSL APIs ("hello {0}@@John"). See also XSL API.

RCM always uses the exact match option on the columns with this formatter when searching in the filter by and advanced filter.
