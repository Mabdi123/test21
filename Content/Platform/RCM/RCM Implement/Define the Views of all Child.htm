Define the Views of all Child Alert Types

An Alert Type that is defined as a child must have a defined view, which defines the detailed table of its item picker. Sorting definitions for these Items is not mandatory.

When the picker is open there is a hard coded filter that includes all undeleted items of this Alert Type.

Currently, you cannot format the fields as HTML code or as links.

  • From the View tab of the Alert Type define the formatting of the fields in the Fields subtab. For more information on formatting, see Configure the Fields of the View.
  • You also have the option to define the sorting for the Alert Type in the Sorting subtab. For more information on sorting definitions, see Add Conditional Formatting.
