Create Condition Display Modes

Condition display modes are defined in the query XML. A standard query XML contains nodes such as comparison_condition, boolean__condition, etc. These nodes store the details of the field, operator and values used by the condition.
The display modes are defined in a new XML node named “displayMode” as displayed in the following example:

<comparison__condition>

<operator>LT</operator>

<disabled>false</disabled>

<displayMode>Locked</displayMode>

<children>

<field__term>

<fieldId>20002</fieldId>

</field__term>

<value__term>

<value>1</value>

</value__term>

</children>

</comparison__condition>

The available display modes are:

  • Editable – The user can view the condition, change its field, operator or values, and even remove it completely from the query. This is a normal DART condition and it is also the default display mode when no display mode is specified.
  • Mandatory – The user can view the condition and change its value but the field cannot change and the operator must be “is”. Also, the condition cannot be removed. This is used to force the user to set a value for the query (for example, for performance reasons).
  • Locked – The user can view the condition but cannot change any part of it or remove it. This type of display mode is useful for letting users know how the data was filtered.

All locked conditions will appear in one group at the top of the query builder, and the user will be able to hide or show them using an expand/collapse link (the visibility of the locked section will be kept between requests).

  • Hidden – The entire condition is hidden from the user and cannot be changed in any way.
