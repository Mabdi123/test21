Set up Notes Policy

You can set up a report definition that allows RCM users to add notes to the reports, or you can setup a definition for reports that does not support this feature. It is also possible to specify the conditions under which the user can add notes.

The options that are available within the Notes Policy page are:

Option

Description

Always allowed

When this policy is in effect, an Add Notes option is available for RCM users within the Report Action activity bar when the report is selected. In this case there is no limit on when the user can add a note.

NOTE: Notes added after archiving will not be archived.

Allowed X days after sign-off

This option is only relevant if you select, in the previous Sign-off page, either mandatory sign-off or optional sign-off. If you indicate that the reports never require sign-off then the option to allow notes after sign off is not applicable.

When this policy is selected, you must indicate the amount of time after sign-off at which notes will be supported.

Allowed until archived

If this policy is selected the user is not allowed to enter notes after the materialized report is archived.

This option is only relevant if you select, in the Archive Policy page, that archiving is required. If you indicate that the reports should not be archived, then the option to allow notes until archived is not applicable.
