Filter via a Parameter

This feature provides support for the following runtime parameters:

  • System parameters

    • '{$userIdentifier}' – Currently logged in User identifier. If default value was specified in parameter then this definition is not used.
      • '{$userBu}' – Currently logged in User’s BU identifier. If no BU is assigned to the user then the default value is used. If no default value defined then an empty string is returned.

        • '{$tenant}' –

          • In Multi-Tenant environment value is replaced with the user tenant or returns an exception when no tenant is defined.
          • In a non Multi-Tenant environment value is replaced with the “"actimize.common.tenant.defaultValue" parameter.
  • '{$userlang}' – Logged in user language
  • User defined parameter:
    {$<parameter_name>:<parameter_default_value>}
  • Custom parameter:
    '{$<Custom parameter>}'

Follow these rules when adding filtering parameters to the UDDT query:

  • Parameter must be in single quotes when a Char value is provided:
    Example:
    Select id, value from tbl where char_value = ‘{$char_param:any}’
  • Numeric values should not be in single quotes.
    Example:
    Select  id, value from tbl where int_value = {$int_param:5}

Using a Custom Parameter

  • Use a read only table that includes HTML links in one of its columns. This read only table calls another table in the database via the HTML links.
  • The called table is a regular table (not read only).

Query from UDDT table:

select * from uddt_country_city where country_name = '{$custom_param:}'

Relevant cell in column of read only table that includes a link in HTML:

<a href="display.settings.act.go?thresholdId=threshold_list&custom_param=USA&task=runSQL">Thresholds</a>
