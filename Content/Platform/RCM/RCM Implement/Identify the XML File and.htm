Identify the XML File and XSL Location

The alert details for each alert type are stored in an XML format, usually defined by the AIS Model corresponding to that Alert Type. This XML can be retrieved from:

  • Within the database
  • A file
  • A URL

Each Alert Type is associated with one or more XSL files used to render alert details to a display and printing format.

To set up retrieval of the Alert Details:

Indicate one of the following methods for retrieving the alert details:

Option

Description

Use XML from Database (recommended)

Choose this option if the AIS is configured to write the XML to the database.

Note that the XSL path, in this case, is specified in the RCM_md_config_params table as “actimize.alerts.pathTo XSL”. You can access this table from the RCM Admin Site – Configuration tab (See Configuring RCM Parameters– Admin Site Config Parameters Tab and the acm.ini File,).

The XSL path is also specified within the alert_types table of the RCM database.

Use XML that is stored on disk

Select this option if the AIS is configured to write the XML to files on a disk. When XML is stored on a disk, the location of this XML file is specified in the RCM_md_config_params table as “actimize.alerts.pathTo XML”. (See Configuring RCM Parameters– Admin Site Config Parameters Tab and the acm.ini File).

Note that the XSL path, in this case, is specified in the RCM_md_config_params table as actimize.alerts.pathTo XSL and is also specified within the alert_types table of the RCM database.

Use an absolute URL

Select this option if the alerts are generated by an application other than the AIS.

When you indicate that you are using an absolute URL for retrieving the alert details, and the URL is an XML file, then the path of the XSL for this XML is specified within the XML file itself. Formatting is done on the client side.

Use a relative URL

Select this option when alerts are generated by an external application and the URL is relative to the path specified in the RCM_md_config_params table as “actimize.alerts.relativePathToXML”.

If the URL is an XML file, then the location of the XSL is specified in the XML file and formatting is done on the client side.
