Add a Customized Image

In order to be able to use an image, upload the image to the Virtual File System using RCM Designer. Images should be stored in the following path:

<VFS>\app_gui_items\rcm\<theme>\actions\icons_picker

Alternatively, insert the image file in the following path (inside the application .war file):

<Web root>\app_gui_items\rcm\<theme>\actions\icons_picker

The <theme> folder is RCM’s current theme identifier (currently only ‘new_york_blue’). It includes each image in each state with an additional CSS file.

You also have the option to create a sprite image that contains all images. This is the recommended method for improved performance. See also Use Sprite Images, page 1.

NOTE:

If two files with the same name are added to the system from two different sources (the VFS and the Web root folder), then the file from the VFS is the only file loaded into RCM and the one in the Web root is ignored.

The VFS should be enabled in RCM. This can be done by enabling the configuration parameter: ‘actimize.general.virtualFileSystemState’.
