Configuring JReport for a Non-English Language

In order to configure the JReport for a Non-English Language, you must initially retrieve the True Type Font (*.ttf) files for the fonts you want to use in the JReport template. You can retrieve the *.ttf files from your system's font directory (e.g., C:\WINDOWS\Fonts).

After the relevant fonts have been retrieved, do the following in the JReport Designer:

  1. Copy the*.ttf files to the <JReport Home>\Font directory.
  1. Run the JReport Designer in the Design mode.
  1. Click the arrow next to the font types and scroll down. Find the fonts that have the * character before their name (these are the True Type fonts that you have deployed). Select a TTF font for the specific label.

  1. Click the View tab in order to view the result of the modification.
  1. Open the report, go to Report Inspector and find the property EmbededFonts that belongs to the root node. The drop-down list displays all the TTF fonts used by the report. Select one or multiple fonts by pressing the Ctrl or Shift key. In the following screen shot a specific TTF font is selected.

  1. Save the report catalog and data files.

After the JReport Designer tasks have been completed, copy the *.ttf files to the Font directory of the JReport Enterprise Server.

NOTE:

If you using an RCM version prior to version 3.04.22 then add the following line to the server.properties file in the bin directory of the JReport Enterprise Designer:

engine.default_encoding=UTF-8
