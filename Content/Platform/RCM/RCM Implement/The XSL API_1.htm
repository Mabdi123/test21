The supported parameters are:

  • dataSourceIdentifier – The identifier of the DART data source that the widget is based on. This parameter is mandatory.
  • widgetGroup – A string representing the group of the widget(s) on the client side. The group is used for referencing the client-side widget object when trying to access its selection or other properties. This parameter is optional.
  • collapsedTitleDisplay – The text that appears in the link that is displayed inside the cell before the widget is expanded. This parameter is optional. If not provided, then value of the cell is displayed.
  • expandedTitleDisplay – The text that appears in the link that is above the widget that is used to collapse the widget. This parameter is optional. If not provided, then value of the cell is displayed.
  • collapsedToolTip – The text that is displayed as a tooltip for the cell while collapsed. If not provided, then it is “Expand”.
  • expandedToolTip – The text that is displayed as a tooltip for the cell while expanded. If not provided, it is “Collapse”.
  • width – The width of the widget. This parameter is optional. If not provided, then width is 100%.
  • height – The height of the widget. This parameter is optional. If not provided, then height is 5 lines (currently 100 pixels).
  • autoload (true/false) – Indicates whether or not the widget auto loads and expands upon expanding the layout. Default is false.
  • showIcon (true/false) – Indicates whether or not to display the expand/collapse icon. Default is true.
  • hideExpandLink (true/false) – Indicates whether or not to hide the link to expand and collapse. If true then auto load occurs. Default is false.

Refer to the RCM Self Developer Guide for more information on the Records Widget JavaScript API.
