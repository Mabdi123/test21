Overview of Languages Setup

The base language of RCM is English (U.S.). Its data is composed of resource strings that belong to one of the following modules:

  • RCM module – Includes all RCM platform built in resource strings that cannot be removed or updated but can be translated. For example, RCM messages that are displayed to the user during specific scenarios. This module cannot be modified (resource strings cannot be added or removed from it) and includes the strings of fields that cannot be changed via the RCM Designer. For example, the About link of .
  • RCMMD module – Includes metadata resource strings in existing objects and objects that may be created by the implementer such as alert type names and custom fields names. These are basically resource strings of fields that can be changed via the RCM Designer. The following table includes the fields that may be prepared for translation in these objects:

  • Additional modules that include implementer defined resource strings and are used in VTL files and XSL files of alert types. The default name of a new module is ACTIMIZE. The resource strings can later be used in a related VTL or XML file for the following features:

See also APIs for the Languages Feature.

The following tasks may be done in the base language:

  • Add/remove metadata resource strings of the RCMMD module.
  • Create or import custom resource strings of additional modules or from other RCM servers.
  • Export the language list of resource strings to a comma delimited (CSV) file. This is usually done in order to provide a translator with a simple file for entering the translations.
  • Export a specified list of resource strings.

The following tasks may be done after the base language is defined and updated with the required resource strings:

  • Create a new language node.
  • Enter the relevant translation for each base value in the Translation page.
  • RCM can also be done in the Base language). After the table has been filled in by the translator, the CSV table can be imported back to the language node.
  • Set the Non-English language as the current language for RCM.
