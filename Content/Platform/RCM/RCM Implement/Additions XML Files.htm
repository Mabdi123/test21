Additions XML Files

The additions XML files enable implementers to add their configuration, without changing the original solution.xml file. This also enables easy upgrades of the Work Item Display Infra. Up to two “additions” files may be used to hold extra sections and information not already included in the solution XML file and to hold values that override the values in the solution XML file, if necessary. These files are merged with the solution XML file in the following order:

  • <filename>_additions_1.xml overrides values in the solution XML file, and adds any new sections to it.
  • <filename>_additions_2.xml overrides values in the solution XML file (that now already includes values from the additions_1 XML file), and adds any new sections.

For example, to change the case details UI displayed to the user, see the examples below to:

  • Add a new preset (tab)
  • Add a new tab with an existing section, overwriting some attributes of that section
  • Remove a preset (tab) from the UI

To add a new preset (tab) with a new section, see the following sample CaseDetails_additions_1.xml file:

<?xml version="1.0" encoding="UTF-8" standalone="no" ?>

<DisplayConfiguration internalConfigurationFileName="internalXml/CaseDetails_internal.xml" infra-version="1.0.0.0">

 

   <Presets id="presets">

 

      <Preset media="screen" name="My New Tab" order="1" show="true">

         <Sections>

            <Section expanded="true" identifier="myNewSection" order="1" show="true"/>

         </Sections>

      </Preset>

   </Presets>

 

   <SectionsConfigurations id="SectionsConfigurations">

 

      <SummarySection name="my New Section" numOfCols="2" identifier="myNewSection" size="L" alignment="left" />

 

   </SectionsConfigurations>         

 

</DisplayConfiguration>

This new preset will be displayed as the second tab in the UI if the solution.xml file already has a preset with order=1 as it will be merged into the XML after the solution.xml file.

To add a new preset (tab) with a summary section, overwriting the text size and number of columns defined in the solution.xml file, see the following sample CaseDetails_additions_1.xml file. (The singleview.xml file has the following SummarySection definition: <SummarySection name="Summary" numOfCols="2" identifier="summarySection" size="S" alignment="left" />)

<?xml version="1.0" encoding="UTF-8" standalone="no" ?>

 

<DisplayConfiguration internalConfigurationFileName="internalXml/CaseDetails_internal.xml" infra-version="1.0.0.0">

 

   <Presets id="presets">

 

      <Preset media="screen" name="My New Tab" order="1" show="true">

         <Sections>

            <Section expanded="true" identifier="summarySection" order="1" show="true"/>

            </Sections>

      </Preset>

 

   </Presets>

 

   <SectionsConfigurations id="SectionsConfigurations">

 

      <SummarySection identifier="summarySection" size="L" numOfCols="3"/>

 

   </SectionsConfigurations>

 

</DisplayConfiguration>

To remove a preset (tab), see the following sample Presets section in the CaseDetails_additions_1.xml file:

   <Presets id="presets">

 

      <Preset media="screen" name="Content" show="false">

         <Sections>

         </Sections>

      </Preset>

 

   </Presets>

Since the setting in the CaseDetails_additions_1.xml file overwrite the settings in the CaseDetails.xml file, the Content preset (tab) will no longer appear.
