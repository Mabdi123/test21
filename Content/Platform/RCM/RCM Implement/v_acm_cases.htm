v_acm_cases

Description: Returns all cases with the following information:

Field Name

Description
case_join_id

The internal Join ID to this Case
case_identifier

The Identifier of the Case.
name

The Case name
description

A description of the Case, if available.
case_date

The business date reference to Case

create_date

The date when the Case was created
case_type_join_id

The Join ID to this Case’s Case Type (v_acm_case_types)
case_status_join_id

The Join ID to this Case’s step (v_acm_case_statuses)
owner_join_id

The Join ID to the user who is responsible for the Case (v_acm_users2)
bu_join_id

The Join ID to this Case’s Business Unit (v_acm_business_units2)
original_bu_join_id

The Join ID to this Case’s Original Business Unit (v_acm_business_units2)

fl_deleted

An indication whether Case was deleted
fl_archive

An indication whether Case was archived
fl_read

An indication whether Case was read

fl_read_by_owner

An indication whether Case was read by Case owner

last_read_date

The last read date for the Case
last_read_user_join_id

The Join ID to the user who last read this Case (v_acm_users2)
last_update_date

The date when the Case was last updated
last_update_user_join_id

The Join ID to the user who last updated this Case (v_acm_users2)
closed_date

The date when the Case was closed
custom_attr_join_id

The Join ID to custom attributes for this Case (v_acm_case_custom_fields)
priority_join_id

The Join ID for Case priority (v_acm_case_priority)

deadline_date

This field stores the date at which a work item will reach its deadline for processing. On that date or later the work item will perform escalation based on the workflow definition.

Both deadline_date and email_date actions are triggered by running a workflow utility that should run daily.

highlight_date

This fields stores the work item highlight date. On that date or later the work item will show a UI indication on the work item grid.

email_date

This fields stores the work item email date. On that date or later an e-mail will be sent to the owner on that work item.

Both deadline_date and email_date actions are triggered by running a workflow utility that should run daily.

rfi_state

Indicates the status of RFIs within the Case. It has one of 3 possible values:

0 – No RFIs

1 – Has RFIs but all messages have been read

2 – Has RFIs but there are unread messages
