The following mapping definitions are available:

Drill Down Query

The Report Generation Wizard generates a drop-down list that is the result of the Drill Down query. Instead of presenting the user with the task of entering a parameter value into a simple textbox, a list of choices is generated in real-time based on the predefined query.

For example, the drill down query low_medium_high has the following SQL query:

  • select label from icons where label is not null

This query results are the score icons low, medium and high. The score high is selected as the default value.

Therefore, it is displayed with the default value in the Report Generation wizard, and enables the user to select an alternate value via the generated drop-down list.

NOTE:

  • Queries with variables cannot be used with this feature and they will be filtered out from the list of queries in the window.
  • Only parameters of type String and Integer support the drill-down query feature.

RCM will not present the user with a full list of choices in the drop-down list if the query resulted in too many records. Instead, a partial list of values (taken from the beginning of the full list) is displayed. In addition, a special warning icon will be shown next to the list with a tool tip explaining to the user that the list is not full. The number of records displayed is configurable via the actimize.reports.wizard.maxListSize parameter. For details, see Configuring RCM Parameters– Admin Site Config Parameters Tab and the acm.ini File.

  

Business Unit

The attribute of the Business Unit that is selected is sent during the generation of the report. The Default Business Unit field enables you to select the default BU that is displayed in the Report Generation Wizard. Clicking the button moves you to a screen where you can select an alternate default business unit. The Default Hierarchy field enables you to select the default BU hierarchy that is displayed in the Business Unit Explorer.

For example, if a geographical hierarchy by the name of Geo is selected and the business unit CA is selected as the Default Business Unit then the parameter is displayed in the following way in the Report generation wizard. Notice that the parameter default display is CA and the Business Unit Hierarchy opens by default to the Geo hierarchy.

Selecting the checkbox enables the RCM user to select a Business unit with its entire sub Business Units.

Sending two values via one parameter is not an option, since JReport cannot retrieve a complex value. Sending a comma delimited list of all BU’s in a specific sub tree is also not feasible as the list can potentially store thousands of values and there is a limit to the strings that can be sent to JReport.

Therefore, the solution focuses on sending two separate parameters to JReport – one with the BU and one with the hierarchy that stores the hierarchy of the sub BUs. Due to these facts another string parameter must be utilized to store the identifier of the hierarchy for sub BUs. If you would like to enable the user to also select a hierarchy then you must select the checkbox Enable RCM user to select a Business Unit with all its sub Business Units, and you must also select a parameter of the type string that stores the identifier of the sub Business Units.

In the following screenshot the attribute Id of the Business Unit that is sent during the generation of the report, and the checkbox Enable RCM user to select a Business Unit with all its sub Business Units is selected. A second parameter of the type string is used for storing the hierarchy parameter.

Therefore, it is displayed in the Report Generation wizard with two fields for this source. The first one displays the default Business unit (The Business Unit the user belongs to) and provides the user with an option to select an alternate Business Unit via the Business Unit Explorer. The second field displays the Business Unit’s Hierarchy and also enables the user to select an alternate hierarchy. The attribute Id is sent during the generation of the report.

Report Business Unit

This source enables the user choose one Report Instance BU per report, but allows using more than one attribute for multiple report parameters. If more than one parameter is configured as a Report Instance BU business type, then the Designer automatically turns all parameters to hidden excluding the initial parameter.

The options of defining the Report Business unit source are identical to those of the Business Unit. For more information, see the Business unit source, page 1.

NOTE:

When JReport receives parameter values from RCM it converts them all to strings before planting them into the report queries. When the received value is NULL, string parameters end up getting an empty string as a value. However, integers, floats and other non-string parameters end up receiving the string value “NULL”.

This is a known limitation that requires the report designer to handle optional non-string parameters with extra care.

  

Tenant

Enables you to pass of the following Tenant’s attributes as a report parameter: Id, Identifier, and Name.

This parameter is available only if the RCM is in a Multi-Tenant mode. The parameter must be of the type string and hidden.
Select the logged on user Tenant or a specific Tenant from the Default Tenant dropdown menu.
