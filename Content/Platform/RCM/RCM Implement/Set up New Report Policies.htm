Set up New Report Policies

The RCM Designer is delivered with a Default Policy that can be selected for any report. It is also possible to create a new policy according to the requirements for specific types of reports. Each Policy is comprised of a set of individual policies. Assigning a selected policy to a report means that you are assigning all of the individual policies that were set for the selected policy.

Setting up a new set of Report Policies using RCM Designer involves the following tasks:

  • Accessing the Policies window
  • Assigning a name and description to the new set of policies
  • Setting up the Sign-off Policy for the new set
  • Setting up the Notes Policy for the new set
  • Setting up the Materialization Policy for the new set
  • Setting up the Archive Policy for the new set
  • Setting up the Purge Policy for the new set
  • The Default Policy includes the following settings:
  • Sign-off policy – Optional
  • Notes Policy – Always allowed
  • Materialization Policy – When the report is generated
  • Archive Policy – Just before the report is being deleted
  • Purge Policy – 60 days after the report is deleted

NOTE:

Report types that use an External URL provider must have a policy that is set as follows:

  • Materialization Policy must always be set to never.
  • Archiving must always be set to never.
  • Purge policy must always be set to never.
  • Notes and Sign off can be set to any of the existing options.
