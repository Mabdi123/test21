Active and Hover States

In addition to the main image file, you have the option to create a unique look when the user hovers on an image or when the action related to the image is active. Add another file for every state to the following path:

  • To the VFS:

<VFS>\app_gui_items\rcm\<theme>\actions\css

  • Inside the .war file:

<Web root>\app_gui_items\rcm\<theme>\actions\css

You can use any file name that you want, but it is recommended to use the suffixes ‘_hover’, ‘ _active’  in order to distinguish between the different states:

  • ‘{original_file_name}_hover’ (e.g. ‘add_new_alert_hover.png’)
  • ‘{original_file_name}_active’ (e.g. ‘add_new_alert_active.png’)

For each file, a CSS class needs to be added to the CSS file in the same method as it was added for the main image file. The CSS class name should be in the format: .custom-<file name>:hover, and .custom-<file name>:active.

Example:

.custom-add_new_alert:hover{

background: url(‘../add_new_alert_hover.png’) no-repeat;

}

.custom-add_new_alert:active{

background: url(‘../add_new_alert_active.png’) no-repeat;

}
