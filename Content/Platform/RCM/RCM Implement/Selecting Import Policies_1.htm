Select Import Policies

  1. Select a data update policy from the dropdown menu:
  • Overwrite – Data in the package overwrites conflicting data in the RCM.
  • Selective – Only the objects that do not modify site-specific customization are imported.
  1. Select a reference update policy from the dropdown menu:
  • Permissive – Import nullifies any references in the package to objects that do not exist in both the repository and the package.
  • Strict – Import fails if package includes references to objects that do not exist in both the repository and the package.

  1. Click Next, the Select objects screen appears.
