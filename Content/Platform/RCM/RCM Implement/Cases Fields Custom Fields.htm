Cases Fields - Custom Fields

Custom Fields are the additional fields, which may appear in a specific Case View and in the Advanced Filter window. (Each Case View can be composed of Built In Fields and Custom Fields).

You can remove, add or modify Custom Fields. Each Custom Field is assigned to a specific table and record in the RCM database.

The purpose of a Custom Field is to enable the Risk Case Manager to capture and store business data on the case. These fields are an extension to the basic data that can be stored by a case such as workflow fields, description, and score, and are only useful once they are associated with a case type.

Once the data is stored in a Custom Field for a case, RCM can then provide this data as enriched case details. These details are available within the List View displays, the Filters, and the Edit Case action.

You can remove, add or modify Custom Fields. Each Custom Field is assigned to a specific table and record in the RCM database.

The icons available within the Custom Fields window are as follows:

Icon

Description

The options that are available within the Custom Fields page are:

Option

Description

Display Title

Title of field as it will appear in the Advanced Filter window.

Identifier

Enables you to enter a reference ID for this Business Type, which later can be identified by an external application.

Description

Custom Field description.

Table

A view only field that indicates the table where this field is stored. All tables are stored in the database of Risk Case Manager.

Field

Select field where Custom Field data is stored via the drop down list. Each field has a specific number and an icon that indicates its type (e.g., is an integer).

Business Type

An optional setting that enables you to associate custom field(s) with a Business Type. See Business Types.

Has Range

True – User will view two fields that allow entering data for a specific range

False – One field appears for end user.
