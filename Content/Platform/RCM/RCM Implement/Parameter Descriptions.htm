Parameter Descriptions

The following legend explains the edit recommendations for each parameter as a guide for working:

Can be edited – Means that you can edit this parameter value in order to customize the output of your application.

Recommended not to edit – Means that you should only change the parameter value after consultation with authorized Technical Support.

Recommended not to edit after initial setup – Means that this parameter is correlated with other setup information within the system and changing it after initial setup may hinder the application from working.

Do not edit – Means that changing this parameter will seriously alter the ability of the application to work.

Some of the parameters in the list are for features that require a separate license.

actimize.authentication.header.overridePrincipalLocation

There is no default value

Change the value of this parameter only if the parameter actimize.authentication.type has the value header and the default extraction method defined in actimize.authorization.type that is not suitable for your environment. Possible values are:

headerOnlyPrincipalLocation - Obtains the username by looking for the value of the HTTP request header whose name is specified by actimize.authentication.headerName.

LDAPPrincipalLocation - Obtains the base username using the same mechanism as the headerOnlyPrincipalLocation. But also prepends the <domain-alias>\ to the beginning. This alias is defined in the domainsInfo bean within applicationContext.xml.

NTLMPrincipalLocation - Uses the getRemoteUser of the HTTP request to get name of the principal.

For example: If you are using Active Directory authorization (actimize.authorization.type=ActiveDirectory) with SSO (header) authentication rather than NTLM authentication, the value of this parameter should be set to LDAPPrincipalLocation.
