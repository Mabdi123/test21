Alert Fields - Custom Fields

Custom Fields are the additional fields, which may appear in a specific alert view and in the Advanced Filter window. (Each view can be composed of Built In Fields and Custom Fields).

The purpose of a custom field is to enable the RCM to capture and store business data on the alert. These fields are an extension to the basic data that can be stored by an alert such as workflow fields, description, and score, and are only useful once they are associated with an alert type.

Once the data is stored in a custom field for an alert, RCM can then provide this data as enriched alert details. These details are available within the List View displays, the filters, and the Edit Item button.

You can remove, add or modify custom fields. Each custom field is assigned to a specific table and record in the RCM database.

The icons available within the Custom Fields window are as follows:

Icon

Description

The options that are available within the Custom Fields page are:

Option

Description

Display Title

Title of field as it will appear in the Advanced Filter window.

Identifier

Enables you to enter a reference ID for this Business Type, which later can be identified by an external application.

Description

Custom field description.

Table

A view only field that indicates the table where this field is stored. All tables are stored in the database of RCM.

Field

Select field where custom field data is stored via the drop-down list. Each field has a specific number and an icon that indicates its type (e.g. is an integer).

Size

Size of the field.

Business Type

An optional setting that enables you to associate custom field(s) with a Business Type. See Business Types.

Has Range
  • True – User will view two fields that allow entering data for a specific range
  • False – One field appears for end user.

Predefined Value List

Enables you to link a custom field with a value list. In the column heading, select Add New from the drop-down list. The following window opens.

See also General Configuration - Value Lists.

Sensitive Data Mask

See Map a Custom Field to a Mask.

Indexed for global search

Adds the values of this custom field to the Global Search index of RCM. Refer to the RCM Installation Guide for more information on installing and configuring the Solr search server. A custom field that is masked for sensitive data cannot be indexed for global search.

In Selective mode the following custom field attributes are not overwritten:

    • Display Title
    • Description
    • Sensitive Data Mask
  • Indexed for global search

All other attributes are overwritten.
