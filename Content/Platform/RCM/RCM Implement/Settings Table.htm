Settings Table

The table within the Settings tab includes the following information:

Column

Description

Id

The id number assigned by the system for the setting definition that appears in this row.

Name

The name that is assigned to the threshold, list or reference definition. This is the name that appears within the Threshold/List/Reference activity bar within the Risk Case Manager application.

Page title

The title that appears at the top of the page when a specific threshold, list or reference item is selected within the Risk Case Manager application.

Table name

The table that is accessed in order to read or write required information for the setting definition associated with this row.

In the case of thresholds and lists, this is the table that enables the Risk Case Manager and the AIS to coordinate information regarding the threshold and list requirements. In the case of reference settings, this is the data source table that provides the reference data.

Select statement

The SQL statement that defines the relations between the Risk Case Manager and the data source.

Connection id

Indicates the connection used for running the query (e.g., -1 is the RCM Repository). For more information about connections, see Managing Connections - Admin Site Connections Tab.

NOTE:

If your database is NMSSQL, the database table or view must use NVARCHAR (not VARCHAR) column types.

Sub title

Displays the category (if defined) that appears within the Thresholds/ListsReferences Activity Bars. For example, below (taken from the Thresholds in the RCM UI) Cities title is the category or “sub title” that was defined for this threshold setting, and Cities was assigned as the name of the threshold.

Type

Indicates whether the setting is defined as a threshold, list or Reference.

Fl enable import

Indicates whether or not the Import xsl/csv feature is in effect. You can set up this feature from the Advanced Features window. When import is enabled, an “Import xsl/csv” button appears within the corresponding module of the end users application.

Access type

Indicates whether the current setting has been configured for Full Access, Update or Read-only. You set up user access from the Settings window that opens when you create a new setting or edit an existing setting.

Default script

Displays the full SQL script that was (optionally) entered in Advanced Features window within the Restore defaults script field. When a script is entered, a Restore Default button appears within the corresponding module of the end user application that runs this script.
