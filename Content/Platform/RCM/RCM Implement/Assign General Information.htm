Assign General Information for Alert Type

The following describes the Alert Type General tab.

In the Alerts bar Types tab select an alert type. The General tab opens.

The following parameters are found in the General tab:

Parameter

Description

Name

Name of the alert type.

Identifier

Enter the identifier that you want to appear within the RCM.

Description

Provide a description, if required. You can access a larger text area for the description by clicking the Edit button beside the field box.

Abbreviated Name

Enter an abbreviated name for the alert type.

Display Name Template

This field is used for creating a complex displaying value of the item in the RCM Related Items tab > Items Picker window. For information refer to the RCM Reference Guide.

It is used to add a static string combined with the values of the common and custom item fields. The default value of this field when it is left blank is the value of its identifier. Examples for entering values into the field:

The c1 field value is: ${c1}

The item identifier is: ${identifier}

Create New Identifier

Select to create a unique numeric identifier in order to support backward compatibility.

Document type declaration

Indicates whether or not there is an inclusion of a doctype at the beginning of the HTML output for the alert display. The value of this parameter must be supported by the VTL, XSL and CSS files used by the alert type. Select the document type declaration:

  • Explicit (HTML): Multi Web browser support. Refer to the RCM Installation Guide for the full list of supported web browsers.
  • Without (Compatibility Mode): Backward compatibility mode. Supports the Internet Explorer web browser.

NOTE:

  • When an alert type is created in the Case Item category and the Use default display for case item details checkbox is selected (in the Alert Details tab), the alert type will always have the Explicit document type.
  • In order to use the updated attachments functionality in item details, the Document type declaration must be set to Explicit. Items with the Without (Compatibility Mode) declaration type, display the existing attachments functionality from the details page. For more information about attachments, refer to the RCM Reference Guide.

From Version

An optional version string (made up of numbers separated by dots such as, 10.3.4 or 1.0.0.33). The version is the minimum solution version in the XSL pathfor the specific alert type. When specified, this value determines the minimal alert type version that should have the doctype header. Older alerts will still appear without the doctype declaration.

NOTES:

  • The From Version value only applies to alert types in Explicit (HTML) mode which are not of the Case Items Alert Category.
  • If no value is specified, all alerts of the given type will have the doctype header.

Category

Select the category for this alert. The available categories are the following:

  • Alerts: Alert type is for work items. Work items of this type contain information about suspicious transactions or other activities for a specific entity or party (the default setting).
  • DART: Alert type is for DART (Investigation Tool Alert). See Setting Up DART Views Setting Up DART Views, page 1.
  • Ad-hoc Blotters: Alert type is for ad-hoc Blotters.
  • Case Items: Alert type is for case work items. Case items are work items with cases functionality.
  • Entities: Alert type is for entities. An entity is an individual (customer, party, counterparty, institution) and information about the individual (such as, address) from an external perspective.
  • Custom: A customized work item. See also Work Items.

Allow Manual Alerts

Select the checkbox to enable the RCM user to create manual alerts of this type.

If you selected this option then you must also do the following:

  1. In the GUI tab of the admin.jsp site, edit the GUI item alerts.actions.titleworkItems.createalert.

  1. Select the checkbox Visible and click Finish.
  1. Click Apply Changes.

Manual alerts are created via a wizard where the user enters all the required information on the alert.

For more information refer to the RCM Reference Guide or online help.

Indexed for Global Search

Select the checkbox in order to add all alerts of this alert type to the Global search index of RCM.

Refer to the RCM Installation Guide for more information on installing and configuring the Solr search server.

Apply Entity Matching/Resolution

Select the checkbox to enable RCM to enable the entity matching or entity resolution feature and verify whether or not any incoming work item of this type already exists.

Details Can be Shown in Visual Story

Select the checkbox to enable the Visual Story View tab. This allows the alert type details to be displayed in the Visual Story View format in RCM. For details see Setting Up Visual Story View.
