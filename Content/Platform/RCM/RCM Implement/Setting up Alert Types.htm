Setting up Alert Types

The RCM Designer enables implementers to set up the alert (work item) types that are used within the RCM application.

  • For information about Alert Type Settings related to entity relations, see Setting Up Work Items/Alerts Type Relations
  • Changing an alert type must be done in coordination with definitions within the AIS Model that generates the alert type.

Setting up an alert type involves the following tasks:

  • Providing a name, description, alias (optional) and ID for this alert type.
  • Indicating the XML and XSL files that are used for this alert type.
  • Assigning the predefined notes for the alert type.
  • Assigning the predefined steps or selecting a predefined workflow for the alert type.
