Email

The text is interpreted as a valid email address and is displayed as link to an email message for the specific address. For example sales@actimize.com is displayed as the email link sales@actimize.com.

Number

Formats the value based on the input mask that is entered in the Format String text box (input mask example: ###0,00).

Hidden zeroes

Hides zero values and displays all other values as strings with no additional formatting.

Date

This formatter is relevant only for fields that display a date. In this selection, you must also enter the format string for the date (e.g., mm/dd/yyyy) in the Format String field.
For example: Format string field input is mm/dd/yy displays the text 4/12/2004 as 04/12/04. Additional values include:

ACM_date – formats the date using the configuration parameter “actimize.common.dateDisplayFormat” (i.e., “12/31/2006”)

ACM_time – formats the date using the configuration parameter “actimize.common.timeDisplayFormat” (i.e., “11:59 AM”)

ACM_datetime – formats the date using both of the above in this order: <DATE><SPACE><TIME> (i.e., “12/31/2006 11:59 AM”)

If no formatter data is supplied then configuration parameter date format is used (either date or datetime – depending on the field’s data type).

If the formatter data contains one of the built in constants (acm_date, acm_time & acm_datetime) then the appropriate format is used accordingly.

If the formatter contains some other string then that string is used as the specific date format (for example: mm-dd-yyyy).

If an incorrect value is entered in this field then the date is rendered to the MM/dd/yyyy format.

User Local Date

Enables you to set the time zone for the view according to the user’s local time zone.

Format String for User Local Date is the same as the Date option format (see above).

For this configuration you must set the system time zone in the actimize.common.timeZone config parameter, and the user local time zone in the Administration module. For more information, refer to the RCM Reference Guide.

For LDAP users, the time zone can also be set using LDAP properties.

Icons

A number of special fields can be displayed as icons. For example, the Owned by me field can be displayed as . In order to display these special fields as icons, select the following:

For Attachments select Attachments as the column display formatter.

For case step field select Case Step – issue in order to display an image of the case step (Issue/Non-Issue/Undetermined).
Alternatively, select Case Step – state in order to display an image of the case of the step state (Open/Closed).

For Priority field select Case Priority in order to display an image that is based on its priority’s value.

For Step ID field select Step – issue as the column display formatter.

For State field select Step – issue as the column display formatter.

For Owned by me field select owner as the column display formatter.

For Score field select Score in order to display an icon that is based on the score’s value.

For Signoff field select Signoff in order to display an icon that displays the signoff step (Signed off, Not signed off).

For Reviewed select Trade Reviewed in order to display an icon that indicates whether the trade is reviewed or not.

XSL

Generates a pop-up window with alert details that are enriched via an XML file. The format is based on a specific XSL field.

Enter the following parameters in the Format String field: <xsl_file> (relative to actimize.alerts.pathToXSL config param) ,<width>(optional) ,<height> (optional)

Example: BX.XSL,200,200

XSLT rendering allows the developer to add a custom alert pop-up window that is a subset of the alert details XML as follows:

1.In RCM Designer, create an Alert View.

2.In the fields tab select any field and select XSLT as the "Column Display Formatter".

3.In the “Format String”, enter the rendering tool parameters:

<XSL-FILE-NAME>[,px-width[,px-height]]

where <XSL-FILE-NAME> is the XSL file path relative to the “actimize.alerts.pathToXSL” configuration parameter. The px-width and px-height parameters are optional. You also have the option to only enter the px-width parameter as the px-height parameter is calculated automatically.

Jump to Alerts

Displays any field with a link to related alerts, and resembles the JumpToDynamicAlerts JavaScript function.

Format String textbox should include the following parameters: <DrillDownQueryIdentifier>,[< drillDownQueryParameter(s)>],<list | details>,<alertsViewIdentifier>,<free text view name>

Parameter

Comments

<DrillDownQueryIdentifier>

A drill down query identifier that should be in the following format "select alert_id from alerts [where ...]"

<drillDownQueryParameters>

An ampersand delimited list of custom field identifiers (not common fields) that are matched respectively with the drill down query parameters (or empty if the drill-down query has no parameters).

<list|details>

Displays the results in an Alert list or in an Alert details mode

<alertsViewIdentifier>

Any valid system or user defined Alert View identifier.

<free text view name>

A custom title of the alert list view

Example of input for a query named accountAlertsQuery with one parameter, and a transaction custom field with the identifier accountNumber:

accountAlertsQuery, accountNumber,list,allAlerts,Account Alerts

Example 1:

DDQ: getAlertsByDateRange(fromDate, toDate)

Identifiers of the custom fields that the DDQ parameter values will be taken from (respectively): cdFromDate, cdToDate

Display mode: list

View Identifier: allAlerts

Custom view name: Account Alerts

Format string structure: getAlertsByDateRange,cdFromDate&cdToDate,list,allAlerts,Account Alerts

Example 2:

DDQ: getFirstAlert()

Display mode: details

View Identifier: allAlerts

Custom view name: Account Alerts

Format string structure: getFirstAlert,,details,allAlerts,Account Alerts

Trade Alerts

This option is only available for Transaction views and only for fields that are associated with icons.

For example, the field no. of alerts is associated with the icon, and it is displayed in the RCM UI when Trade Alerts is selected as the Display Formatter.

Blotter Alerts

Displays an image indicating whether or not the Blotter has related alerts.

Blotter Trades Link

Formats a Blotter identifier string value into a link that opens the trades (transactions) window.
