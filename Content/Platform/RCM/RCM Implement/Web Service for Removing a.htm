Web Service for Removing a Record from a Platform List

removeListItems

RCMResult removeListItems(in String listDataVersion, in String listIdentifier, in byte[] listData)

Receives a list identifier and an XML structure that contains the key metadata structure along with the specific row(s) for deletion.

  • listDataVersion – For future versioning.
  • listIdentifier – The identifier of the list from which the data is removed.
  • listData – Zipped list data, in an XML format (see below) with the key metadata and the values for removal.

The web service validates the metadata key structure in the XML and that is valid compared to the list. After the metadata is validated the rows for deletion are removed from the list one by one. Once the process is complete a single audit is added to the list indicating which records were deleted. If a row supplied does not exist in the list, a warning message is logged but the transaction does not fail. Before running this Web Service, verify that you have the sufficient permissions to delete its items.

XML Structure

<list identifier="">

  <fields>

    <field aisIdentifier="x1" type="string" />

    ...

  </fields>

  <rows>

    <row>

      <item value=”one” />

     ...

    </row>

    ...

  </rows>

</list>

The XML structure is similar (but not identical) to that of the add/update web service. The fields section contains only the keys of the list and in their order. The rows section contains the items of the list in the corresponding order. The items values can never be null as they are all part of the key.
