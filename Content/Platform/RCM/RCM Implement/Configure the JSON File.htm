Configure the JSON File

It is recommended to copy an existing JSON file and then modify it for the Best Practice that you are defining. Initially, save the copied JSON file with the name of the Best Practice that you are creating and create a folder that has the same name. For example, ‘Create a Platform List.json’ and the folder ‘Create a Platform List’.

Each JSON file contains the Best Practice name, description and steps’ properties. The Best Practice JSON file includes the following properties:

  • BestPracticeName: Enter the name of the Best Practice as it will appear on the menu. For example, Create a Platform List. This name should be identical to the Best Practice folder name.
  • BestPracticeDescription: The description of the Best Practice. For example, "This best practice explains how to create a Platform List in the RCM Designer".
  • Steps: Each step is defined via the following properties:
  • Name: Enter the name of the step as it will appear on the panel. For example, "Create a New Platform List Definition".
  • Type: There are two types of steps:
  • Designer: The step is done in the RCM Designer.
  • Informative: The step is done on another application (e.g., AIS) and information is provided on this step without pointing to its location.
  • Description: Provide a description of the specific step as it will appear on the Best Practice Panel. For example, Select between an AIS or RCM defined Platform List.
  • Icon: Define the step icon. The available icons are defined in the BestPracticeIconsTable.doc file that is provided in the Best Practices subfolder of the RCM Designer’s build folder. For example:

C:\Program Files (x86)\Actimize\CaseManagerDesigner\Bin\Best Practices\ BestPracticeIconsTable.doc

  • Images: The images are defined for the related help (More Details) window of the step. Initially, you should create a subfolder in a specific Best Practice folder that has the same name as the step. For example, the folder “Create a New Platform List Definition” is created as a subfolder in the “Create a Platform List’ folder. In this folder all the help files for the step are inserted.
    Insert the image files in the subfolder. These files can be any type of image files (bmp, jpg, png, etc.).

Enter the names of the images according to the order in which they will appear in the More Details window. For example,
"Step1Image1.png",
"Step1Image2.png",
"Step1Image3.png",
"Step1Image4.png",
"Step1Image5.png"

  • GoLocation: Define the location in the RCM Designer that is displayed once the Go button of the step is pressed. Note that this configuration is relevant only to Designer type steps and not to informative type steps. If you are defining an informative step then define the value of the GoLocation as ‘null’. The GoLocation includes the following properties:
  • Module: This is the name of the module as it appears on the Entities bar. For example, Platform Lists.

  • Tab: This is the name of the tab as it appears on the screen. For example, Platform Lists

  • Document: Optional property that defines an object in the RCM repository. For example, List 2 (one of the lists displayed in the previous screenshot):

  • DocumentTab: Optional property that defines the specific tab of a page. For example, Fields:

  • Command: Optional property that defines the specific button in the toolbar of the tab. Name of command is the same as the tooltip name without the text that may appear in parentheses .For example, Add new field.


If you do not define the command property then the orange arrow will not be displayed after the Go button is pressed.
