Configure the Custom Fields

Go to the Custom Fields subtab of the Alerts Type Fields tab and configure the specific custom fields of the CWI.

The icons available within the Alert Type Custom Fields window are:

Icon

Description

Enables you to add a new custom field to the alert type.

Deletes the selected custom field.

and

Move a specific alert type custom field up or down in the list to change its viewing order.

Toggle the display of the identifiers of the fields.

Link to the Alert Fields tab for editing the custom fields.

The options that are available within the Alert Type Custom Fields page are:

Option

Description

Field

Select a custom field by clicking the arrow next to this field

Is Mandatory

Select True or False:

True – The user must populate the field

False – The user does not have to populate the field

Update Mode

Not Updateable – The user cannot update this field

Updateable – The user can update this field

Updateable On Creation – The user can only edit this field when a manual alert is created

Hide on Edit – The field does not appear when the user edits the alert

Hide Always – The user cannot view this field.

Aggregation Function

Select an aggregation function for the field. Note that not all aggregations are available for a specific data type. See the table below:

 

First

all types of fields

Last

all types of fields

Sum

only numeric field types (integer and real)

Min

numeric, dates and timestamp

Max

numeric, dates and timestamp

Count

numeric only

Concatenate

strings only

 

The custom field Consolidation Key must be defined in this table. It can be any custom field that is assigned to the alerts.consolidation_key field in the RCM repository. (The original name of this field is Consolidation Key.) The Consolidation Key must be defined with the aggregation function First. All other custom fields are optional for consolidation. For details about defining custom fields, see Add Custom Fields to the Alert Type .
