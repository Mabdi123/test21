Export Translations

Exports the language list to a comma delimited (CSV) file. This is usually done in order to provide a translator with a simple file for entering the translations. CSV files can be opened by popular spreadsheet programs such as Microsoft® Excel®.

Export and Import >

Exports only the selected (marked) rows of the language list to a comma delimited (CSV) file.

Groups the list via the selected column. Note that the selected column must be dragged to the specified area in the page as shown in the following screenshot.

Opens a menu that provides you with the option to drag additional columns to the table. To remove a column form the table, drag the selected column heading to the Select Columns window.

The following columns can be added:

Allow Update – Displays whether or not the resource string can be modified

Screen Modules – Displays the screen modules where the resource string appears

Translation Max. Length – The maximum length of each base value’s translation string. For example, the base value Case has a Translation Max. Length of 5 characters. Therefore, the translation for Case in any language should not exceed 5 characters. Note that if the length is exceeded then the string may not be displayed properly in the RCM UI. If you plan on exceeding a specific string length, then you should view the GUI to verify that it is displayed properly in the RCM UI.

Description, Modules, and Screens columns include the relevant metadata on the string (the Base Value). This metadata is received from the Risk Case Manager.

Step - displays the state of each row (invalid/valid identifier). Provides you with the option to group all the invalid identifiers.

Opens a font window that enables you to customize the font attributes of the text that is displayed in the list.

The Language Information Bar

The information bar includes the following information:

  • Marked For Export – Indicates the number of strings that have been marked for export to a CSV file.
  • Modified – The number of rows that have been modified.
  • Translation – The number of Base Values that have been translated out of the total number of strings.

The Languages Table

By default, the Languages Table includes the following columns:

Column

Description

Indicates that the row has been modified

Identifier

The identifier is the internal Id of the Base Value and is part of the metadata. It cannot be modified.

Base Value

The original US - English string for the Identifier

Translation

This translation for the base value. Translation fields can be filled in by the implementer or imported to a CSV file.
