Create a New Connection

The following window opens when you create a new AIS connection:

To create a new connection:

  1. Enter the required information for the new connection. The following fields should be filled in:

Field

Description

Virtual Connection

Select this checkbox only if you are planning to create AIS connections in a Multi-Tenant environment. For more information refer to Create AIS Connections in a Multi-Tenant Environment.

Identifier

Enter a unique identifier for the AIS connection.

URL

Enter the AIS URL.

Use LDAP Authentication

Select LDAP authentication to only require the user name and have LDAP authenticate the user. If selected, the password field is disabled.

AIS supports two methods for authenticating AIS users – internal authentication and LDAP authentication.

With internal authentication, AIS receives from RCM a user name and password upon connection, and confirms that they match the credentials stored internally in AIS.

With LDAP authentication, AIS receives from RCM an LDAP domain and user name upon connection, confirms that the user name is defined in AIS and also that the user name is defined in the LDAP server.

In both methods, the user name provided is only checked on the AIS side, and is not authenticated as an RCM user, whether or not there is an RCM user defined with the same user name.

The connection will use LDAP authentication if the Use LDAP authentication checkbox is selected, or internal authentication otherwise.

User name

Enter the user name of an AIS user with admin privileges. If using LDAP authentication, enter both the domain and user name in the format Domain\User.

Password

Enter the password of the AIS user (in User name field). If using LDAP authentication, this field is not applicable.

Keep alive

Select this checkbox to keep the TCP/IP connection with the AIS alive by verifying it at predefined intervals (see Keep alive timeout field). By default, this option is enabled.

Keep alive timeout (sec)

Enter the interval in seconds at which to verify that the connection is alive. Default interval is 149 minutes (8940 seconds).

Pool Size

Enter the maximum number of connections to AIS that RCM may keep concurrently for this connection definition.

Call timeout (ms)

Enter the call timeout in milliseconds. The value of this parameter indicates the amount of time that RCM waits for AIS to answer before it displays an error message. -1 indicates no timeout, RCM waits for an AIS response indefinitely.

 

  1. Add AIS functions. AIS and the function names are not bound during solution development (i.e., not hard coded in XSL).   1. Save the connection.
