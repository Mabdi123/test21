Manage DBCP Connections

When DBCPConnectionManager is set as the connection management option, the following fields appear within this page:

Field

Description

Connection Name

Assign a name to the new connection.

Database Type

The type of database for the new connection. The options are

DB2

Hive-Cloudera

Hive-Hortonworks

Impala (using the hive2 driver)

Impala (using the impala driver)

MSSQL (Unicode)

MSSQL

Oracle (service)

Oracle (sid)

Oracle 10

Spark

Sybase

BigSQL

NOTE:

The Database Type must always be configured.

URL

The URL for this connection

Database Name

The database name for this connection.

User Name

A user name that has access privileges for the database to which you are connecting.

Password

The password, for the above user, for the database to which you are connecting.

Max active

The maximum number of active connections that can be allocated from this pool at the same time, or zero for no limit.

Max Idle

The maximum number of active connections that can remain idle in the pool, without extra ones being released, or zero for no limit.

Min Idle

The minimum number of active connections that can remain idle in the pool, without extra ones being created, or 0 to create none.

Max Wait

The maximum number of milliseconds that the pool will wait (when there are no available connections) for a connection to be returned before throwing an exception, or -1 to wait indefinitely.

Max Open Prepared Statements

The number of prepared statements that RCM should cache per connection. Enter 0 to disable caching, -1 for unlimited.

Test Query

Enter an SQL select statement that is activated in order to validate that the connection is successful.
