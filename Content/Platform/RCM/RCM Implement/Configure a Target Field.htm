Configure a Target Field

The Target field that is mapped to the Consolidation Key field should be the initial row in the Target field table. The row highlighted in yellow impacts the decision to which consolidation item the Transaction Work Item is mapped.

The table columns are:

  • The aggregation function is a view only column that displays the aggregation for this target field, which was defined in the alert fields configuration tab.

  • The Source Type column indicates the type of source, which can be:

    • Source – Mapping is done to the alert field.
    • Expression – Mapping to the field source is done via an expression.
  • In the Source Field / Expression column, enter the specific alert field source or expression. If the source type is an alert field, from the drop-down menu, select the alert field as displayed in the following:

If the source type is an expression, enter the expression in the Spring Expression Language (SpEL), for example:

For more information about entering expressions, see Fill in Expressions.

The CWI must have a BU field with a value mapped to it and the value can be defined by providing an alert field or an expression. Data types must match for all mappings.

  • Populating the Condition Expression column is optional. This column adds a condition for mapping the field. The field is mapped only when the condition is true. The condition applies only to the consolidation of the specific field unless it is the consolidation key field (alerts.consolidation_key field in the RCM repository). When the consolidation key condition evaluates to false, no consolidation action occurs for that relationship. This mechanism enables determining which consolidation relationship is used when two work item types are related by more than one relationship. A blank (empty) condition is the same as a condition that evaluates to true.

Conditions are populated using the Spring Expression Language (SpEL).

The condition expression can only store Boolean conditions, and the expression as a source can store different types of results.

Examples of expressions as a source:

${alertEntityId}+' '+${score}

${score} < 50 ? ${alertId} : ${owner}

(In the second example, there is a condition, not a Boolean; if the score is less than 50, the alertId field is shown, otherwise the owner field is shown.)

Examples of condition expressions:

${score}>50 and ${owner} == 'admin'.

${createDate} > toDate('2009-11-11',’yyyy-MM-dd’)
