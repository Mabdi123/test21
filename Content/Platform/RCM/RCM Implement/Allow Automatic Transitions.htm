Allow Automatic Transitions

Select the Allow automatic transition when touched checkbox to enable this parameter.

An alert (work item) is considered as “touched” when one of the following actions occurs:

  • The alert or case item priority is modified
  • The alert score is modified
  • The alert is assigned to a user and/or BU.
  • A note is added to the alert.
  • The alert is edited.
  • Change priority is performed on a case item.
  • Item details are viewed (see the actimize.common.touchOnView configuration parameter).

NOTE:

Each step node can have only one transition node that enables automatic transition when touched.

Automatic Transitions

In addition to the checkbox, automatic transitions are enabled as follows:

  • When the alert is not associated with a workflow, any common action (assign, add note, change score, change priority, update) causes the alert to move from the default step to the In process step.
  • When the config parameter touchOnView is on, the alert is in the default step, and the user views the alert details, it will change automatically to the In process step.
