Overview of RCM Customization

Authorized administrators can use the GUI tab within the RCM Customization application to:

  • Show and hide modules (tabs) within the Risk Case Manager application
  • Show and hide actions and links that are available within each tab
  • Customize the general look and feel of the application

In the RCM Designer, click the Admin Site link or enter admin.jsp in the RCM URL to open the Admin Site.

NOTE:

The GUI Items tab is not available in production mode.

The GUI tab displays a table with all GUI items within the application and their definitions:

The following options and information are displayed in the GUI Items table.

Option

Description

Allows you to select a row in order to perform one of the actions available within the GUI Actions activity bar.

Icon

Displays the icon of the GUI item when relevant. For custom icons, the class name is displayed as a tooltip.

Name

The name of the GUI item as it appears within the RCM application. Click the link to edit the GUI item. See Edit GUI Item.

Auth

Shows the object, associated with this GUI item, as it appears in the authentication permission environment.

The string that displays in this column contains the following information within it:

  • The tab name that this item is assigned to,
  • The type of GUI item
  • The name of the GUI item.

For example alerts.actions.titleworkItems.addnote indicates that this GUI item appears within Work Items, as a button, inside the main toolbar group actions, and performs the Add Note action.

Visible

Indicates whether this item is currently visible within the application. The icon means that it is visible and no icon means that it is not displayed.

Type

The type of GUI item, for example, Action or Menu.

NOTE:

When you filter the type, it is displayed according to type number defined in the database and not by the type name display in GUI item table

Group Type

The classification of the displayed GUI item, for example, Drop Down Icons, Toolbar Icons.

Column Order

Indicates the position of the GUI item in relation to other GUI items in the same container. For example, the order of an Action in an Action Group.

Script

Displays the script used to perform the action associated with this GUI item.
