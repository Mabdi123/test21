assignableItemsDDQIdentifier Parameter

This parameter includes the identifier of the initial mandatory DDQ. This DDQ finds and orders the work items that can be assigned to the user. This DDQ is defined in the RCM Designer. It sets the applicable fields and prepares place holder parameters for optional on-site RCM application configuration. See also Changing the Values of the DDQs in the RCM, page 1.

If there is an error regarding the DDQ default values then an error message is returned.

NOTES:

The DDQ must return the identifier of the work item

The result set size is limited to one record

Examples of DDQ:

Example 1:

select a.item_id

from v_acm_items a

where

   a.owner_join_id is null and fl_deleted=0 and closed_date is null

   and {$permissions$alerts$a}

   order by a.item_date desc

Example 2:

select a.item_id

from v_acm_items a

left join v_acm_alert_custom_fields b on a.custom_attr_join_id = b.custom_field_join_id

where

   a.owner_join_id is null and fl_deleted=0 and closed_date is null

   and {$permissions$alerts$a}

   and b.cs11 = 'Securities'

   order by a.item_date desc
