The PDF Parameters Variable

All PDF-related parameters must be supplied via the $pdfParams argument of the XSL API call as follows:

<xsl:value-of select="actimize:exportRecordsWidgetToPdf( 'dataSourceIdentier',  $pdfParams, $optionalParams)" disable-output-escaping="yes" />

The supported PDF parameters are:

  • maxRows – When exporting record widgets to the PDF, this parameter determines the maximum number of rows to export to the PDF. The actimize.widget.maximumRecordsForPrint parameter in admin.jsp will be used if this parameter is not defined. If the value is supplied, the lower of the two determines the result.

  • inPageSequence – Determines whether the widget is enclosed by the <fo:page-sequence> tag. The default is False.

    • If inPageSequence is True, the widget output assumes that the page sequence is currently open. In this case, the widget will not start with a page break, and the output of the widget will not close the page-sequence tag.
      • When the table size row count per page is 100 (default setting) and the widget is wrapped with a <fo:page-sequence> tag, no page breaking is required. The widget is printed without adding the page sequence tag.
        • If the table size row count per page is greater than 100, the first <fo:page-sequence> opening tag and the last </fo:page-sequence> closing tag are omitted.
          • If inPageSequence is False, the widget will start and end with a page break. The widget will be printed in new page and page breaks will occur every 100 rows.
  • rowCountPerPageSequence – Determines the number of rows displayed (not including secondary layout rows) before an intentional page break occurs. Intentional page breaks are required to release memory accumulated while building the page. The default value is 100, which is also the maximum value. It must be a positive integer greater than or equal to 1. If the value is not valid, the default value will be used and a warning message is entered in the log.
  • masterReference – Determines page size reference as declared in alert2fo.vm. You have the option to add page structures in alert2fo.vm and later reference them using this parameter. Its default value is A4.
  • Title – Adds a title to the widget. This may be useful when ‘inPageSequence’ is set to false.
  • page-header – Adds a string value for each page header.
  • page-footer – Adds a string value for each page footer.

If the results contain more rows than are allowed by the settings, this is indicated in the PDF immediately after the grid that is generated.
