Set up Archive Policy

When a report is archived the materialized report instance, along with supporting data such as notes, is copied to an external file system. (Report instance materialization is when the report becomes a static file managed by RCM. A report instance must first be materialized before archiving is possible.) Once archived, the files are managed by a third party document management system or as part of other file system backup processes.

The options that are available within the Archive Policy page are:

Option

Description

Archive X days after sign-off

This option is only relevant if you select, in the Sign-off page, either mandatory sign-off or optional sign-off.

When this policy is selected, the materialized report is archived at the specified amount of time after a user signs-off.

Archive on first report maintenance after sign-off

All the instances of the Report Type that were signed off are archived during the first time that the report maintenance utility runs (reports_maintenance.bat). When this option is selected together with the Materialize just before archiving the report option, the archiving actions are performed after the office hours in order to optimize the RCM performance during the office hours. For more information on the reports_maintenance.bat utility, refer to the reports_maintenance_readme.txt file in the Utilities\bin folder.

Archive just before the report is being deleted

When this option is selected, the materialized instance of the report is archived (using the format specified in the Report Definition General page) immediately before the materialized instance is deleted from the system by the user.

Archive when the report is being viewed for the first time

When this option is selected, the materialized instance of the report is archived (using the format specified in the Report Definition General page) when a user opens it within the RCM application. Note that the report must be materialized before archiving.

Archive when the report is generated

When this option is selected, the materialized instance of the report is archived (using the format specified in the Report Definition General page) when the report is generated either by the system or by a user.

Never Archive

When this option is selected, reports of this type are never archived.
