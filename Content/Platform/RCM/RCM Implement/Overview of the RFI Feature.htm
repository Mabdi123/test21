Overview of the RFI Feature

It provides the RCM user with the ability to select a work item or a case and send an email for more information on the item. The email can be sent to recipients that are not RCM users. Once the email is answered it is attached to the correct item. In order to support the process of incoming email messages to the RCM you need to define a dedicated email account on organization’s mail server.

The RCM scheduler sends/receives new email messages (using SMTP and POP3/IMAP respectively) from the dedicated email account into the RCM database. RFI currently supports work items and cases (without regulatory forms).

The steps for setting up the RFI are the following:

  1. Set an email account in the organization’s mail server that will be dedicated to the RCM email feature. The mail server should support POP3 or IMAP and SMTP.
  1. Enable the Scheduler parameter. Go to the Config Parameters tab of admin.jsp and set the actimize.scheduler.enable parameter to 1 (Yes). Note that you must restart the server after modifying the value of this parameter.
  1. The parameter actimize.scheduler.user defines the authorized user for the scheduler. The default user is ‘admin’. If necessary, modify this property with the name of the authorized user.
  1. Define the values for the mail server. Go to the Configuration Sets tab of admin.jsp and select the Email Collaboration action link. For more information, see Configure the Mail Server Parameters.
  1. Set the relevant permissions for the designated users and groups. The related permissions are Create New RFI and View RFI.
  1. Optionally, add RFI related actions to the relevant case and Alerts Workflow steps and define them as mandatory.

  1. Enable the RFI GUI items for cases and work items to visible.

  1. Add the RFI State field as visible in the relevant Alert and Case Views via the RCM Designer.
  1. Restart the application server of the RCM.
