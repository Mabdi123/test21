Parameters Related to Setting up the Distributed Cache

RCM uses a cache system in order to improve performance when accessing global application data such as Alert Types, Alert Views, etc. In a distributed environment this cache is mirrored in all RCM instances using JBoss distributed cache.

Once you have deployed the RCM s on each of the machines within the distributed environment, edit the acm.ini file for each deployment in order to switch from a standalone to a distributed cache environment.

To set up a distributed cache:

  1. Remove the comment characters from the following lines:

    • actimize.cache = distributed
      • actimize.cache.distribution.host=239.0.0.0
        • actimize.navigation.cache.distribution.port=45559

    Note that all the machines must be configured to use the same ports.

  1. If there are other systems on your subnet that are using the multicast IP and port that are included in this section, then you should change them. If you decide to change these values, use a Class D IP address. Class D address falls within the range of 224.0.0.0 through 239.255.255.255.

This configuration does not work when machines are in different subnets.

When the RCM servers are spread across a number of WANs (subnets), you must also configure following parameters:

  • Remove the comment characters from the parameter actimize.cache.wan as shown in the following example:

    actimize.cache.wan = true

  • Remove the comment characters from the parameter actimize.cache.wan.machines and enter the local machines IP addresses as the values.
    Note that the list of distributed machines must start with the address of the local machine with the RCM server.
    For example:

    actimize.cache.wan.machines = 212.64.45.54,212.64.45.44
