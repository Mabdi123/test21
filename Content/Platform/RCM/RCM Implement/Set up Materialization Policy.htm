Set up Materialization Policy

A report instance may be materialized at any point from the initial generation of the report instance through to its archival. Report Instance materialization is when the report becomes a static file managed by RCM.

A report is materialized (saved as a static file) to the format that is selected within the General page for the Report Definition (see Define a new Report Definition – Adding General Information).

It is also possible to create a report definition that does not require materialization. In this case, the report is kept in a DHTML format and you can open it again as a dynamic report. You can perform “slice & dice” functions on a report that is in the DHTML format (such as filtering, sorting etc.).

You cannot perform “slice & dice” functions on materialized reports.

The options that are available within the Materialization Policy page are:

Option

Description

Materialize just before archiving the report

When this option is selected, the report instance is materialized to the specified format (e.g., to a PDF file) immediately before archiving.

This option is only relevant if this Report Definition requires archiving for the report. You set the policy for when the report should be archived in the next page – the Archive Policy page.

Materialize immediately after sign-off

When this option is selected, the report instance is materialized to the specified format immediately after a user signs-off. This option is only relevant if this Report Definition requires sign-off for the report.

Materialize when the report is generated

When this option is selected, the report instance is materialized to the specified format when it is generated.

Never Materialize

This option enables you to set up a Report Definition for reports that you want to keep in the DHTML format. This format enables you to open a dynamic report from the RCM and perform slice & dice operations on the report (e.g. filter, sort etc.).

Note that only materialized reports can be archived. This option, therefore, should not be selected if you want the reports to be archived.
