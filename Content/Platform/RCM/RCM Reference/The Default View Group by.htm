The Default View – Group by Document Type

The following is the default view of the Case Contents tab.

The Case Contents subtab has two views. A view can be selected by clicking the arrow next to the subtab:

The default view is Group by Document Type.

A number in parentheses follows the document type title. This number indicates the number of documents attached to this case.

Clicking the Show All link to view the selected documents list opens a window that displays the relevant list. For example, clicking the Show All Notes link opens the Notes list page that displays all notes that are related to the case.

Action Buttons

The following actions are available for each type of document.

Option

Description

Opens a window that lets you add another document for the particular document type.

To remove document(s) from the case, select the document(s) and click the Remove button.

Expands the list of documents for the specific document type and collapses lists of other document types. The button changes to when it is clicked. Click the button to return to the original view.

Attributes for Work items

The following displays the work items associated with the case.

You cannot add work items to the case from this window. To add a work item go to the work items page. (See Add To Existing Case.)

Additional information on work items can be found in Work Items. The columns that appear within the Work Items List of a case are:

Option

Description


(checkboxes)

Click on one (or more) checkbox to remove selected work item(s) from the case. Note that you cannot undo a remove operation.

An icon that indicates whether the corresponding work item is open or closed. This is an attribute of the current step of this work item.

For example, a work item that currently has the In Process step is usually also an Open work item, and this work item state is part of the step definition.

An icon that indicates whether this work item is an issue or not. The work item can be one of the following:

  Issue: This work item requires further consideration

Non-issue: This work item does not require further     consideration

   Undetermined: This work item has not been evaluated.

An icon indicates that the corresponding work item is assigned to me. When no icon appears, the work item is assigned to a different user or is unassigned.

Item ID

The ID number of this Item. This number is used internally by the RCM and can be used when referring to a work Item.

When this Item ID appears within the list as a link, you can click the link to view the item details for the corresponding work item. The Work Items Details window opens enabling you to view various levels of details about this work item, often including links to related Work Items, when relevant. (See Work Items Details).

Item Date

The date on when the work item was triggered.

Item Type

The AIS Model that generated the work item (for example, Best Execution BEX or Front Running FR) according to a rule or standard that caused the work item to be triggered. The list of work items types is defined according to the needs of your organization.

Step

The current step of the work item. Step is defined according to your organization’s needs. For example, typical step definitions may be Ready, In process.

Description

A summary of the work items details.

Score

Enables you to select the priority for this case. Following are the priority options:

  red represents high priority.

  yellow represents medium priority.

  green represents lowest priority.

Owner

The name of the user responsible for investigation of this type of work item. For example, when you select to view only My Work Items the user name is always your name.

Business Unit

Indicates the full name of the current business unit, as it displays within the work items and reports.
