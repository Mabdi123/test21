Add External User

Click the Add External User link within the Manage Users activity bar to open the Select External User window that enables you to search for any LDAP user that you want to add to the . This feature is only available for specific users. See your administrator for more information.

Once all LDAP groups are defined within the database, it is possible to automatically synchronize the list of users with the LDAP list of relevant users, using the Manage LDAP -> Synchronize option. (For more information about automatic synchronization, see Synchronize List of Users with LDAP Users List). If, however, a specific LDAP user does not appear within the list of users after synchronization, Administrators can search for that user and add the user manually using the Select External User window.

To search for an external user:

If there are multiple domains, select the domain under which the user is registered.

  1. Within the User name prefix field, enter the user name or any letter that is included within the name.
  1. Click Go to search through the LDAP user list for the user that you require. A list of all LDAP users that contain the entered letter(s) appears within the Available Users list box.
  1. From the Available Users box, select the user that you want to add to .
  1. Click Next to proceed to save this user to the database and to continue to the next status in the wizard. The second status in the wizard displays additional information. Some of the information may be supplied by LDAP and some may be information that the user must supply.
  1. If you want to allow this external user to also be able to access the using the internal login page, click the Allow logging into RCM using a login name and password checkbox.

The window expands, enabling you to enter an internal user name and password for this user.

An external user is added to the only if that user is linked to a group and the group is linked to a role. If an external user attempts to login to the via Single Sign On and they are not linked to a group that has a role then they will not be able to login to the .

You cannot assign users to external LDAP groups since this information is based on the LDAP server configuration.

You can delete external users.
