Additional Drop-down Menus

After the initial drop-down menu of the condition, the additional drop-down menus provide the following options:

  • Field – The default selection that enables you to select a field from the drop-down menu.
  • Expression – Click the Open Picker icon to open the expression builder window, which enables you to enter expressions for the condition. See also The Expression Builder, page 1.
  • Value – Enter a value for the specific element of the condition.

The Expression Builder

The Expression Builder can be used to:

  • Set a field expression (instead of selecting a pre-defined context field).
  • Set a condition expression (instead of selecting a fixed value or a pre-defined context field).
  • Set a full rule expression.

The Expression Builder includes the following tabs:

  • Fields: Enables you to enter fields into the expression
  • Functions: Enables you to enter functions into an expression. This may include AIS user defined functions.

Toolbar of the Expression Builder

The toolbar of the Expression Builder includes the following options:

  • Clear – Clears the current contents of the expression
  • Validate – Validates that the expression is valid and does not include errors. To validate an expression, click on the Validate icon. If the expression is valid then an Expression is valid message appears with a timestamp.
    If the expression is not valid then an Expression is invalid message appears with a timestamp. A list of errors also appears as shown in the following example:

    The expression’s errors are displayed in red fonts.
  • OK – Adds the expression to the condition and closes the Expression Builder window
  • Cancel – Closes the Expression Builder window without saving the expression

Enter an Expression

  1. Select the field/function you want to add to the expression and click the Insert field/function icon that appears next to the field/function.
  1. Select the returned data type from the Return type drop-down menu (e.g., string, Boolean). Note that this option is only available for the first field of the condition. Other fields’ return types are defined according to the element that was selected.
  1. Click Validate in order to verify that the expression does not include errors.
  1. Click OK.

Use the Auto Complete Feature

This feature provides you with the option to autocomplete constants, platform lists, fields and functions. A list of related auto fill results is displayed when you start typing an expression and then press the Ctrl and Space keys.

Results starting with the search term appear before results that only contain the search term and are grouped in the following order:

  1. Constants
  1. Platform lists
  1. Fields
  1. Functions

All grouped results are ordered alphabetically.
Select the relevant result and press the Enter or Tab key in order for the result to be displayed in the expression builder. The keyboard shortcuts for undo (Ctrl + Z keys) and redo (Ctrl + Y keys) are supported.

When the AIS server is down an indication is displayed:

It is recommended to use this feature when the AIS server is up and running.

When the AIS server is down there are partial autocomplete results that include functions, fields and any previous cached results that are kept as long as the expression builder window remains open.

The Functions Tab

This tab includes the following expressions:

This guide includes only the descriptions of the functions. For a full explanation of the functions refer to the Actimize RCM Policy Manager Functions and Operators Guide.

Check functions:

  • IIF: Returns one of two values, depending on the value of a Boolean expression. If the value of the Boolean expression is true, then «expression1» is returned, and if the value of the Boolean expression is false, then «expression2» is returned.
  • ISEMPTY: Returns a Boolean value indicating whether a char (string) is empty or not. A RCMreturned value of 1 (true) indicates that the char is empty, and a returned value of 0 (false) indicates that the char is not empty.
  • ISNULL: Returns a Boolean value that indicates whether an expression is null, that is, contains no valid data. Returns True if the expression is Null. Otherwise, returns False.
  • ISNUMERIC: Returns a Boolean value indicating whether an expression can be evaluated as a number. A returned value of 1 (true) indicates that expression can be evaluated as a number, and a returned value of 0 (false) indicates that expression cannot be evaluated as a number.
  • NVL: Returns the second expression if the first expression is null, or the first expression if the first is not null. NVL is equivalent to the function IIf(IsNull[expression],expression2, expression1).
  • SWITCH: Returns one of multiple possible results, depending on a specified value. This function is similar to an IIF function, but enables you to define multiple cases.
  • ZVL: Returns the second expression if first expression is zero. Otherwise, it returns the first expression. ZVL is equivalent to the function IIf(expression1=0.0,expression2, expression1).

Date/Time Zone functions

  • CONVERT_TIMEZONE: Converts one time zone to another time zone.
  • DATE: Returns the current date and time.
  • DATEADD: Returns a new date value, based on adding an interval to the specified date.
  • DATEDIFF: Returns the number of interval boundaries crossed between two dates.
  • DATEPART: Returns the specified part of a given date.
  • DAY: Returns the day part of the date.
  • HOUR: Returns the hour part of the date.
  • MINUTE: Returns the minute part of the date.
  • MONTH: Returns the month part of the date.
  • QUARTER: Returns the quarter of the year that the date is in.
  • SECOND: Returns the second part of the date.
  • TRUNC: Sets the hour, minute, and second part of the date as 00:00:00.
  • WEEK: Returns the week part of the date.
  • WEEKDAY: Returns the day of the week according to the date.
  • YEAR: Returns the year part of the date.
  • YEARDAY: Returns the number of days that have passed until the given date since the beginning of the year (including the day of the date).

Conversion/Creation Functions

  • Create_Set; Takes any number of tuples and returns an in-memory set containing all of those tuples.
  • Create_Tuple: Creates a tuple with the given attributes.
  • To_Bool: Returns 1 if the argument is not zero or if the char is "false". Otherwise, it returns 0.
  • To_Char: Converts the argument to a char.
  • To_Date: Returns the date or converts the argument to a date.
  • To_Double: Converts the argument to a Double.
  • To_Integer: Converts the argument to an integer.

Mathematical Functions

  • ABS: Returns the absolute, positive value of the given numeric expression.
  • ATN: Returns the arctangent of a number.
  • Ceiling: Returns the smallest integer that is larger than or equal to the given number.
  • Cos: Returns the cosine of the given angle (in radians) in the given expression.
  • Exp: Returns e (the base of natural logarithms) raised to a power.
  • Floor: Returns the largest integer that is smaller than or equal to the given number.
  • INT: Returns the integer portion of a number.
  • Log: Returns the natural logarithm of a number. The natural logarithm is the logarithm to the base e. The constant e is approximately 2.718282.
  • Max: Returns the largest value of two or more expressions included in the argument.
  • Min: Returns the minimum value of two or more expressions included in the argument.
  • PHI: Returns the normal cumulative distribution for the specified mean and standard deviation.
  • Round: Rounds a number to the closest integer.
  • Sgn: Returns an integer indicating the sign of a number.
  • Sin: Returns the sine of the given angle (in radians) in the given expression.
  • Sqr: Returns the square of an expression.
  • Sqrt: Returns the square root of a number.
  • Tan: Returns the tangent of the given angle (in radians) in the given expression.

Char Functions

  • ASC: Determines the ASCII character code corresponding to the first letter in a char.
  • Char: Converts an int ASCII code to a character.
  • InStr: Returns the position of the first occurrence of one char within another.
  • LCASE: Returns a char that has been converted to lowercase.
  • Leading: Adds leading characters to an expression (either numeric or char) and fills to a defined size.
  • Left: Returns a specified number of characters from the left side of a char.
  • Len: Returns the number of characters in a char.
  • Ltrim: Returns a copy of a char without leading spaces.
  • Mid: Returns a specified number of characters from a char.
  • Replace: Locates a sub-char within a given char and replaces it with a third char.
  • Right: Returns a specified number of characters from the right side of a char.
  • Rtrim: Returns a copy of a char without trailing spaces.
  • Trim: Returns a copy of a char with neither leading nor trailing spaces.
  • Ucase: Returns a char that has been converted to uppercase.

Mathematical Functions:

  • ACCUMULATE: Returns the given set, with an additional attribute containing the results of the given sub-expression. This function allows use of values from a previous row in the sub-expression.
  • Alias: Modifies the attributes of a set.
  • Avg: Returns the average of the values in a set.
  • Bottom: Returns the highest / lowest value of any set. You can use two attributes in order to sort the set by the first attribute and return the value of the second attribute.
  • Concat: Takes 2 or more sets or tuples and returns an in-memory set containing all given tuples and all entries from all given sets.
  • Count: returns the number of entries in a set.
  • Count_Value: Returns the number of appearances of the given value in the given attribute of the given set.
  • Filter: Returns a set based on the given set that contains those entries for which the sub expression evaluates as TRUE.
  • First: Returns a scalar attribute value from the first / last member of a sorted set and has two arguments - a set and one of the attributes of the set (optional).
  • FirstN: Returns a set with up to N (number of) entries from the beginning of the given set.
  • Get_by_Offset: Returns any specified member of a sorted set as a tuple. You specify the member by its position relative to the first or last member.
  • Get_first: Returns a tuple with attributes that are the set's attributes. If the set is null or empty then the function returns a null tuple. Otherwise, it returns a tuple with the values of the first row of the set.
  • Head: Enables you to specify a filter and returns all consecutive members at the beginning of a set, which comply with the requested filter, until the first member that does not comply.
  • Inner_Join: Returns a new set containing entries for all combinations of an entry from the first set with an entry from the second set. If attribute names are given, they must be pairs of attributes from the first set and the second set alternately, and in this case the resulting set will contain only those entries where the corresponding attributes from the first set and the second set are equal.
  • Join: Returns a new set containing entries for all combinations of an entry from the first set with an entry from the second set. If attribute names are given, they must be pairs of attributes from the first set and the second set alternately, and in this case the resulting set will contain only those entries where the corresponding attributes from the first set and the second set are equal.
  • Last: Returns a scalar attribute value from the last member of a sorted set and has two arguments - a set and one of the attributes of the set (optional).
  • Lastn: Returns a set with up to N (number of) entries from the end of the given set. The given set must be an ordered set, for example, a compound index set.
  • Member: Returns a scalar attribute value from any specified member of a sorted set. You specify the member by its position relative to the first or last member.
  • MemDistinct: Returns a set of all distinct values for the specified attribute(s) i.e., there are no duplicate values.
  • Ostat: Returns a scalar attribute value from a member according to the member’s rank. For example, you can retrieve the third highest value (i.e., from the top), or you could retrieve the fourth lowest value (i.e. from the bottom). You can use a second attribute in order to sort the set by the first attribute and return the value of the second attribute.
  • Outer_Join: Returns the same set as that is returned by Inner_Join, except that entries from the first set that have no matching pairs from the second set will also appear in the results. These entries will have NULL values in the attributes from the second set.
  • Rename_SET: Returns the given set with any specified set name instead of the existing name. All attributes of the returned set will have names prefixed by the new set name, regardless of their previous set names.
  • Select: Returns a set containing the specified attributes from the given set.
  • Set_to_char: Converts the set argument to a character.
  • Shortest_paths: Checks a set of links for the shortest paths between nodes according to a set of nodes that are of interest. A link is any tuple, where the 2 first attributes are scalars and have the same data type. These attributes are treated as the link end points. The shortest path is the path with the minimum number of links.
  • Sort: Sorts a set according to the specified attribute.
  • StDev: Returns the standard deviation of set's entries.
  • Stdevp: Returns the standard deviation (population) of set's entries.
  • Sum: Returns the sum of the given attribute in all the entries of the set
  • Tail: Enables you to specify a filter and returns all consecutive members at the end of a set that comply with the requested filter.
  • Top: Returns the highest value of any set. You can use two attributes in order to sort the set by the first attribute and return the value of the second attribute.
  • TopN: Returns a set with up to N (number of) entries from the given set, that have the highest values in the given attribute.
  • Transform: Returns the given set, with additional attributes containing the results of the given sub-expressions.
  • Var: Returns the variance of set's entries.
  • Varp: Returns the variance (population) of set's entries.
  • User Functions: The Expression Builder may also include User Functions that have been created via the AIS Modeler.

Arithmetic Operators:

  • + (Add): Sums two expressions.
  • - (Subtract): Subtracts one expression from another. If only one expression is supplied, returns the negative value of the expression.
  • * (Multiply): Multiplies two expressions.
  • / (Divide): Divides one expression by another.
  • ^ (Power): Returns the value of the first expression to the specified power.
  • \ (Integer Quotient): Divides an integer by an integer, and ignores the remainder.
  • MOD: Provides the remainder of one number divided by another.

Comparison Operators:

  • < (Less Than): Returns True if the first expression is less than the second expression. Otherwise, it returns False.
  • = (Equals): Returns True if the first expression is equal to the second expression. Otherwise, it returns False.
  • > (Greater Than): Returns True if the first expression is larger than the second expression. Otherwise, it returns False.
  • >= (Greater Than or Equal To): Returns True if the first expression is equal to or larger than the second expression. Otherwise, it returns False.
  • <= (Less Than or Equal To): Returns True if the first expression is equal to or less than the second expression. Otherwise, it returns False.
  • <> (Not Equal To): Returns True if the first expression is not equal to the second expression. Otherwise, it returns False.

Logical Operators

  • AND: Returns True if neither of the expressions is False or 0. Otherwise, it returns False.
  • OR: Returns True if one of the expressions is not False or 0. Otherwise, it returns False.
  • NOT: Returns True if the expression is False or 0. Otherwise, it returns False.
  • Dot: Can be used instead of the Get function.

Options for an Applied Condition

Hovering over an applied condition displays the following options:

  • Remove – deletes the condition
  • Add condition – adds a new condition for the same group that the current condition is in.
  • Elaborate condition – wraps the current condition into a group and adds a new nested condition into it.

By default the OR operator is provided for the nested condition(s).



To change the OR operator click on the OR link. A popup window provides you with following options

  • Switch And/Or – changes all the OR operators to AND. Clicking once again on this link changes all the operators back to OR.

  • Delete branch – removes the specific nested condition
  • Add condition –add a new condition after this group.
  • Click the condition while hovering over it to edit the condition.

Nested conditions can be made up to a maximum of 20 levels of branches.

Add Multiple Conditions to a Rule

You can add multiple conditions to a rule and add the AND & OR operators between them. A group of conditions is 2 or more consecutive conditions that are connected via the same operator (either OR or AND). The default operator for a group is AND.

The Operator Pop up Window

Options that are available on the pop up window include:

  • Switch And/Or – changes all the OR operators of the specific branch level to AND. Clicking once again on this link changes all the operators of the specific branch level back to OR
  • Delete branch – removes the specific condition.
  • Add condition – enables you to add another condition
  • The Top Level Links

  • Add Condition– adds a new condition at the end of the query.
  • Group and add condition– groups all the conditions that are above it into one clustered condition that is composed of two or more conditions and are connected via the AND or OR operator.
    The new condition that has opened is not a part of the grouped condition.
  • Clear all – removes all the conditions from the rule.

Add Definitions and Save the Rule

Once you have finished adding the conditions for the rule, do the following:

  • In the General subtab enter a name for the rule. You also have an option to enter a description for the rule. The Is test rule checkbox enables you to create a test rule. When a test rule is activated its results are stored only for statistical purposes. A test rule does not affect the final policy result (i.e., the final decision on what to do with the transaction such as block, delay, etc.)
  • In the Actions subtab, define the actions for this rule. The actions that are available in this subtab depend on the definitions that were initially made by the RCM implementer. In order to define a specific action, click on its checkbox and select the desired action from its drop-down menu.
  • For example, you may have a Response action that has Block and Pass options. In order to define the action as Block, select the checkbox next to Response and select Block from the drop-down menu.

  • You have the option to view the history of modifications that were previously made to a rule by clicking the History subtab. The History subtab of a new rule is initially blank.
