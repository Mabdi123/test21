Case Work Items Tab

The Case Work Items tab lists all of the work items that have been added to the case item using the Add to Case Item button.

This tab includes the following fields.

Option

Description

Item List Filter: Enter any string in the field and press Enter to search through all the items within the list.

The item category.

Indicates that the corresponding work item is assigned to me. When no icon appears, the work item is assigned to a different user or is unassigned.

Indicates whether the corresponding work item is open or closed. This work item state is an attribute of the current step of this work item.

For example, a work item that currently has the step of In Process is usually also an open item, and this item state is part of the step definition.

Indicates that the work item has attached notes.

An icon that indicates whether this work item is an issue or not (work item findings). The work item can be one of the following:

Issue: This work item requires further consideration.

Non-issue: This work item does not require further consideration.

   No-Determination: A decision has not been made for this work item.

Item ID

The unique identifier of this work item. This number is used internally by the system and can be used when referring to a work item.

The Work Items Details window opens enabling you to view various levels of details about this work item, often including links to related work items, when relevant. (See Work Items Details).

Item Date

The business date of the work item.

Step

The current step of the work item. Steps are defined according to your organization’s needs. For example, a typical step definition may be Ready or In Process.

Owner

The name of the user responsible for investigation of this type of work item.

Score

A numeric value that enables you to evaluate the severity of this work item (i.e., the risk to the relevant entity at the time that this work item was created). Different score levels may be assigned different colored icons. The following color coding may be used for different ranges of scores:

red represents scores of high severity

yellow represents scores of medium severity

green represents scores of lowest severity

The score range for each score level is configured for all work item types according to your organization’s needs.

The score can be based on a number of factors, including previous work items that were triggered for this entity and thresholds defined for the work item type of the work item.

Description

A summary of the work item details.

Business Unit

Indicates the full name of the current business unit, as it is displayed within the work items and reports.

Click to refresh the related items list.

Click to export to csv all work items or all selected work items

To remove documents from the case, select the documents and click the Remove button.
