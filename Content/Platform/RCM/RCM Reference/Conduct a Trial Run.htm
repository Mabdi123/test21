Conduct a Trial Run

Trial runs may be conducted on sample data that has been predefined by your organization. It provides a quick and simple procedure of viewing the type of results that are generated from the rule.

To conduct a trial run, do the following:

  1. Click the Trial Run button.
  1. The Trial Run window opens. It includes the following parameters:
    • Sample size: select from one of the options:
    • Full: The Trial Run use all the records from the sample database
    • Percentage: Enables you to specify the percentage of the sample database that is used for the trial run.
    • Number of records: Enables you to specify the number of records of the sample database that are used for the trial run.
    • Timeout: Specify the timeout interval in minutes or leave it with the default value that was set by your organization. If results are not returned after the timeout has exceeded then the trial run stops.
    • Business Date: Specify the business date of the transactions.

  1. Click Go.
  1. When the trial run finishes, the results subtab appears.

The Results Subtab

The results subtab displays the results from the sample set that meet the criteria of the rule. It includes a toolbar that provides you with the following options:

  • Exporting the data to Excel® as a comma separated values file
  • Reloads the results of the last run.
  • Resetting any filter on the list of items.

Your organization may set the tool bar of the results subtab to include additional options or less options than are displayed in this example.

The list of records includes the sorting and filtering options of DART views.
