Add New User

The Create User window enables a System Administrator to add the standard user details for users who access the by means of internal authentication (i.e., accessing the internal login page and entering an user name and password). Mandatory fields are marked with an asterisk. To access the application via the login page, an user must be assigned an user name and password by the Administrator.

If the Administrator wants to force the user to reset the password after the initial login, then the option User must change password at next login must be selected.

Click the Add New User link within the Manage Users activity bar to open the Create User window. This enables you to add internal users.

When Single Sign On is used by your organization, external user information can be selected using the Add External User option or retrieved automatically from the external authentication system using the Manage LDAP à Synchronize option. (See Synchronize List of Users with LDAP Users List)

The Create User window includes the following fields:

Option

Description

Login name

The user login name.

User identifier

The user identifier.

Full name

The user’s full name.

Business Unit

 

Indicates the business unit to which this user belongs.

Each organization is mapped according to at least one organizational hierarchy. This hierarchy usually reflects the divisions, departments etc. within the organization. Each user is a member of one business unit, which reflects the position of this user within the organizational hierarchy.

The user’s business unit does not determine the permissions that this user is assigned, but may be used during the assignment of roles (which do determine permissions for a user). When roles are assigned to a user, it is possible to assign a role that pertains only to that user’s business unit. For more information about the relationship between business units and roles, see About [%=RCM Actimize.ProductCode%] Permissions.

Email

The user’s email address.

Language

Select the relevant language for the user from the drop-down list.

Time Zone

Select the time zone in which the specific user is located. This is used by any feature that displays date data in terms of a user's own local time zone (including daylight savings time).

In addition to the standard user details, the Add New User window enables you to select a business unit for this user, as follows:

To enter the business unit for a new user:

  1. Click the icon beside this field to open the Business Unit Explorer, to select the Business Unit that you require for this user.
  1. Use the Business Unit Hierarchy drop-down list to select the hierarchy that you currently require.

    2. Once you select a hierarchy, the Hierarchy tree shows the nodes of the highest level Business Units for this hierarchy.

  1. Click in the tree to open a node and view the business units that are children of the selected node.
  1. Select the Business Unit name that you require and click to add it to the User Details page.
