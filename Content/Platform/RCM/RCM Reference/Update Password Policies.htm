Update Password Policies

Click the Update Password Policies link within the Manage Passwords activity bar to open the Manage Passwords Policies window that enables the administrator to setup password policies for the users.

The Manage Passwords Policies window includes the following fields and options that enable you to define general information and behavior of the User passwords:

Option

Description

Minimum Length

Enables you to define the minimum length of an login password. Select checkbox to enable this option.

Previous Password

Enables you to define the number of historical passwords that cannot be reused as a new password. A historical password is a password that was once used as the login password by the user.
Select checkbox to enable this option.

Dictionary

Enables you to add a list of strings to the dictionary. You can select one of the following options:

The strings in the list cannot equal a user’s password. For example, if the string is admin then admin cannot be a password.

The strings in the list cannot start a user’s password. For example, if the string is admin then admin1 cannot be a password.

The strings in the list cannot be contained in a user’s password. For example, if the string is admin then 1admin1 cannot be a password.

To manage this list, click the Edit Dictionary link. For more information see Edit Dictionary.

Login name

The login names cannot be used as passwords in one of the following conditions:

The login names cannot equal a user’s password. For example, if a login name is admin then admin cannot be a password.

The login names cannot start a user’s password. For example, if a login name is admin then admin1 cannot be a password.

The login names cannot be contained in a user’s password. For example, if a login name is admin then 1admin1 cannot be a password.

Select checkbox to enable this option.

Full name

The full names of users cannot be used as passwords in one of the following conditions:

The full names cannot equal a user’s password. For example, if a full name is John Smith then John Smith cannot be a password.

The full names cannot start a user’s password. For example, if a full name is John Smith then John Smith1 cannot be a password.

The full names cannot be contained in a user’s password. For example, if a full name is John Smith then 1John Smith1 cannot be a password.

Select checkbox to enable this option.

Enforce mixed case

Select checkbox to force user to select a password that has both uppercase and lowercase letters.

Enforce numbers and letters

Select checkbox to force user to select a password that has both numbers and letters.

Show date time in header

Select checkbox to display date or date and time on the GUI. This setting takes effect only after pressing the browser’s refresh button.

Number of failed login attempts

Enables you to define the number of attempts a user has to enter a password before the goes into a lock duration mode. During the lock duration the password cannot be entered.

Lock duration (seconds)

Define the length of the lock duration in seconds. A password cannot be entered during a lock duration. Lock duration occurs when the user exceeds the number of failed attempts for entering a password. Entering -1 as the value indicates that the lock duration is unlimited and the user must be manually unlocked. See also Unlock Users.

See also information on the actimize.authentication.passwordExpires parameter in the Installation Guide.
