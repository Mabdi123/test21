Cases Tab

The following is an example of the Cases tab in the Update Role window.

Select the Case Type or all Case Types in the Case Type drop-down menu.

You may allow Supervisors to perform all actions in the Cases module (edit, delete etc.) while someone who is assigned a different role may only have permission to view cases without permission to perform any actions.

To add a permitted action, select one of the following actions from the Actions drop-down menu and click the Add button:

  • All - enables users with this role to have all the available permissions for cases.
  • Add Attachments - enables users with this role to add attachments to a case.
  • Add free text Note - enables users with this role to add text in the text box of a note for a case.
  • Add Items to a Case - enables users with this role to add work items to a case.
  • Add predefined note - enables users with this role to add a predefined note for a case.
  • Archive Case - enables users with this role to archive a case.
  • Assign to BU - multiple selection – enables users with this role to assign one case or more to a specific business unit.
  • Assign to BU - single selection - enables users with this role to assign one case at a time to a specific business unit.
  • Assign to User - multiple selection - enables users with this role to assign one case or more to a specific user.
  • Assign to User - single selection - enables users with this role to assign one case to a specific user.
  • Change Priority - enables users with this role to change the priority of a case.
  • Change Step - multiple selection - enables users with this role to change the step of one case or more.
  • Change Step - recall - enables users with this role to recall the workflow step of a case.
  • Change Step - reset - enables users with this role to reset the workflow step of a case.
  • Change Step - single selection - enables users with this role to change the step for one case at a time.
  • Change workflow task to done - enables users with this role to set a case’s workflow task to done.
  • Create Case - enables users with this role to create a case.
  • Create RFI - enables users with this role to create an RFI for a case.
  • Delete - multiple selection - enables users with this role to delete one or more cases at a time.
  • Delete - single selection - enables users with this role to delete a case.
  • Edit Case Fields- enables users with this role to edit the fields of a single case.
  • Export List- enables users with this role to export a list of cases to a CSV file.
  • Print Details - enables users with this role to print a case’s details.
  • Print List - enables users with this role to print a list of cases.
  • Remove Attachment - enables users with this role to remove attachment(s) from a case.
  • Remove Items from Case - enables users with this role to remove work item(s) from a case.
  • Send Mail - enables users with this role to send a link of a case via an email message of a default email application (e.g., Outlook).
  • Open Case Details - enables users with this role to open and view case details.
  • Quick Change Status - users with this permission are able to change a step of a selected work item based on a predefined configuration that bypasses the change step permission.
  • View RFIs - enables users with this role to view RFIs for cases.
  • View - enables users with this role to view the list of cases. For example, viewing the results of a search.
  • View Attachments - enables users with this role to view the attachments of a case.
  • View History - enables users with this role to view the history of actions for a case.
  • View Notes - enables users with this role to view the notes for a case.

You have the option to set a permission to one of the following in the Assigned to drop-down menu:

  • Any: Cases that are assigned to any user (). This is the default setting.
  • Assigned to me: Cases that are assigned to you ().
  • Assigned to others: Cases that are assigned to other users.
  • Unassigned: Cases that are not assigned to any user.

To add another permitted action, select it and click the Add button. You also have the option to select another Case Type and set the desired permitted actions for it.
