Forms Tab

The following is an example of the Forms tab in the Update Role window.

Select the Form Type or all Form Types in the Form Type drop-down menu.

To add a permitted action, select one of the following actions from the Actions drop-down menu and click the Add button:

  • Amend Form – enables users with this role to amend a form.
  • Change Form Status - enables users with this role to change the form status.
  • Close as Non-issue - enables users with this role to change the status of the form to close as Non –issue.
  • Create Form - enables users with this role to create a new form.
  • Delete Form - enables users with this role to delete a form.
  • Export Form - enables users with this role to export a list of forms as a CSV file.
  • Print Forms List - enables users with this role to print a list of forms.
  • Report Externally - enables users with this role to change the status of the form to Reported Externally.
  • Update Form - enables users with this role to update a form.
  • View Form Details - enables users with this role to view the form’s details.

You have the option to set a permission to one of the following in the Assigned to drop-down menu:

  • Any: Forms that are assigned to any user. This is the default setting.
  • Assigned to me: Forms that are assigned to you.
  • Assigned to others: Forms that are assigned to other users.
  • Unassigned: Forms that are not assigned to any user.

To add another permitted action, select it and click the Add button. You also have the option to select another Form Type and set the desired permitted actions for it.
