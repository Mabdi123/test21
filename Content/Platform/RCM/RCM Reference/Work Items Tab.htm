Work Items Tab

The following is an example of the Work Items tab in the New Role window.

Select a Work Item Type or All Item Types in the Item Type drop-down menu.

You could create a role called Supervisors and allow users who perform this role to view all work item types, while a different role called Officers may be allowed only to view those Work Item Types that are relevant to their tasks.

The Bypass Workflow Review Rules action is not included in the All item type. If you want to include this action, you must select both All and the Bypass Workflow Review Rules action.

To add a permitted action, select one of the following actions from the Actions drop-down list and click the Add button:

  • All - enables users with this role to have all the available permissions for work items.
  • Add Attachments - enables users with this role to add attachments to a work item.
  • Add free text Note - enables users with this role to add text in the text box of a note for a work item.
  • Add predefined note - enables users with this role to add a predefined note for a work item.
  • Archive an Item - enables users with this role to archive a work item.
  • Assign to BU - multiple selection – enables users with this role to assign one work item or more to a specific business unit.
  • Assign to BU - single selection - enables users with this role to assign one work item to a specific business unit.
  • Assign to User - multiple selection - enables users with this role to assign one work item or more to a specific user.
  • Assign to User - single selection - enables users with this role to assign one work item to a specific user.
  • Bypass Workflow Review Rules –overrides review rules for this role and enables users to change to the next step
  • Change Score - enables users with this role to change the score of a work item.
  • Change Step - multiple selection - enables users with this role to change the step of one work item or more.
  • Change Step - recall - enables users with this role to recall the workflow step of a work item.
  • Change Step - reset - enables users with this role to reset the workflow step of a work item
  • Change Step - single selection - enables users with this role to change the step of a work item.
  • Change workflow task to done - enables users with this role to set a work item’s workflow task to done.
  • Create Manual Item - enables users with this role to create a manual work item (Manual Alert).
  • Create RFI - enables users with this role to create an RFI for a work item.
  • Delete - multiple selection - enables users with this role to delete one or more work item at a time.
  • Delete - single selection - enables users with this role to delete a work item.
  • Delete RFI Messages – enables users to delete RFI messages associated with a work item.
  • Edit Item - enables users with this role to edit a work item.
  • Export - enables users with this role to export a work item to a CSV file.
  • Handle Doubt Resolution – enables users to perform doubt resolution on items.
  • Open Item Details - enables users with this role to open and view a work item’s details. To see the work item details page, the following other permissions are also required:
  • View Attachments
  • View History
  • View Notes
  • Print Details - enables users with this role to print a work item’s details.
  • Print List - enables users with this role to print the list of work items.
  • Quick Change Status - users with this permission are able to change a step of a selected work item based on a predefined configuration that bypasses the changing step permission.
  • Remove Attachment - enables users with this role to remove an attachment of a work item.
  • Send Mail - enables users with this role to send a link of a work item via an email message of a default email application (e.g., Outlook).
  • Show as PDF - enables users with this role to convert a work item to a PDF document.
  • View RFIs - enables users with this role to view RFIs of work items.
  • View - enables users with this role to view the list of work items. For example, viewing the results of a search.
  • View Attachments - enables users with this role to view the attachments of a work item.
  • View History - enables users with this role to view the history of actions for a work item.
  • View Notes - enables users with this role to view the notes of a work item.

You have the option to set the permission to one of the following in the Assigned to drop-down menu:

  • Any: Work items that are assigned to any user. This is the default setting.
  • Assigned to me: Work items that are assigned to you.
  • Assigned to others: Work items that are assigned to other users.
  • Unassigned: Work items that are not assigned to any user.

Work items can be assigned to specific users using the Assign to Users action within the Work Items tab.

To add another permitted action, select it and click the Add button. You also have the option to select another Work Item Type and set the desired permitted actions for it.

Single Vs Multiple Selection of Assignments (Bulk Actions)

exposes a new set of actions in the role in order to improve control over users performing actions on multi-selection lists (a.k.a. bulk actions). These actions include enabling a multiple or a single selection for the following:

  • Change Step
  • Assigning to BU
  • Delete (Work Item)

This new capability can be used to prevent entry level reviewers from easily closing a long list of work items.

The feature is available for both work items and cases.
