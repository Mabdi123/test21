Cases List

The following displays an example of the Cases list.

You can sort the list according to the contents of a selected column by clicking that column’s title (For example, click Case Date to sort the list by date, or click Step to view all cases sorted by the Step).

Typically, there is one column within the Cases list that is a link that lets you access the Case Details for the corresponding case (for example, in the example above, Case ID is the link to the Case Details).

The columns that appear within this list are configured according to the needs of your organization and the permissions that are currently in effect. As a result, the options that you see may not be the same as the standard list presented below.

Some of the typical columns that appear within the Cases List are:

Option

An icon that indicates whether this case is an issue or not. The case can be one of the following:

  – Issue – this case requires further consideration

– Non-issue – this case does not require further consideration

   – No-Determination – this case has not been evaluated.

Case ID

The Identifier of this case. This string is used internally by the RCM and can be used when referring to a case.

When this Case ID appears within the list as a link, you can click the link to view the case details for the corresponding case. The Case Details window opens enabling you to view various levels of details about this case, often including links to related cases, when relevant. (See Case Details).

Case Date

The date when the case was created.

Case Type

The category for this case (for example, SAR, AML, etc.).

Step

The current step of the case. Steps are defined according to your organization’s needs. For example, typical step definitions may be Ready, In process.

Owner

The name of the user responsible for investigation of this type of case. When you select to view only My Cases, the user name is always your name.

Priority

Enables you to select the priority for this case. Following are the priority options:

  – red represents high priority

  – yellow represents medium priority

  – green represents lowest priority

Original BU

Indicates the original business unit that this case was assigned to. This is useful where the case was reassigned to a different business unit.

Business Unit

Indicates the full name of the current business unit.

Description

A summary of the case details.
