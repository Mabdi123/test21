Work Items List

Click the Workbench tab and click Work Items from the drop-down menu to open the Work Items list.

 

Click a column title in order to sort the list according to the selected column (for example, click on Item Date to sort the list by date, or click on Item Type to view all work items sorted by the work item type)

Typically, there is one column within the Work Item List that is a link to the Work Item Details for the corresponding work item (For example, in the screen above the Item ID is the link to the Work Item Details). A new item appears in bold in the Work Item list, for the item's owner, until any user views the item's details. For additional ways an item can appear read, see Work Item Indication .

Select a work item by selecting the corresponding checkbox, or clicking anywhere on the item that us not selectable, and the row within the list changes color.

The columns that appear within this list are configured according to the needs of your organization and the permissions that are currently in effect. As a result, the options that you see may not be the same as the standard list presented below.

Item Name

The name of the item given when creating the item.

Item ID

The unique identifier of this work item. This number is used internally by the system and can be used when referring to a work item.

When this Item ID appears within the list as a link, you can click the link to view the Item Details for the corresponding work item. The Work Items Details window opens enabling you to view various levels of details about this work item, often including links to related work items, when relevant. (See Work Items Details).

Item Date

The date when the work item was triggered.

Item Type

The AIS model that generated the work item according to a rule or standard that caused the item to be triggered. The list of work items types is defined according to the needs of your organization. 

Step

The current step of the work item. Steps are defined according to your organization’s needs. For example, a typical step definition may be Ready or In Process.

Owner

The name of the user responsible for investigation of this type of work item. When you select to view only My Work Items, the user name is always your name

Score

A numeric value that enables you to evaluate the severity of this work item (i.e., the risk to the relevant entity at the time that this work item was triggered). Different score levels may be assigned different colored icons. The following color coding may be used for different ranges of scores:

red represents scores of high severity

yellow represents scores of medium severity

green represents scores of lowest severity

The score range for each score level is configured for all work item types according to your organization’s needs.

The score can be based on a number of factors, including previous work items that were triggered for this entity and thresholds defined for the work item type of the triggered work item.

Description

A summary of the work item details.

Original BU

Indicates the original business unit that this work item was first assigned to. This is useful when the alert was reassigned to a different business unit.

Business Unit

Indicates the full name of the current business unit, as it is displayed within the work items and reports.
