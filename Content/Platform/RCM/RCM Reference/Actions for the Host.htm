Actions for the Host

The host is able to define common rules that are a recommended library of rules for all the tenants. Each tenant can decide to use any of the host common rules and add their own private rules.

Common Rules

Common Rules are generated by the host and are universal for all tenants. The host enables tenants to use the rules by activating them (i.e., before they are activated they are kept as a draft only by the host).

The host can define whether or not each rule (i.e., common rule) should be activated by the tenants (the rules that should be activated are referred to as recommended rules). When a new tenant logs in to the Policy Manager they will be able to view all the common rules that the host activated. The tenant can change the host recommendation by deactivating a recommended rule or activating a rule that was not recommended by the host. The tenant can also add their own unique rules,

To mark a rule as common the host needs to define the rule as Activate.

The Rule is defined Activate as default when the host wants a rule to be a recommended part of the tenant’s library of rules. 

The rule is defined Don’t activate as default when the host decides that this rule is not recommended for the tenants.

 

The host has the ability to define priority rules (including their order) while the tenant does not have this capability.

Only the host has the capability to define Test Rules (vs. Production Rules) while the tenant does not have this capability.

Export/import Rule and Activate Policy are host only actions.

All the host actions are found in the More actions drop-down menu.

The For Tenant selection field allows the host to view specific tenant’s rules with all of the host’s features enabled.

In this mode the host can only modify the tenant’s existing rules.

Additional General Actions by Host

The host can also create all of the other actions that are provided by the Policy Manager including:

  • Importing/exporting rules,
  • Activating the policy and using its related utility,
  • Viewing history of the entire policy,
  • Viewing the current policy waiting to be activated by the AIS (Policy activation tab).
