Create a Sub Query

A sub query is a query that is embedded inside another (parent) query and is used as a part of a condition on a specific column. The sub query that is selected is a query that has been previously saved.

To create a sub query:

  1. In the condition operator drop-down, select In Sub Query as the filter for the condition. This action opens another drop-down menu.
  1. Select one of the saved queries for this Data Source file as the sub query and apply the condition.

  1. The sub query is defined as a condition of the main query.

It is possible to select sub queries from other data sources as long as you have linked the relevant data sources via the business types and have the required permissions to use these data sources. For information, refer to the DART Implementation Guide.
