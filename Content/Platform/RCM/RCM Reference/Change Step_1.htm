Change Step

This option only appears when the logged on user has permission to change the step of the case items. You can select one or more case items in order to change the step of the items.

Select a case item and click the Change Step button in the Case Items toolbar to open the Change Step window.

The Change Step window lets you select the required step from a drop-down list of available items. The available step assignments are defined according to your organization’s requirements and may differ in each type of work item.

  • The list of available steps may be short and you may not be able to locate a step that you want to assign to a specific item. As the RCM administrator may have created a specific Workflow of available steps for the item type that you have selected.
  • Certain steps may be disabled according to relevant review rules.

The Change Step window includes the following:

The and icons and the associated filter options may not appear within this window if they were not configured for your organization.

Button

Description

Option

Description

Current Step

The current step you are in for the case item.

Next Step

The name of the step that you want to assign to this work item. For example, your organization may include a step In Process that indicates that the work item is still under consideration and has not yet been resolved.

 

Each step is defined as being in either an Open or Closed state. The state of the work item is part of the work item step definition, and is automatically displayed when you select a step. For example, if you select the Step Archived this is usually associated with the Closed state.

Each step is defined as one of the following:

Issue: This step requires further action.

Non-issue: Does not require further action.

No Determination: This item has not been evaluated.

Whether or not a step is considered an issue is set as part of the definition of the step. For example, an In Process step may be defined as associated with being an issue.

Get Workspace

Enables you to get workspace information and add it to the note for this step. This button is enabled when a workspace contains content. For details, see Open Workspace.

Add to Workspace

Enables you to take note content and append it to the item’s workspace. This button is enabled when note data has been entered. For details, see Open Workspace.

Add Note

This tab enables you to add a note to the selected item. This tab is displayed once you have selected the next step. For details see Add Note.

Filtering the Step List within the Change Step Window

If the list of available step items is very long, filter the list to view only those step names that are relevant. For example: View only those step items that are considered Issues.

To filter the list of available step names:

  1. Click on the arrow beside the Step icon or beside the Issue icon to select the appropriate State/Issue of interest.

    2. For example, if you want to view only steps that are defined as Issues, click the Issue icon to open a selection box.

    Similarly, if you want to filter the list of steps to view only those that are Open, click the State icon.

  1. Select the appropriate item within the resulting drop-down list (Example: ).

    2. Only the step names that are associated with the selected item appear within the drop-down list of steps.

    For example, if you selected to filter the list by selecting Open, only those steps that are considered Open appear within the list of steps.

Select All to see all steps again.

Once you have selected the step that is appropriate for the selected case items, click Save to change the step and this action is recorded in the case item history. If required, add a note for this action to occur.

Adding an Attachment when Assigning Step

Click Add Attachment to add a selected file to the current alert. Use the Browse option to select the file to attach.

For large attachments in Internet Explorer, make sure to set the following browser configuration: Navigate to Tools ->Internet Options -> Advanced -> Enable Native XMLHTTP Support. This is essential for RCM's AJAX calls and jquery widgets to work properly.
