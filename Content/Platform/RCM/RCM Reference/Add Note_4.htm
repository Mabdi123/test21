Add Note

Select one or more case items and click the Add Note button in the Case Items toolbar to open the Add Note window. You can select a note that is available within a list of predefined notes, or to type in a note that is relevant for describing the progress of the items.

The type of note that you can add depends on the items that you selected. Different item types support different note options (for example, some case item types do not support adding free text, or do not support all of the predefined note options).

When you add a note to more than one work item at the same time, you only see those options that are supported by all selected work item types. If you require an option that is not supported by one of the selected work items, try selecting the work items one at a time.

To select a predefined note:

  • Click the arrow at the end of the available field to view the complete list of predefined notes that are configured for your RCM.
  • Select any existing note and then add or edit text according to your needs, or you can directly type into the available field.

The table of case items may include a note column that has the following icon: All items that include this icon are with note(s). Clicking on a note icon of a specific case item displays its related note(s).

Rich Text Format

This feature provides you with advanced styling and editing capabilities for the entered text. For details, see Rich Text Feature.

The Add Note window contains the following buttons:

  • Get Workspace: Enables you to get workspace information and add it to the note. This button is enabled when the case item’s workspace contains content. For details, see Open Workspace.
  • Add to Workspace: Enables you to take note content and append it to the case item’s workspace. This button is enabled when there is data in the note section.

The Rich text format feature may not be enabled by your organization.

Adding a Confidential Note

Users that have the confidential level feature can create a confidential note that cannot be viewed by users that do not have this feature enabled.

To add a confidential note, select the Confidential Note checkbox from the Add Note window.

  • This checkbox may not be available for all users.
  • When a note is marked as confidential, it cannot be added to a workspace.

Note with Tags

Some of the work items and Blotter transactions may include notes with tags.

Tags are fields within the note window that enable you to enter specific information on the note. For example, a tag may be an Account field that enables you to enter the related accounts for the note. Tag values are verified by the RCM. For example an account tag can include validation for a minimum length of 9 characters and a maximum length of 14 characters and it must begin with the letters aac (e.g., aac12345678). If the value is not entered correctly then the account is underlined in red and the note cannot be saved. A tooltip provides information on the reason why the value is not valid.

Values that are underlined in yellow indicate that the value was entered in the correct format but one of the following scenarios occurred:

  • A drill down query (DDQ) on the entered value did not return any results. The value is underlined in yellow and a tooltip provides the reason for the invalid value. For example, Jane Smith is entered as a value for a Broker tag and the system could not find a Broker named Jane Smith when it ran a drill down query on this value.

    • You have the option to confirm this value even when it is not validated by the DDQ.

  • A drill down query returned more than 1 result for the value. The value is underlined in yellow and a tooltip provides the reason for the invalid value. For example, John Smith is entered as a value for a Broker tag and the system found more than one Broker named John Smith when it ran a drill down query on this value.

    • You have the option to confirm this value even when it is not validated by the DDQ.

    A valid value is underlined in black and a tooltip indicates that the value is valid.

The Broker and Account tags are examples for possible tags within notes. Your organization may define different tags than these examples.
