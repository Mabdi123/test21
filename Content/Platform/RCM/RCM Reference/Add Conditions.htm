Add Conditions

Click the link to open a new condition line. The new condition is in editing mode and allows you to set the various properties of the condition.

To add conditions to a query.

  1. Click Add Condition.
  1. Select the relevant field and operator from the drop-down menus. Some options may open additional option menus.
  2. Click Apply to confirm the conditions.
  3. Continue to add conditions.
  4. Click Go to update query results.

The options within the condition are:

Option

Description

Checkbox for the condition. The condition is enabled when the checkbox is selected. Clearing this checkbox from a condition that has been applied strikes it out. This indicates that the condition was removed from the query but can be quickly enabled by selecting its checkbox.

Drop-down menu for selecting a column for the condition. Additional drop-down options are displayed depending on the selection.

Removes the condition from the query.

Applies the changes that have been made to the query.

Discards the changes that have been made to the query, editing mode and displays the link.

Operator Menu

The operator menu opens after selecting a field for the query:

The options in the menu change according to the type of data that is in the column (for example, a column that stores data on dates has different operators than a column that stores data on amounts).

Strings

The following menu provides the following options for columns that store strings (for example, account numbers, city names, account holder name, etc.):

Option

Description
Is

Opens an additional field that enables you to enter or select an entity that must be included in the result of the query. For example, if the column stores account numbers then entering ACC-1 ensures that this account is included in the query’s result.
Is Not

Opens an additional field that enables you to enter or select an entity that should not be included in the result of the query. For example, if the column stores account numbers then entering ACC-1 ensures that this account is not included in the query’s result.
In List

Opens an additional field that enables you to enter or select multiple entities that should be included in the result of the query. For example, if the column stores account numbers then entering ACC1,AAC-2 ensures that these accounts are included in the query’s result. A picker icon () may appear when this filtering option is selected. This picker opens a window which enables you to select or type a specific filtering string. It appears only when your organization defined it to be displayed and when there is data for the selected field in the repository.
Not In List

Opens an additional field that enables you to enter or select multiple entities that should not be included in the result of the query. For example, if the column stores account numbers then entering ACC1,AAC-2 ensures that these accounts are not included in the query’s result. A picker icon () may appear when this filtering option is selected. This picker opens a window which enables you to select or type a specific filtering string. It appears only when your organization defined it to be displayed and when there is data for the selected field in the repository.
Starts with

Opens an additional field that enables you to enter the initial character(s) of the entities that you are searching.
Contains

Opens an additional field that enables you to enter character(s) in the entities that you are searching.
Is Null

Searches for string(s) that have no value.
Is Not Null

Searches for string(s) that are not of a null value.
In Sub Query

Opens an additional field that enables you to select a sub query or create an internal sub query. The query displays the result of the selected sub query

Not In Sub Query

Opens an additional field that enables you to select a sub query or create an internal sub query. The query displays all the results that are not part of the sub query’s outcome.
