Report List

You can click on a column title within the Report List to sort the list by that column (For example, click Business Date to sort the list by that date, or click Report Name to view all reports sorted by their names).

The columns for the Reports list are:

Option

Description


(checkboxes)

Click on a checkbox to perform an operation on the selected report (e.g., add note or view history for the selected report).

When no icon appears within this column, the corresponding report is presented for information purposes and does not require any action.

Report ID

The ID of this report. The report ID’s are hyperlinks that enable you to open the report by clicking on the link.

Report Name

The name of this report.

Status

The current status of the report. Possible report status can be:

New – the report has not yet been viewed or dealt with by any user.

Purged – The has purged the report according to the policy that was in effect for that report. Purging a report instance is the act of removing the materialized file from the RCM and leaving the metadata intact. Once purged, you can no longer open the report from the Risk Case Manager.

Archived – The materialized report instance, along with supporting data such as notes, has been copied to an external file . Once archived, the files can be managed by a third-party document management system or managed as part of other file system backup processes.

  • If a user from a specific BU archived a report, it can then be viewed by a user from other business units.
  • In the Archived view, you can view the details of the user who performed the sign off.

Signed-off: The report has been signed off by the responsible user.

Viewed: The report has been opened and viewed (it is no longer New). Note that a report has the status of Viewed only until some other action has occurred, such as signing off, or archiving.

Certain statuses are stronger (more interesting) than others and therefore override the display of the weaker Status indicators.

Generated on

The date on which the query for this report was run.

Generated by

This column indicates whether the report was generated automatically by the as a scheduled report or was generated On demand by the user. (For more information about On-Demand reports, see On-Demand Reports).

Business Unit

The business unit to which this report was distributed. For more information about business units, see About RCM Permissions.

Owner

The user who is responsible for investigating or viewing this report.

Sign-Off date

When Sign-Off is required this column indicates the date on which Sign-Off was performed. This column is only relevant for reports that have been signed off (for example, reports that appear with an icon).

Sign-Off user

When Sign-Off is required this column shows the name of the user who performed Sign-Off.

Availability

Indicates whether the report is maintained:

Online: The report can be viewed and opened from the .

Offline: The report has been purged and archived, so it is available as an archived file offline. To access an archived report you can make a request to the System Administrator to restore the report.

Not available: The report has been purged but not archived (and therefore is not available any more).

Report Type

Indicates the type of the report.
