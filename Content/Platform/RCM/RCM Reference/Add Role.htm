Add Role

Click the Add Role link within the Manage Roles activity bar to open the Add Role window that enables the administrator to setup permissions for the new role.

The Add Role window includes the following fields and options that enable you to define general information and behavior of the new role:

Option

Description

Role Name

Name assigned to the new role.

Role Identifier

The ID of this role. This string is used internally by the system and can be used when referring to a role.

The following characters are reserved and should not be used in the role ID:

:(colon)

;(semi-colon)

/(forward slash)

\(backwards slash)

'(upper tick)

Description

A description that provides more information about this role.

Apply this Role to all descending Business Units in the following Hierarchy: Select this option to select a relevant hierarchy to ensure that the permissions that are assigned to this role apply to the business unit (that is specified when you assign this role to a user) and all descendants of that business unit within the selected hierarchy.

For example, if you define a role called Supervisor and you assign this role to a user for the business unit USA, this role will also apply (for this user) for all descendants of USA such Eastern USA, New York etc.

You select the specific business unit that the role applies to when you assign the role to a user. (See Assign Roles per Business Units).

Select the relevant permissions and click to delete.

Filter By

Enter the search string to filter the permission list.

Select to view permissions that have not yet been saved. This setting does not affect the permissions list, it is for viewing purposes only.

Selected Count

Each option has a count of the number of permissions that you have selected, which is then calculated as a final count beside the Add button. For example:

Permission combinations are calculated according to the selections of an Item Type, then each Action for the Item Type, and each Assigned to selection for Item Type and Action. In the example, for the Item Type All, you can have the Action Add Attachments and Assigned to Any. As you add permission selections, the Selected count is increased.

Modules

In addition to the general information at the top of the window, the tabs on the left side of the window enable you to set up the required permissions. Each tab refers to a specific category of permissions (e.g., permissions for work items related tasks or report related tasks). A role includes all permissions that are added within each of the tabs.

The following module tabs that appear on the left side of the Assign Roles window enable you to assign permissions for the new role:

Option Description

Work Items

Assign permissions per Work Item Type. For example you can create a role for all users who are permitted to view only a certain Work Item Type, while a different role can be created for all users who are responsible for all work item types and can perform all actions on all work items types. For details, see Work Items Tab.

Reports

Assign permissions for this role per Report type. This tab enables you to determine which actions can be performed per Report type for a specified role. For details, see Reports Tab.

Blotters

Assign permissions for this role per Blotter type. This tab enables you to determine which actions can be performed per Blotter type for a specified role. For details, see Blotters Tab.

Cases

Assign permissions for this role per case type. This tab enables you to determine which actions can be performed per Case type for a specified role. For details, see Cases Tab.

Workflow Restrictions

Assign permissions for workflow templates you have created. For details, see Workflow Restrictions Tab.

Settings

 Assign permissions for this role for the various thresholds, lists and references that are included within the Settings Modules. For details, see Settings Tab.

Forms

Assign permissions for this role for the Form types that are included within the RCM Cases module. For details, see Forms Tab.

DART

Assign permissions for this role for the DART actions that can be performed. For details, see DART Tab.

Dashboards

Assign permissions for this role for the Dashboard actions that can be performed. For details, see Dashboards Tab.

Visual Link Analysis (VLA)

Assign permissions for this role for the VLA actions that can be performed. For details, see Visual Link Analysis (VLA) Tab.

Policy Manager

Assign permissions for this role for the Policy Manager actions that can be performed. For details, see Policy Manager Tab.

Platform List

Assign permissions for this role for the Platform Lists actions that can be performed. For details, see Platform Lists Tab.

Tuning

Assign permissions for this role for the Tuning actions that can be performed. For details, see Tuning Tab.

Admin

Assign permissions for this role for the actions that are available within the Administration module (e.g. you can allow one role to update Users and Groups only, while another role can update Users, Groups and Roles). For details, see Admin Tab.

Modules

Specify which modules (i.e. tabs that appear at the top of the application) appear for users that are assigned the current role. For details, see Modules Tab.

Other GUI Items

Enables setup of permissions for views and miscellaneous options within the application. For details, see Other GUI Items Tab.

Module Tabs

The tabs enable you to define permissions for a role. When users (who are assigned this role) access the specific module, they have use of those actions that you add to the permissions of this role.

For example:

You could create a role called Supervisors and allow users who perform this role to view all work item types, while a different role called Officers may be allowed only to view those Item Types that are relevant to their tasks.
