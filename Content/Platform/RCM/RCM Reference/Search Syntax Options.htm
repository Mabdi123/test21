Search Syntax Options

The following illustrates the various search options that can be used.

Regular Words

  • Search string: John
    Finds items that contain the word ‘John’.

Multiple Words

  • Search string: John Jane
    • Finds items that contain the word ‘John’ or the word ‘Jane’ or both.
    • Each word is optional, but at least one word is mandatory.
    • Higher ranking is given to items that contain all the words, or the entire expression.
Search words delimited by a space character create an "OR" search.

Mandatory and Prohibited Words

  • Search string: +John Jane
    Finds items that must contain the word ‘John’ and optionally contain the word ‘Jane’
  • Search string: +John -Jane
    Finds items that must contain the word ‘John’ and must not contain the word ‘Jane’. The restriction applies for the whole item, not per field.
  • Search string +John Jill -Jane
    Finds items that must contain ‘John’, may contain ‘Jill’, but must not contain ‘Jane’
  • Search string +John -John
    Finds no items.

Boolean Operators

  • Search string: John and Jane
    Finds items that contain both words. Basically the same as “+John +Jane”.
  • Search string: John or Jane
    Finds items that contain at least one of the words.

Phrases

  • Search string: “Jill Jones”
    Finds items containing the text “Jill Jones” as if it was one word.
  • Search string: “Jill Jones” John
    Finds items containing either ‘Jill Jones’ or ‘John’.

Search String by Specific Field

You can search according to fields that are designated in the in implementation, as follows:

Built In (Common) Fields:

  • identifier: Work item identifier. Example - identifier:01345677
  • type_name: Work item type full name
  • type_abbr_name: Work item type short name
  • bu_exact: Business unit of work item
  • owner: Identifier of work item owner. Example - owner:“John Smith”
  • owner_name: Name of work item owner
  • description: Description of work item
  • score_field: The score of the work item
  • note: The work item notes
  • audit: The work item audits
  • attachments: The work item attachments
  • rfi: The RFI text messages
  • priority_identifier: Priority of case items, possible values: low, medium, high.
  • alert_name: Names of all items (such as, work item, case item).
  • rfi_attachment: Attachments received from RFI replies.

Custom Fields:

Custom fields that are indexed in global search are dependent on your organization’s implementation. If you want to search according to other fields contact your administrator for a list of custom field identifiers.

Distance Factor

  • Search string “Jill Jones”~4
    Finds items containing the word ‘Jill’ and the word ‘Jones’ within a distance of 4 or less words.

Wild Cards

  • Search string: mith*
    Finds items containing words that begin with “mith”
  • Search string: h*o
    Finds items with the words ‘h2o’, ‘hello’, ‘halo’, etc.
  • Search string: h?0
    Finds items with the words ‘h2o’, ‘HBO’, etc.

Special Words for Filtering by Field

The word ‘has’ followed by ‘:’

  • Search string: has:notes
    Finds items that include notes.
  • Search String: has:owner
    Finds items that are assigned.
  • Possible values for this word are notes, attachments, owner, children, parents, cases, RFIs, deadlines, graphs.

The word ‘is’ followed by ‘:’

  • Search string: is:open
    Finds items in open state.

Possible values for this word: Open, closed, read, manual, issue, non_issue, no_determination

Boosting Results

  • Search String: John^2 Jane
    Finds items with the words ‘John’ or ‘Jane’. The term ‘John’ is boosted and items with this word will show up higher in the results.

Range Query

Range query can be done on a specific field. The word ‘TO’ in the brackets has to be in upper case characters

  • Search string: score_field:[10 TO 20]
    Finds items with score field between 10 to 20.
  • Search string: business_date:[NOW/DAY-7DAY TO NOW/DAY]
    Finds items from the last week, rounded to the day (hours and seconds are not counted)
  • Search string: business_date:[NOW-200DAY TO NOW-100DAY]
    Finds items with business date between 200 days ago to 100 days ago
  • Search string: business_date:[2007-03-06T00:00:00Z TO NOW]
    Using a specific date. Possible formats are:
    Without milliseconds: yyyy-mm-ddThh:mm:ssZ
    With milliseconds: yyyy-mm-ddThh:mm:ss.sssZ
  • Other supported dates include create_date, business_date, deadline_date, email_date, highlight_date, global_deadline_date, global_email_date, global_highlight_date, last_update_date, last_refresh_modified_date
Search words delimited by a space character create an "OR" search.
