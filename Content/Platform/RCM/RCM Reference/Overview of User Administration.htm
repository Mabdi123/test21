Overview of User Administration

The User Administration module enables authorized users to:

  • Manage Tenant information and Business Unit Hierarchy assignment when the multi-tenant features are enabled providing a high level grouping of all user information and data logically separating named organizations within the application.
  • View and Manage User Profiles, Groups, and Role definitions. (See About RCM Permissions page 1). This also includes managing the ongoing synchronization with External User directories when the application is integrated with an additional enterprise user repository.
  • Define Password Policies for authentication not managed through external Single Sign-On.  When SSO is being used, system users and others requiring direct access credentials must comply with defined password policies.
  • Maintain and apply Workflow Restrictions as a part of limiting use of defined workflow processes for work items applicable to a subset of the user roles.
  • Support operational needs of users through Managing Delegation of permission for time constrained delegation of application privileges.

Click the Administration link in the upper right corner of the RCM window that opens the User Administration module, labeled as Admin.
