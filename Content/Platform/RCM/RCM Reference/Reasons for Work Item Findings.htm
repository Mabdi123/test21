Reasons for Work Item Findings

This feature enables you to attach relevant reasons to specific steps you select for one or more work items. The reasons are based on findings from work item investigations.

Single Work Item Selection

Select a work item and click Change Step in the toolbar. The Change Step window opens displaying the Reasons tab.

When you select the Next Step for the work item the Reasons tab is displayed. The tab lists the available reasons for the selection of the next step in alphabetical order. You can also de-select a reason to remove the reason from the work item reasons.

If it is mandatory to select a reason when changing the step, an asterisk is displayed on the Reasons tab, meaning you cannot change the step without selecting a reason.

The selected reasons at the top of the list are those that were previously added for steps of the relevant work item.

The clock icon indicates which step the reason was selected for.

If there are no reasons for the selected work item then the Reasons tab is not displayed.

  • If the Reasons tab is displayed but the list is either empty or partially displayed, it may be because of your organization’s configurations.
  • For Reset and Recall actions there are only previously selected reasons displayed. These actions do not require you to select reasons. You can de-select reasons related to the step that you returned to during reset and recall.
  • The selected reasons can be viewed in the work item history. For details, see View History.

Multiple Selection of Work Items (Bulk Items)

You can select more than one work item and click Change Step to display the Change Step window. When you select a step for the Next Step field the Reasons tab displays only the common findings reasons for the selected work items. If you de-select a checkbox the reason is removed from the steps of all of the relevant work items.

Findings Reasons checkboxes can appear as follows:

Common Reasons : All of the selected work items have previously selected specific reasons.

Indeterminate: In this state some of the selected work items have common existing reasons (but not all have). When you select the next step, the system identifies that the common reasons that were selected for only some of the work items that are now common to all for the next target step. This reason is now found on the reasons list for the next step selection and displayed with a partially selected checkbox. Reasons that have been previously selected but are not common to all of the work items in the bulk selection, are not displayed and the following message appears:

<Number of reasons> Reasons that were previously selected are not listed because they are not common to all work items.

The Reasons tab may not appear if there is no platform reasons list defined.
