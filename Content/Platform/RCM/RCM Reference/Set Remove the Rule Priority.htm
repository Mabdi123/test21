Set/Remove the Rule Priority

This option enables you to define when the rule is executed in relation to the other rules. When priority rules are triggered, they override any other action regardless of the action’s severity. For example, if a rule stipulates that transactions performed by employees on a white-list should be passed (e.g., action = PASS TRANSACTION), then such transactions will pass even if some other rule recommends blocking the same transaction. A specific rule may be required to be processed before all other rules. Therefore it should be defined to be rule with order number 1.

To set a priority to a rule:

  1. Select the rule that you want to prioritize·
  1. Select the rule and then in the More actions drop-down menu click the Priority option. The number 1 is now added to the Order column for that rule.
  2. To prioritize the next rule, repeat the previous statuses. The number 2 is now added to the Order column for that rule.

All rules that are not prioritized are processed after the prioritized rules.

To remove the rule’s order number:

  1. Select the rule that you no longer want to prioritize·
  1. Select the rule and then in the More actions drop-down menu click the Non priority option.
  2. The order number is now removed from the Order column of that rule.
