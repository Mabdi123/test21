Save On-Demand Reports to the RCM Repository

The second step in the Report Generation wizard enables you to select the required output information for the report in order to save it to the database.

This step in the Report Generation wizard enables you to select your preference for the output of this report, as follows:

  • Generate to screen: generates the report instance on screen within a DHTML viewer only. This DHTML instance of the report is exactly the same as the instance that is created when you select Generate from the previous wizard screen. Once you close the viewer, the report no longer exists. This report will not be saved under any of the Report Views.
  • Save for further referencing: saves this report as a file within the RCM database. The report is saved to the format that is specified by the policy that is in effect for the report. Once it is saved as a file within the RCM database, this file can be opened from the Views pane of the Report Module (e.g., from the My Reports View). Once the report is materialized, the report opens in the format specified for materialization (according to the policy that is in effect). For more information about the Report Output options, see below.
  • Save and Generate to screen: generates this report within a DHTML viewer and saves this report as a file to the RCM database. The report is saved to the format that is specified in the policy that is in effect for the report and can be accessed again from the Views pane of the Report Module. Once the report is materialized (depending on the policy that is in effect) the report opens in the format specified for materialization.

The Report Output section of this screen includes the following options:

Option

Description

Report Name

A default report name automatically appears within this field. You can change this name to any other name, according to your needs. The report is saved with this name and can be opened again by selecting this name from the Report Views.

Business Unit

Select the business unit to which this report is to be distributed.

If one of the parameters for this report is mapped to a business unit, then this field is disabled.

Overriding Policy

Shows the default policy that was assigned for this Report Type. You can override this default policy and select any available policy from the drop-down list of policies. The selected policy will be implemented for the current report that you are generating (but does not apply to other reports based on the same template). A report policy determines factors such as:

  • If it requires Sign-Off.
  • When it should be materialized (frozen and published into a saved format, for example, pdf, html, etc).
  • When it should be archived.
  • If and when the report should be purged.
  • When notes can be added.

Business Date

Enables you to select the business date that is appropriate for the report that you want to save.
