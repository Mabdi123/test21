RCM Reference Guide Overview

Some modules are configured to appear within the application tabs (at the top of the screen) according to the permissions of the logged-on user. For example, administrative tasks are available only for users who have been assigned the role of administrator.

This manual is intended to provide a comprehensive description of all available features and options. However, since this application is customizable certain features that are described in this guide may not appear in the application with which you are currently working. In addition, certain features and functionality described in this manual are available only when licensed as part of specific Business Solutions, RCM, or with the version of additional Value Added Products. Some of the features and functionality may not be supported or available in your implementation, as they require a separate license.

For information about licensing please contact your Actimize representative.

This guide includes the following tabs and modules.

Tabs Options
Cases

Adding, viewing and managing groups of work items

Enables you to:

  • View, sort, and filter Case List views
  • View detailed information for cases
  • Create new and update existing
  • Add summary information, work items, notes, and files to a case for further investigation
  • Assign cases to user and business units
  • View the history of all actions performed on cases
  • Send information on a case by email
  • Print a list of cases and print case details
Workbench

Includes modules that are viewed and managed on a regular basis.

 Work Items

Viewing and managing work items

  • View, sort, and filter list views
  • View detailed information for work items
  • Assign each work item a step
  • Add notes to work items
  • Create manual work items
  • Assign a work item to a case
  • View the history of all actions performed on work items
  • Export work item details to external files, send information by email or prepare a PDF version of work items
  • Print a list of work items and print work item details

 

 

 

 Reports

Enables you to:

  • View management reports that provide statistics about work items, workload and other activities
  • View and sign off those reports that require this process
  • Add notes to reports and view history of notes associated with the report
Research

Includes modules that are related to reporting, research and investigation.

 DART

Enables you to create, save, access, and run queries on external tables that reside outside the RCM repository

On-Demand Reports

Generate management or other parameterized reports based on predefined templates

Ad-hoc Blotters

Enables you to perform an ad-hoc review of the overall transaction activity across blotters.

Tuning

Enables defining different thresholds and configurations in order to test them.

Settings

Includes modules that enable authorized users to adjust setting values required during work item detection.

Settings Navigator

Includes a folders hierarchy for the currently viewed solution, enabling users to:

  • Quickly navigate to specific setting(s) via the Settings Tree.
  • Find setting(s) in the Settings Tree via its filter utility.
  • Add/remove row(s) to a specific setting.
  • Edit values in a setting.
Thresholds
  • Providing input to the AIS models that are responsible for work item distribution. Authorized users can:
  • Set threshold values that are used by the AIS model during analysis
  • Set scoring scales and score values that are used when calculating scores
Lists
  • Enabling users to manage lists, according to the needs of their organization. Authorized users can:
  • Define exclusion and inclusion lists
  • Import data from CSV files
  • Export data to CSV files
Platform Lists
  • Enabling you to manage and maintain various thresholds and scoring as required by your specific solution.
References
  • Read only reference information that application end-users would find useful in the course of their work. Reference data may include processing statistics and definitions on standard reference codes.
Admin

Managing Users. Groups & Roles
  • Add and remove users/groups
  • Maintain user details
  • Assign users to groups
  • Assign groups to groups
  • Define roles that users perform and assign actions and views that are appropriate for those roles
  • Set up permissions by assigning roles to users and groups, according to business units
