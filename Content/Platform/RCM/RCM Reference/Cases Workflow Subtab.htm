Cases Workflow Subtab

A predefined workflow may be defined for each role that has permissions to work on cases. This workflow determines the steps that can be assigned to a case at each stage of the resolution of the case.

For example, there may be a workflow defined for a specific role that restricts the step assignment and ensures that the user cannot progress from a Ready step to any Closed step.

The Workflow page displays the following options and data:

  • An option for moving to the next appropriate step
  • The current step that the case is in (colored green). In the example above the current step is Under Investigation. If steps were already performed, you can also see these steps (shown as inactive).
  • A tasks table, including
  • The tasks that are to be performed during this step.
  • The reviewer’s name (when relevant)
  • The date on which the task was performed.

A workflow may not be available for all case types
