Reports Tab

The following is an example of the Reports tab in the Update Role window.

Select the Report Type or all Report Types in the Report Type drop-down menu.

You may allow Supervisors to perform all actions in the Report module (edit, delete etc.) while someone who is assigned a different role may only have permission to view reports without permission to perform any actions.

To add a permitted action, select one of the following actions from the Actions drop-down menu and click the Add button:

  • All - enables users with this role to have all the available permissions for reports.
  • Assign to User - enables users with this role to assign a report to a specific user.
  • Assign to User/BU - enables users with this role to assign a report to a specific user/business unit.
  • Delete - enables users with this role to delete a report.
  • New Report - enables users with this role to create a new report.
  • Open - enables users with this role to view/open the list of reports in the Workbench-Reports tab.
  • Record note - enables users with this role to add a note to a report.
  • Show History - enables users with this role to view the history of actions for a report.
  • Sign off - enables users with this role to sign off a report.
  • View - enables users with this role to view the list in the Workbench-Reports tab. Note that the Open permission enables you to both view the list of reports and open specific reports.
  • View Notes - enables users with this role to view the report’s note.
  • View Info - enables users with this role to view the report’s data.

You have the option to set a permission to one of the following in the Assigned to drop-down menu:

 

  • Any: Reports that are assigned to any user. This is the default setting.
  • Assigned to me: Reports that are assigned to you.
  • Assigned to others: Reports that are assigned to other users.
  • Unassigned: Reports that are not assigned to any user.

To add another permitted action, select it and click the Add button. You also have the option to select another Report Type and set the desired permitted actions for it.
