Case Items List

The following displays an example of a Case Items list.

You can sort the list according to the contents of a selected column by clicking that column’s title (For example, click Item Date to sort the list by date, or click Step to view all cases sorted by the Step). Typically, there is one column within the Case Items list that is a link that lets you access the Case Item Details for the corresponding case (for example, Item ID is the link to the Case Item details).

The columns that appear within this list are configured according to the needs of your organization and the permissions that are currently in effect. As a result, the options that you see may not be the same as the standard list presented below.

Some of the typical columns that appear within the Case Items List are:

Item Name

The name of the case item given when creating the item.

Item ID

The unique identifier of this case item. This number is used internally by the system and can be used when referring to a case item.

When this Item ID appears within the list as a link, you can click the link to view the Item Details for the corresponding case item. The Work Items Details window opens enabling you to view various levels of details about this work item, often including links to related work items, when relevant. (See Case Item Details).

Item Date

The date when the case item was created.

Item Type

The AIS model that generated the case item (For example; Best Execution —BEX or Front Running — FR) according to a rule or standard that caused the alert to be triggered. The list of work items types is defined according to the needs of your organization. 

Step

The current step of the case item. Steps are defined according to your organization’s needs. For example, a typical step definition may be Ready or In Process.

Owner

The name of the user responsible for investigation of this type of case item. When you select to view only My Case Items, the user name is always your name.

Priority

Enables you to select the priority for this case item. Following are the priority options:

  – red represents high priority

  – yellow represents medium priority

  – green represents lowest priority

Score

A numeric value that enables you to evaluate the severity of this item. Score is not supported for case items.

Description

A summary of the case item details.

Original BU

Indicates the original business unit that this case item was first assigned to. This is useful when the alert was reassigned to a different business unit.

Business Unit

Indicates the full name of the current business unit, as it is displayed within the work items and reports.
