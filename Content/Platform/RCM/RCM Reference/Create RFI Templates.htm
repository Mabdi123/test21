Create RFI Templates

You can create RFI templates to then use when you create and send RFI emails relating to work items. Click the Create RFI Template link within the Manage RFI Templates activity bar to open the Create RFI Template window.

The Create RFI Template window contains the following fields:

  • Template Name: The name for the template which can then be used in the Create RFI window. See Create RFI. (This field is mandatory)
  • Identifier: A unique identifier for the template. (This field is mandatory)
  • Template Details Tab: Contains the parameters for creating a template
  • RFI Name: The name of the RFI email.
  • To, CC, Bcc: The email addresses of the recipients in one or more of these fields. These fields are identical to email message fields. Use a ';' to separate between addresses.
  • Subject: The subject for the email.
  • Attach item data as PDF: Select to include the relevant work item details in the email. This checkbox remains selected for this template each time you select it for an RFI.
  • Attach all work item attachments: Select to include all work item attachments in the email. This allows you to add attachments automatically without manually selecting each one. This checkbox remains selected for this template each time you select it when creating an RFI. The default is deselected.
  • Message: Text box for entering data to the recipients on the RFI. Identical to the body of an email message.
  • Item Type Associations Tab: Enables you to associate the template to work item types. Each template must be associated with an item type. This enables you to attach only relevant templates to work items of a specific type.

  • All Item Types: Associates the template with all work item types. This is the default selection.
  • Selected Item Type: Displays the list of item types. Select one or more work item types to be associated with the template.

Click Save to save the RFI template details.

Using two number signs (##) in a row of text in the RFI body, deletes the row.

VTL in Templates

You can create a template using VTL that then uses the relevant work item details when you create an RFI.

The context for the VTL is the work item common fields and custom fields. The syntax is as follows:

  • For extracting supported common fields and custom fields: $fields.get(<fieldidentifier>).
    For example: $fields.get("alertId")

Supported Common Fields:

alertDate

alertId (identifier)

alertTypeIdentifier

alertOriginalBusinessUnitIdentifier

buIdentifier

closedDate

createDate

deadline_date

details

email_date

lastReadUser (read user full name)

lastUpdateUser (update user full name)

highlight_date

lastReadDate

lastUpdateDate

alertOriginalBusinessUnitName

alertTypeName

businessUnitName

score

alertTypeShortName

statusIdentifier

status

alertTypeCategory

owner (owner full name)

ownerLoginName (owner login name)

ownerExternalIdentifier

statusIssue

statusState

  • For extracting user information the VTL syntax is:
    $userInfo.get("buName") or $userInfo.get("fullName") or $userInfo.get("loginName").

  • You can also extract user attribute information based on the user attributes xml file. For more information, refer to the RCM Self Developer Guide. For user attributes, the syntax is:

  • For extracting business unit attribute information the VTL syntax is: $alertBU.get(“BU_ATTR_ID”)

VTL can be used in all fields of the RFI template, except for Template Name and Identifier.

Relations in VTL

You can also create templates that apply to relations of entities in work items. For example, if a person has more than one account you can connect to several accounts related to one person.

Example:

first relation name: $relations.get("boss-of","companyx_Person").get(0).get("companyx_FIRST_NAME")

You can also connect to relations in a loop, meaning that all available relations are included.

Example:

#foreach( $relation in $relations.get("boss-of","companyx_Person"))

          Relation name: $relation.get("companyx_FIRST_NAME")

#end
