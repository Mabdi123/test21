Properties (Report Information)

Select a report and click the Properties link in the More menu to open a page with a summary of the report information for the selected report.

This action may appear directly in the toolbar. Buttons and action links are customizable depending on your organization implementation.

The information that appears within this page is configured according to the needs of your organization. The following provides examples of information that may appear within the Report Information page:

Option

Description

Report Name

The name of the selected report

Report ID

The ID number of the selected report. This should be a unique Id that can be used by external applications for reference.

Status

The current status of the selected report. The status can be:

New: The report has not yet been viewed or dealt with by any user.

Purged: The RCM has purged the report according to the policy that is in effect for that report. Purging a report instance is the act of removing the materialized file from the . After a report is purged you can still audit report information. You will not be able to view the content of the report.

Archived: The materialized report instance, along with supporting data such as notes, has been copied to an external file system. (Report instance materialization is when the report becomes a static file managed by . A report instance must first be materialized before archiving is possible.) Once archived, the files are managed by a third party document management system or as part of other file system backup processes.

Signed-off: The report has been signed off by the responsible user.

Viewed: The report has been opened and viewed (i.e., is no longer New). Note that a report has the status of Viewed only until some other action has occurred, such as signing off, or archiving.

Generated By

Indicates whether the report was automatically generated, as a scheduled report, or was generated On Demand by a user. Automatically generated reports can be generated either by the AIS or, during the implementation stage, by the Designer.

Creation Date

The date on which the query for the required report was run.

Business Date:

The date when the actions occurred

Bu Name

The business unit to which this report is currently assigned.

Owner Name

The user to whom this report is currently assigned (i.e. responsible for this report).

Sign-Off Date

The date on which this report was signed off (if applicable).

Sign-Off User

The user who is responsible for signing off this report (if applicable).

Sign-Off Policy

Indicates whether signoff is required or not.

Archive Date

If the report was already archived, this shows the date of archiving by the system.

Archive Policy

The archive policy determines the rule according to which the report will be archived (i.e., when the materialized report instance, along with supporting data such as notes, has been copied to an external media).

For example, the archive policy may stipulate that the report will be archived 5 days after Sign Off, or 3 days after the report is opened by the user. When a report is generated on-demand, the Archive Policy is set according to the policy that is in effect for the selected Report Type. The user can over-ride this policy set when saving the generated report.

Materialization Format

Indicates the format in which the materialized instance of the report will be saved. The materialization format can be, for example, PDF, Excel, or HTML.

Materialization Policy

Indicates when the report will be materialized in the specified format. For example, the policy may stipulate that the report will be materialized just before archiving or after Sign-Off.

Materialized

A report instance may be materialized at any point from the initial generation of the report instance through to its archival. Report Instance materialization is when the report becomes a static file managed by .

A report is materialized (saved as a static file) to the format that is defined within the materialization policy (see above).

Notes Policy

Indicates when you can add notes to the report (for example, only until archived, or only for a specified number of days after Sign-Off).

Policy Name

Shows the name of the policy that was used for generating the current report.

Purge Date

Indicates the date that the current report was purged. Purging a report instance is the act of removing the materialized file from the and leaving the metadata (for example, report information data) intact.

Purge Policy

Indicates the policy, which is in effect for the current report, that determines when the report should be purged (when the materialized file is removed from the ).
