Policy Manager Rules

In the Policy Manager you can create the rules that determine the score value of the match level of the existing entity compared to the incoming entity. The rules compare fields from the incoming entity to fields from an existing entity.

For details on policy manager, see Policy Manager.

Rule Conditions

You can create rules based on policy types such as, person or account, using the relevant fields. This enables you to set criteria for the system to find potential matches for incoming entities. Entity matching rules must compare incoming and existing entity fields.

Rule Operator

The Is similar to operator for the Entity Matching feature, enables you to check if two fields are similar to each other. This allows you to compare two fields with a maximum of two different characters.

For example if an incoming full name has a similar spelling to an existing full name, as follows:

  • Full name of existing: John Doe
  • Full name of incoming: Jon Do

In this case the rule is triggered and the pair is given a score.

Rule Scores

After you have set a rule condition, you set rule scores in the Actions tab below the rule criteria. A score is given according to the action you want taken on entities that meet the rule criteria. Rule scores can be defined by tens (10, 20, 30 and so on). Score thresholds are set in the platform lists as follows:

  • Entity match is between 90-100
  • Doubt is between 20-80
  • No match is between 0-10

These are the default Entity Matching thresholds that are customizable in the Entity Matching platform list thresholds.

If a rule states that for the person policy type, the incoming full name must equal the existing entity full name, then you might set a score of 90. This means that if the incoming entity has the same full name as an existing entity, it gets the score of 90 and is a match.

For details on the platform lists, see Entity Matching List Thresholds.

Rule Priority

You can also set rule priorities which designate specific rule criteria as more important and thereby making the entity with a match according to the criteria, a likelier match. For details, see Set/Remove the Rule Priority.

Expression Builder

In the Policy Manager expression builder you can create functions using strings only. In addition you can create Entity Matching functions for use with relation sets, such as a group of addresses that belong to a single entity.

You can set the following functions:

  • ERCM_SETS_ARE_IDENTICAL: All details in the incoming entity must match all details in the all of the existing relation sets.
  • ERCM_SETS_SHARE_ITEM: The incoming entity can share at least one of the strings of the existing set.

For system purposes all data types in policy manager rules are defined as strings.
