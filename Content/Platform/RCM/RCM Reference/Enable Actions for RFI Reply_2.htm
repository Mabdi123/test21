Enable Actions for RFI Reply

You can create an RFI template with placeholders that enable you to create links in RFIs with specific responses based on work item information. When you create and send the RFI, the response then can trigger the system to change a step in a workflow.

The following is the syntax for the reply placeholder.

$acmMail.mailtoLink(<display link text>,<target step identifier>,<body>,<subject {optional}>)

The received email will have a link instead of the placeholder. The text of the link is defined in the display link text parameter

The parameter values are as follows:

  • display link text: The link text as displayed in the mail when HTML is available. You cannot use the “}”, ”{ characters
  • target step identifier: The target step that you want the reply to enable the change.
  • body: Free text, the body of the reply email.
  • subject: Free text, the subject of the reply email (optional - the default subject will be the subject of the reply email)

$acmMail.mailtoLink(“approve to buy”, “approve”, “Thanks for approving”, “approval request”)

A value of the display link text parameter can only be used once in a specific template.

For details on creating the RFI, see Create RFI.
