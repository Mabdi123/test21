Policy Manager Tab

The following is an example of the Policy Manager tab in the Add Role or Update Role window.

Select the AIS Policy Type or all AIS Policy Types in the Policy Manager drop-down menu.

To add a permitted action, select one of the following actions from the Actions drop-down menu and click the Add button:

  • All – enables users with this role to have all the available permissions for the selected AIS Policy Types.
  • Activate Policy - enables users with this role activate an AIS Policy Type for the selected AIS Policy Types.
  • Change Activation State - enables users with this role to change the activation state of an AIS Policy Type for the selected AIS Policy Type(s).
  • Change Priority State - enables users with this role to change the priority state of an AIS Policy Type for the selected AIS Policy Type(s).
  • Create New Rule - enables users with this role to create a new rule of an AIS Policy Type for the selected AIS Policy Type(s).
  • Delete Rule - enables users with this role to remove a rule of an AIS Policy Type for the selected AIS Policy Type(s). (Also includes the Edit Rule permission)
  • Edit Rule - enables users with this role to edit a rule of an AIS Policy Type for the selected AIS Policy Type(s). (Also includes the Delete Rule permission)
  • Toggle Rule Production Mode - enables users with this role to change a Production Mode Rule of an AIS Policy Type to a Test Mode Rule for the selected AIS Policy Types
  • View Activation History - enables users with this role to view the Activation Rule History of an AIS Policy Type for the selected AIS Policy Types
  • View Active Tab - enables users with this role to view the Active tab of an AIS Policy Type for the selected AIS Policy Types
  • View Draft Tab - enables users with this role to view the Draft tab of an AIS Policy Type for the selected AIS Policy Types
  • View Rule - enables users with this role to view a rule of an AIS Policy Type for the selected AIS Policy Types
  • View Rule History - enables users with this role to view the rule history of an AIS Policy Type for the selected AIS Policy Types

In order to delete a rule a user is required to have the View Draft Tab and View Rule permissions, as well as the Delete Rule permission.

To add another permitted action, select it and click the Add button. You also have the option to select another AIS Policy Type and set the desired permitted actions for it.
