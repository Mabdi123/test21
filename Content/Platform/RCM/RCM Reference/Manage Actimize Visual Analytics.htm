Manage Actimize Visual Analytics Users and Roles

If the Actimize Visual Analytics (AVA) license has been acquired and activated, the administrator can create and manage AVA users from RCM.

To create an AVA user from an RCM user:

  1. In the Admin page, select the relevant user.
  1. In the User Management activity bar, click the Edit User link.

    2. The Update User window opens.

  1. Grant the user AVA access by selecting the User has access to AVA checkbox. The AVA Role menu then appears.
  1. Select whether to grant the user the Publisher or Interactor role, as follows:
    • Interactors can view and interact with the AVA dashboards in real time.
    • Publishers can view and interact with the AVA dashboards in real time, and create new dashboards for publishing on the AVA server for general access.

    2. Clicking the Sync button performs another attempt to update the user’s role on the AVA server.

Once the user has been created on the AVA server, the user's attributes can be changed in RCM and updated (upon synchronization) on the AVA server, but the User identifier attribute must not be changed.

To change a user’s AVA role:

  1. In the Admin page, select the relevant user.
  1. In the User Management activity bar, click the Edit User link. This opens the Update User window.
  1. Use the AVA Role menu to select the new role to assign to the user.
  1. Click Save. The user’s role is updated on the AVA server. In the event of an error creating the user on the AVA server, the icon appears next to their role in the AVA Role column.

    2. Clicking the Sync button performs another attempt to update the user’s role on the AVA server.

If the user being updated in RCM is an AVA site administrator and their AVA role in RCM is changed to Interactor or Publisher, their user role on the AVA server will remain as site administrator.

To remove a user’s AVA role:

  1. In the Admin page, select the relevant user.
  1. In the User Management activity bar, click the Edit User link. This opens the Update User window.
  1. Unselect the User has access to AVA checkbox.
  1. Click Save.

    2. The user’s role is updated on the AVA server. In the event of an error removing the user from the AVA server, the icon appears next to their role (which now displays “removed”) in the AVA Role column.

    Clicking the Sync button performs another attempt to update the user’s role on the AVA server.
