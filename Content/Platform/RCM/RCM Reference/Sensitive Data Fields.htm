Sensitive Data Fields

Your organization may configure specific fields' data to be partially or completely hidden for viewing by specific users. For example, a field displaying a credit card number may display only the last four digits of the number. Alternatively, a constant string may be displayed instead of the data (N/A, Sensitive Data, etc.).

When you view the work item's history of actions (audit), the sensitive data fields are always masked, regardless of your permissions. If you do not have sensitive data fields' permissions then you can edit only the non-sensitive fields of the work item.
