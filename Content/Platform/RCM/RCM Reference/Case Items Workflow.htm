Case Items Workflow

A predefined workflow may be defined for a type of case item. This workflow determines the steps that the user should follow in order to close the case item.

For example, there may be a workflow defined for a specific case item that restricts the step assignment and ensures that the user cannot progress from a Ready step to a Closed step.

  • The Workflow window displays the following options and data:
  • An option for moving to the next appropriate step
  • An option for toggling between the Workflow window and Case Item Details window
  • The current step that the case item is in (colored green). In the example above the current step is Open-Issue. If steps were already performed you can also see these steps (shown as inactive).
  • A tasks table, including
  • The tasks that are to be performed during this step.
  • The reviewer’s name (when relevant).
  • The date on which the task was performed.

The workflow may not be available for all work item types
