Enable Actions for RFI Reply

You can send an RFI using an RFI template created with placeholders that request a reply which automatically changes a step in a workflow.

The Create RFI window appears as follows:

The received email will have a link instead of the placeholder. For example:

The text of the link appears as defined in the display link text parameter. You can move, duplicate, delete but not edit the placeholder text in the Create RFI window. You can also add the placeholder in the same format as it is defined in the RFI template, directly to the body of the email.

The RFI recipient clicks one of the relevant links and a reply mail is created in a new window (without any previous correspondence included) and sent to the system. The workflow step is changed according to the specific reply.

For details on creating a template with relevant links, see Create RFI Templates.

If the workflow step has already been changed from the original step where you configured the change, or the original step was changed, the RFI reply is received but does not change the step.
For information refer to the Implementer Guide. If the recipient does not have rich text enabled, the link options are displayed in html.
If the mail is sent to more than one address, then the system acts on the reply that is received first.
